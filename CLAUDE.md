# 项目规则

## 性能优化：减少工具调用次数

核心瓶颈是大模型推理等待时间，不是工具执行时间。每次工具调用触发完整LLM推理周期（3-10秒），工具本身执行只需毫秒级。

必须遵循以下原则：

1. **批量检查代替逐个检查** — 用一次grep正则批量匹配所有目标，禁止多次分别验证
2. **把验证逻辑写进脚本** — 脚本内自带验证输出，一次执行完成，不要写完再单独验证
3. **减少确认性重复检查** — 信任工具返回结果，不要反复读文件确认内容
4. **并行执行无依赖的工具调用** — 多个独立的grep/读取操作必须并行发出
5. **一次性写完再执行** — 完整脚本一次写完一次执行，禁止写一行执行一行看结果
