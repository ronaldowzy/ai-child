## 2. 图标接入要求

### 2.1 Android 图标规范

开发端应按 Android 图标规范生成：

```text
1. adaptive icon foreground；
2. adaptive icon background；
3. legacy launcher icons；
4. round icon 如项目当前使用；
5. mipmap 密度资源。
```

如果项目已有图标生成脚本或资源目录，优先复用。

### 2.2 视觉处理要求

图标接入时必须注意：

```text
1. 小白狐脸部不能被系统圆形/方形/水滴形 mask 裁掉；
2. 耳朵顶部要留安全边距；
3. 下方爪子和云朵不能贴边；
4. 小尺寸下眼睛和脸部仍清楚；
5. 不能出现白边、黑边、透明边异常；
6. 图标不能有文字、logo、水印；
7. 不要棋盘格背景；
8. 不要把源图重复打包成多份大图。
```

### 2.3 建议处理

建议：

```text
1. 先从源图生成一套 launcher icon 资源；
2. 在 xxhdpi、xxxhdpi 和系统圆形 mask 下检查裁切；
3. 如耳朵被裁切，先缩小主体或扩大安全边距；
4. 如小尺寸糊，适度锐化或优化缩放；
5. 图标资源应可控，不引入数 MB 以上无用源文件。
```

## 4. 整体真机复测范围

图标接入后，需要做一轮整体真机复测。

### 4.1 启动与首页

```text
1. App 图标在桌面显示是否清晰；
2. 圆形/方形/系统主题图标下是否被裁切；
3. 打开 App 后小白狐是否是第一视觉；
4. 首屏是否不像普通聊天页；
5. 小白狐舞台是否温暖、干净、不拥挤。
```

### 4.2 语音与等待

```text
1. 低压等待态按钮是否仍是“按一下开始说”；
2. 真实录音态是否才显示“说完了”；
3. 小白狐 speaking/listening/waiting_soft 状态是否切换正确；
4. 首段语音是否明显更快；
5. 文字是否先于语音展示；
6. 停一下是否能停止当前播放或迟到语音；
7. paused 是否过早压掉 listening/speaking。
```

### 4.3 普通聊天

```text
1. 普通短句聊天是否响应更轻；
2. prompt fast path 是否没有影响回复质量；
3. 回复是否短、自然、不追问；
4. 消息区是否不再压过小白狐；
5. 空消息时是否没有重复提示。
```

### 4.4 拍图

```text
1. 拍图入口是否仍像“给小白狐看看”；
2. 上传中是否显示“正在给小白狐看看”；
3. 成功/处理中是否显示“小白狐正在看”；
4. 失败时是否显示“这张图还没给小白狐看到”；
5. 失败时是否没有共创入口；
6. 图片预览卡是否不压过小白狐；
7. image_viewing/retry 状态是否正确。
```

### 4.5 横屏/平板

```text
1. 横屏小白狐是否仍是主要视觉；
2. 消息区是否没有变回聊天窗口；
3. 图片预览是否不挤压小白狐；
4. 底部操作是否仍清楚；
5. 小白狐动图是否被裁切。
```

### 4.6 家长日报

```text
1. 家长日报是否只展示三块：今天一句话、孩子今天提到的内容、需要留意的地方；
2. 是否没有“今晚可以这样接一句”；
3. 是否没有家长话术建议；
4. 内容不足/失败/生成中状态是否清楚；
5. 是否不展示 raw transcript。
```
