# App 界面重设计方向评审与落地收口 v0.1

项目：`ai-child`

用途：作为下一阶段第 4 轮“App 界面重设计与落地”的主控评审收口文档。本文基于 image-2 生成的 4 张界面方向图整理，用于指导开发端做最小可落地界面改造。本文不是要求 1:1 复刻生成图，也不是启动大 UI 重构。

更新时间：2026-05-28

---

## 0. 主控结论

本次界面重设计方向成立。

保留核心判断：

```text
儿童页应从“有小白狐的聊天页”进一步变成“以小白狐为第一感知对象的陪伴页”。
```

第 4 轮开发落地不追求还原生成图里的真实 3D 房间、复杂背景或完整视觉资产，而是优先把当前 Android Compose 页面调整到以下体验：

```text
1. 小白狐是第一视觉；
2. 语音主按钮是第一操作；
3. 拍图入口表达为“给小白狐看看”；
4. 消息区轻量化，不抢主视觉；
5. 等待、看图、准备说、停一下等状态更清楚；
6. 常驻轻共创入口低压出现，且同一张图最多一个共创入口；
7. 家长入口弱化但可达。
```

禁止把本轮做成：

```text
1. 大 UI 重构；
2. 游戏大厅；
3. 学习平台；
4. 作品库；
5. 故事库；
6. 任务中心；
7. 奖励系统；
8. 新角色或新宠物养成系统。
```

## 1. 四张方向图评审

### 1.1 图 1：竖屏主界面

#### 可保留

```text
1. 小白狐作为竖屏第一视觉，位于画面中上方或中部；
2. 底部大语音按钮清晰、可触达；
3. “给小白狐看看”表达自然，明显优于“上传图片”；
4. “换个话题”“停一下”作为低压控制入口是正确的；
5. 整体柔和、低刺激、陪伴感强。
```

#### 需调整

```text
1. 消息气泡数量要更少，避免重新变成聊天页；
2. 底部操作按钮优先级要收敛：语音第一，拍图第二，换题/停一下第三；
3. 家长入口继续弱化，不可成为右上显眼功能按钮；
4. 不要求开发生成完整 3D 房间背景，可用柔和渐变和现有小白狐资源近似。
```

#### 主控结论

图 1 是竖屏主界面的主参考方向。

---

### 1.2 图 2：横屏主界面

#### 可保留

```text
1. 横屏左侧小白狐，右侧消息和输入区的双栏结构正确；
2. 小白狐视觉存在感强，适合平板或横屏手机；
3. 右侧操作区相对集中，语音、拍图、停一下、换话题容易理解；
4. 底部“陪你慢慢说”这样的陪伴提示方向可保留。
```

#### 需调整

```text
1. 右侧消息区仍略像普通聊天页，消息数量要减少；
2. 小白狐状态气泡应比历史聊天更重要；
3. 横屏下不要把右侧做成大聊天记录，最多展示最近 2-3 条；
4. 语音主按钮应保持中心地位，不被聊天内容压下去。
```

#### 主控结论

图 2 是横屏布局的主参考方向，但开发落地时要降低聊天页感。

---

### 1.3 图 3：拍图后“小白狐正在看”状态

#### 可保留

```text
1. “已经给小白狐看啦”状态非常清楚；
2. 本地图片预览卡可以保留；
3. 小白狐处于看图/思考状态，能让孩子知道小白狐正在处理；
4. 起名或讲一句小故事作为低压共创入口方向正确。
```

#### 必须调整

```text
1. 图片卡片不能过大到抢走小白狐主视觉；
2. 同一张图不能同时展示“起个名字”和“讲一句小故事”两个主动入口；
3. 必须遵守第 3 轮规则：同一张图最多一个共创入口；
4. 不出现历史作品、作品库、作品列表；
5. 不复述具体图片隐私细节。
```

#### 主控结论

图 3 的“看图状态 + 本地预览 + 单个低压共创入口”可落地；两个共创按钮同时出现必须删除，改为一次只展示一个建议。

---

### 1.4 图 4：小白狐等待孩子接话状态

#### 可保留

```text
1. “我在听”“你想说的时候再说，不说也没关系”的等待感正确；
2. 没有倒计时，没有催促；
3. 语音按钮保持自然可说话状态；
4. 小白狐表情温和，符合等待和陪伴状态。
```

#### 必须调整

```text
1. 等待状态不要主动抛新共创问题；
2. “要不要给这朵云起个名字？”只能在轻共创适用场景出现，不能作为普通等待态文案；
3. 等待态文案应更短，更偏“我在听 / 不说也没关系”；
4. 等待态不能显得像在要求孩子回应。
```

#### 主控结论

图 4 的等待气质可保留，但上方主动起名气泡不作为默认等待态。

## 2. 第 4 轮落地目标

第 4 轮开发只做“现有界面结构内的可落地改造”，不做完整视觉重构。

目标：

```text
1. 竖屏：小白狐主视觉更明显，消息区更轻，语音按钮更像第一操作；
2. 横屏：保持左狐右聊，但降低聊天记录存在感；
3. 拍图后：图片预览状态更清楚，“小白狐正在看”更明显；
4. 等待时：孩子能看出小白狐在听，但没有压力；
5. 快捷动作：保留“换个话题”“停一下”“给小白狐看看”，并降低任务感；
6. 轻共创：拍图后最多一个入口，且只在适用场景出现。
```

## 3. 建议落地结构

### 3.1 竖屏结构

建议结构：

```text
顶部：弱化家长入口 + 小白狐标题；
中上：小白狐主视觉区；
中部：小白狐状态气泡 / 最近一条核心回应；
中下：轻量消息区，最多强化最近 1-2 条；
底部：语音主按钮居中；
底部两侧：换个话题 / 停一下 / 给小白狐看看。
```

要求：

```text
1. 小白狐主视觉区不小于当前占比；
2. 消息区不再抢小白狐视觉；
3. 输入区不显示复杂工具栏；
4. 拍图入口文案优先是“给小白狐看看”；
5. 保留文字输入能力可行，但视觉上不应成为第一操作。
```

### 3.2 横屏结构

建议结构：

```text
左侧 50%-55%：小白狐主视觉区；
右侧 45%-50%：状态气泡 + 最近消息 + 语音/拍图操作；
底部或右侧中下：语音主按钮；
家长入口放右上或弱化角落。
```

要求：

```text
1. 不把右侧做成密集聊天记录；
2. 最近消息优先，历史消息弱化；
3. 状态气泡优先于消息列表；
4. 小白狐不能被右侧聊天区挤小。
```

### 3.3 拍图后状态

建议结构：

```text
小白狐主视觉仍保留；
图片预览卡在中部或小白狐下方；
状态条：已经给小白狐看啦 / 小白狐正在看；
只展示一个低压共创入口：起个名字 或 讲一句小故事；
保留“先不要 / 换个话题 / 停一下”。
```

要求：

```text
1. 图片预览卡不要超过主视觉；
2. 同一张图最多一个共创入口；
3. 不展示历史图片；
4. 不出现作品库、作品列表、我的作品；
5. 图片失败状态继续使用温和、非技术化表达。
```

### 3.4 等待孩子接话状态

建议结构：

```text
小白狐轻轻听着；
状态气泡：我在听 / 想说的时候再说；
语音主按钮可用；
无倒计时；
无催促；
等待结束自然收回。
```

要求：

```text
1. 不出现“快说呀”“轮到你了”；
2. 不默认发起共创问题；
3. 不把等待状态做成录音监听感；
4. 视觉要表达“可以说”，不是“必须说”。
```

## 4. 视觉与组件原则

### 4.1 可落地视觉方向

```text
1. 背景使用柔和渐变、轻微云朵/星星/纸感即可；
2. 不要求完整 3D 房间背景；
3. 小白狐继续使用现有状态图或已有资源，不新增大规模美术依赖；
4. 圆角卡片、柔和阴影、低饱和色；
5. 蓝白、奶白、浅黄、浅紫可作为主视觉感受；
6. 避免高刺激霓虹色和复杂装饰。
```

### 4.2 组件优先级

从高到低：

```text
1. 小白狐主视觉；
2. 小白狐当前状态；
3. 语音主按钮；
4. 给小白狐看看；
5. 最近消息；
6. 换个话题 / 停一下；
7. 家长入口。
```

### 4.3 文案原则

可用方向：

```text
小白狐在这里。
我在听。
想说的时候再说。
不说也没关系。
给小白狐看看。
已经给小白狐看啦。
小白狐正在看。
起个名字。
讲一句小故事。
换个话题。
停一下。
```

避免方向：

```text
上传文件。
立即开始。
完成任务。
今日挑战。
连续创作。
作品库。
故事库。
学习计划。
你怎么不说话。
明天继续。
```

## 5. 开发边界

第 4 轮开发不得做：

```text
1. 不做全量 UI 重构；
2. 不引入新的导航层级；
3. 不新增游戏、任务、作品库、故事库；
4. 不新增成长档案；
5. 不新增复杂家长端；
6. 不新增大规模美术资源依赖；
7. 不新增长期记忆或画像能力；
8. 不修改语音、拍图、轻共创后端主逻辑，除非为 UI 状态展示做极小适配。
```

建议优先改造范围：

## 6. 验收标准

### 6.1 竖屏主界面

通过标准：

```text
1. 第一眼看到的是小白狐，不是聊天记录；
2. 语音主按钮清楚；
3. 拍图入口像“给小白狐看看”；
4. 消息区存在但不抢主视觉；
5. 家长入口可达但弱化。
```

### 6.2 横屏主界面

通过标准：

```text
1. 小白狐在左侧保持强存在感；
2. 右侧不变成密集聊天窗口；
3. 语音按钮和拍图入口容易发现；
4. 横屏不挤压主要区域。
```

### 6.3 拍图状态

通过标准：

```text
1. 孩子知道图片已经给小白狐看；
2. 小白狐看图状态清楚；
3. 图片卡不压过小白狐；
4. 同一张图最多一个共创入口；
5. 不出现作品库或历史作品。
```

### 6.4 等待状态

通过标准：

```text
1. 孩子知道小白狐在听；
2. 没有倒计时；
3. 没有催促；
4. 不说也没关系表达清楚；
5. 等待结束自然收回。
```

### 6.5 真机验收

至少用以下设备或等价设备录屏：

```text
1. Redmi K60 竖屏；
2. Redmi K60 横屏；
3. Honor Pad 5 横屏；
4. 拍图后看图状态；
5. 小白狐等待孩子接话状态。
```

家长观感通过标准：

```text
1. 更像小白狐陪伴页；
2. 不像客服聊天窗口；
3. 不像游戏中心；
4. 不像学习平台；
5. 不像家长监控界面。
```

## 7. 本轮最容易踩的坑

```text
1. 照图复刻，导致需要大量新美术和大重构；
2. 消息区仍然太强，界面还是聊天页；
3. 图片卡太大，变成作品展示页；
4. 同一张图展示两个共创按钮，违反第 3 轮规则；
5. 等待状态加了主动问题，变成催促；
6. 横屏右侧变成客服聊天窗口；
7. 家长入口过强，破坏儿童主体验；
8. 开发端自行新增文案或入口。
```

避免方式：

```text
基于现有 Compose 结构做小步改造；
只保留可落地的视觉结构；
小白狐、状态、语音优先；
消息和按钮降级；
共创入口一次只出一个；
所有新增儿童文案使用本文范围；
真机录屏验收，而不是只看截图。
```
