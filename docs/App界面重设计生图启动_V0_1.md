## 1. 本轮目标

当前儿童端已经完成小白狐视觉中心化，但整体仍保留较强聊天页结构。第 4 轮目标不是做炫技视觉，也不是做游戏主页，而是把界面从“有小白狐的聊天工具”进一步推向：

```text
以小白狐为第一感知对象的儿童陪伴页。
```

界面要服务已经完成的三条体验主线：

```text
1. 连续陪伴：小白狐轻轻记得，但不监控；
2. 主动回应：小白狐会等、会停、会收住；
3. 常驻轻共创：孩子可以接一句故事、给作品起名、讲一句小故事。
```

孩子第一眼应感到：

```text
小白狐在这里；
我可以直接说话；
我可以给它看看；
它现在是在听、想、说、看图、等待还是收尾；
这不是普通聊天窗口，也不是学习任务页。
```

## 2. 当前界面事实

当前 Android 儿童页基于 Jetpack Compose 实现。

当前结构大致为：

```text
竖屏：
顶部是家长入口提示；
中上方是小白狐主视觉区；
下方是消息区和输入区。

横屏：
左侧是小白狐主视觉区；
右侧是聊天消息和输入区。
```

现有实现已经包含：

```text
1. 小白狐 AgentPanel / CartoonAgentView；
2. 消息区 ChatConversationPanel / ChatMessageListWithPreviews；
3. 输入区 InputBar；
4. 拍图入口；
5. 小白狐状态 presentation；
6. 等待、说话、打断、失败反馈相关状态；
7. 图片本地预览卡。
```

后续开发落地时，应优先围绕现有 Compose 结构改造，不应要求完全重写架构。

## 3. 生图目标

本次 image-2 生图不是生成最终可直接开发的设计稿，而是生成 3-4 个可评审的界面方向图。

图像应体现：

```text
1. 小白狐是画面中心，不是角落装饰；
2. 语音入口是第一操作，不是文字输入优先；
3. 拍图入口像“给小白狐看看”，不是上传文件；
4. 消息区存在但不抢主视觉；
5. 状态反馈清楚：正在听、正在想、正在说、正在看图、等你说、先停一下；
6. 整体温和、低刺激、儿童友好；
7. 不像游戏大厅，不像学习平台，不像客服聊天窗口。
```
