## 1. 本轮完成点

### 1.1 页面结构更偏陪伴页

本轮将儿童页继续向“小白狐第一视觉”收敛：

```text
1. 竖屏小白狐区域权重从 0.55 提升到 0.58；
2. 横屏小白狐区域权重提升到 0.55，消息/操作区域为 0.45；
3. 页面背景改为柔和渐变，而不是单一聊天页背景；
4. 小白狐主视觉区加入柔和光晕和状态气泡感；
5. 空消息状态继续强调“小白狐在这里”。
```

这符合第 4 轮目标：第一眼更像陪伴页，而不是普通聊天窗口。

### 1.2 消息区降级

本轮将可见消息数量限制为：

```text
竖屏最多 2 条；
横屏最多 3 条。
```

这解决了图像评审中的关键问题：消息区不能重新抢走小白狐主视觉。

### 1.3 拍图入口更自然

本轮将相关表达进一步收敛为：

```text
给小白狐看看；
小白狐正在看；
已经给小白狐看啦；
这张图片没有传好。
```

这符合“拍图不是上传文件，而是给小白狐看看”的方向。

### 1.4 同一张图只展示一个低压共创入口

本轮在 Android 层对图片相关快捷动作做了收敛：

```text
1. 没有 pendingImageContext 时，隐藏图片上下文动作；
2. 有 pendingImageContext 时，只展示一个主要图片共创入口；
3. 起名和讲一句小故事不会同时作为两个主动入口露出；
4. 图片动作标签归一为“起个名字”或“讲一句小故事”。
```

这与第 3 轮返修后的后端规则保持一致。

### 1.5 等待态更轻

本轮将等待和倾听相关文案进一步收敛为：

```text
我在听；
想说的时候再说；
不说也没关系；
小白狐在准备说。
```

未新增倒计时、催促、任务或主动共创问题。

### 1.6 家长入口弱化

本轮将家长入口文案从较长说明收敛为：

```text
大人长按进入。
```

按钮视觉也降低了存在感。家长入口仍可达，但不抢儿童主体验。

## 3. 需要真机继续观察的点

虽然本轮可以收口，但以下内容必须进入真机回看：

```text
1. 竖屏 0.58 / 0.42 权重下，小白狐是否足够突出，输入区是否被压得过低；
2. 横屏 0.55 / 0.45 权重下，右侧消息区是否仍有聊天页感；
3. 只显示最近 2-3 条消息后，孩子和家长是否仍能理解上下文；
4. 图片预览卡 112dp 高度是否足够看清，又不抢小白狐主视觉；
5. “大人长按进入”在真实设备上是否仍可理解；
6. “我在听 / 不说也没关系”在等待态是否自然，不像长期监听；
7. “给小白狐看看”入口是否足够明显。
```

这些是真机体感问题，不建议在没有录屏反馈前继续凭想象调整。

## 4. 本轮验收标准

### 4.1 竖屏

通过标准：

```text
1. 第一眼看到小白狐，而不是聊天记录；
2. 消息区只承担轻量上下文；
3. 语音按钮仍是第一操作；
4. “给小白狐看看”入口可见但不抢主操作；
5. 家长入口弱化但可达。
```

### 4.2 横屏

通过标准：

```text
1. 左侧小白狐有强存在感；
2. 右侧不是密集聊天窗口；
3. 语音、拍图、停一下、换话题不拥挤；
4. 平板/横屏手机不出现主要区域互相挤压。
```

### 4.3 拍图状态

通过标准：

```text
1. 孩子知道图片已经给小白狐看；
2. 小白狐看图状态清楚；
3. 图片卡不压过小白狐主视觉；
4. 同一张图最多一个低压共创入口；
5. 不出现作品库或历史作品。
```

### 4.4 等待状态

通过标准：

```text
1. 孩子知道小白狐在听；
2. 没有倒计时；
3. 没有催促；
4. 不说也没关系表达清楚；
5. 等待结束自然收回。
```
