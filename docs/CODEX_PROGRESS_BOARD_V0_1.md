## 4. 当前已定口径

```text
1. 当前阶段统一称为“小白狐关系与轻连续体验版”。
2. 本版核心机制是“小屋小客人”。
3. 小屋小客人不是宠物、作品库、故事库、收藏册或任务奖励。
4. 一个孩子首版最多一个活跃小屋小客人。
5. 召回只在 opening 阶段低频出现，每个会话最多一次。
6. 孩子跳过、短答、换题或沉默后，本会话不再拉回。
7. 睡前不主动召回兴奋故事线。
8. 拍图成功后只允许一个低压共创入口；拍图失败不得出现共创入口。
9. 家长端只展示高层摘要，不展示逐字聊天、图片细节、小客人历史。
10. 所有儿童端文案、小白狐话术、家长端敏感文案必须来自主控 master-copy。
11. 当前仍坚持“说完后的低压等待态”：小白狐说完后可以短暂等待，但手机不自动录音。
12. 等待态按钮应是“按一下开始说”，真实录音态才显示“说完了”。
```

## 5. 当前不要推进的大方向

```text
1. 不继续开笼统的新一轮体验大改。
2. 不继续做全局提示语大改。
3. 不做长期成长档案。
4. 不做作品库、故事库、兴趣画像。
5. 不做积分、徽章、打卡、排行、抽卡、宠物饥饿等机制。
6. 不做复杂学习脚手架。
7. 不做多孩子、多家长、正式账号体系。
8. 不做重型合规和家长治理重设计。
9. 不让开发端自行设计儿童端文案、小白狐提示语、共创玩法或界面方向。
10. 不继续凭想象改界面，界面后续只基于真机录屏和孩子/家长反馈微调。
11. 不恢复自动录音式自然接话。
12. 不做小客人列表、小客人图鉴、小客人详情页、小客人心情值。
13. 不做推送“小白狐想你了”。
```

## 7. 后置真机复测检查重点

```text
1. 打开 App 1 秒内是否有小白狐状态，不空白等 opening。
2. opening 晚到是否不抢孩子输入。
3. 小白狐说完后按钮是否是“按一下开始说”，不是“说完了”。
4. 低压等待期间是否没有自动录音、没有自动上传。
5. 点击按钮后是否才进入“正在听 / 说完了”。
6. 文字是否先于语音出现。
7. 首段语音是否不被后续段拖住。
8. 停一下 / 开始说话是否能丢弃迟到旧语音。
9. 图片失败时是否明确“小白狐还没看到”。
10. 图片失败后是否没有起名 / 讲一句小故事入口。
11. 首次打开是否出现低压小屋体验。
12. 有小客人时是否只轻召回一次。
13. 孩子跳过后是否本会话不再拉回。
14. 家长端是否仍避免逐字聊天、图片细节和监控感表达。
15. 日志是否仍避免孩子原文、图片内容、音频内容和完整接口响应。
```
