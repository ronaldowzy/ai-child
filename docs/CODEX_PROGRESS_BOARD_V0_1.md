## 3. 当前已定口径

```text
1. 当前阶段统一称为“说完后的低压等待态”，不再称为自然接话、免按键对话、自动续听。
2. 小白狐说完后可以短暂等待，但手机不自动录音。
3. 等待态按钮应是“按一下开始说”，不是“说完了”。
4. 只有孩子明确点击/按住后才进入真实录音，真实录音时才显示“说完了”。
5. 当前不做免按键自然对话、常开麦克风、环境声触发录音或长期监听。
6. 当前不继续堆重提示词；prompt fast path 只允许后续基于回归样例评估，不允许开发端自行重写人格。
```

## 4. 当前不要推进的大方向

```text
1. 不继续开笼统的新一轮体验大改。
2. 不继续做全局提示语大改。
3. 不做长期成长档案。
4. 不做作品库、故事库、兴趣画像。
5. 不做积分、徽章、打卡、排行、抽卡、宠物饥饿等机制。
6. 不做复杂学习脚手架。
7. 不做多孩子、多家长、正式账号体系。
8. 不做重型合规和家长治理重设计。
9. 不让开发端自行设计儿童端文案、小白狐提示语、共创玩法或界面方向。
10. 不继续凭想象改界面，界面后续只基于真机录屏和孩子/家长反馈微调。
11. 不恢复自动录音式自然接话。
```

## 5. 后续只接受的任务入口

当前只接受以下任务：

```text
1. 真机复测：验证 P0 SLA 窄修是否真实改善，不作为继续大改入口。
2. 明显 bug 修复：只修可复现问题。
3. 低成本 UI 小修：只修孩子第一眼看不懂、空间挤压、状态不清楚的问题。
4. 等待/语音体感小修：只修首段语音慢、打断不稳、等待时长不自然的问题。
5. 拍图状态小修：只修图片确认、看图状态、失败态、单个共创入口露出问题。
6. 文案小修：只修真实录屏里造成压力、技术感或误解的儿童端短句。
7. P1 家长日报状态机与展示去重复：只能作为窄修，不扩展复杂家长系统。
8. 小白狐状态素材脚本动效试制：只做素材预览，不直接接入 App。
```

每个任务必须写清楚：

```text
当前孩子或家长感受到什么问题；
改完后能直接感受到什么变化；
修改范围；
禁止范围；
如何确认不是原地打转；
远程主分支提交编号。
```

## 6. 后置真机复测检查重点

```text
1. 打开 App 1 秒内是否有小白狐状态，不空白等 opening；
2. opening 晚到是否不抢孩子输入；
3. 小白狐说完后按钮是否是“按一下开始说”，不是“说完了”；
4. 低压等待期间是否没有自动录音、没有自动上传；
5. 点击按钮后是否才进入“正在听 / 说完了”；
6. 文字是否先于语音出现；
7. 首段语音是否不被后续段拖住；
8. 声音慢时是否提示“声音有点慢，你可以先看字”；
9. 停一下 / 开始说话是否能丢弃迟到旧语音；
10. 图片失败时是否明确“小白狐还没看到”；
11. 图片失败后是否没有起名 / 讲一句小故事入口；
12. trace 是否能说明每次慢在哪里；
13. 日志是否仍避免孩子原文、图片内容、音频内容和完整接口响应；
14. 家长日报是否仍存在生成中/失败/素材不足/展示重复问题，作为后续 P1 判断依据。
```
