## 3. 当前不要推进的大方向

```text
1. 不继续开笼统的新一轮体验大改。
2. 不继续做全局提示语大改。
3. 不做长期成长档案。
4. 不做作品库、故事库、兴趣画像。
5. 不做积分、徽章、打卡、排行、抽卡、宠物饥饿等机制。
6. 不做复杂学习脚手架。
7. 不做多孩子、多家长、正式账号体系。
8. 不做重型合规和家长治理重设计。
9. 不让开发端自行设计儿童端文案、小白狐提示语、共创玩法或界面方向。
10. 不继续凭想象改界面，必须基于真机录屏和孩子/家长反馈。
```

## 4. 后续任务入口

当前只接受以下任务：

```text
1. 真机体验回看：围绕 Redmi K60、Honor Pad 5 或等价设备录屏。
2. 低成本 UI 小修：只修孩子第一眼看不懂、空间挤压、状态不清楚的问题。
3. 语音/等待体感小修：只修首段语音慢、打断不稳、等待不自然的问题。
4. 拍图状态小修：只修图片确认、看图状态、单个共创入口露出问题。
5. 文案小修：只修真实录屏里造成压力、技术感或误解的儿童端短句。
6. 明显 bug 修复。
```

每个任务必须写清楚：

```text
当前孩子感受到什么问题；
改完后孩子能直接感受到什么变化；
修改范围；
禁止范围；
如何确认不是原地打转；
远程主分支提交编号。
```

## 5. 真机体验检查重点

```text
1. 打开儿童页，小白狐是否第一眼可见，而不是聊天记录先被注意到；
2. 竖屏下小白狐、消息区、输入区是否比例合适；
3. 横屏下小白狐、消息区、语音按钮是否互相挤压；
4. 语音主按钮是否清楚，拍图入口是否像“给小白狐看看”；
5. 文字是否先出现，首段语音是否仍慢；
6. 停一下是否立刻停止旧语音；
7. 静音后是否保留文字但不自动朗读；
8. 小白狐说完后等待孩子继续说是否自然；
9. 拍照、相册、取消、失败、成功是否顺；
10. 普通图片是否进入创作分享，题目图片是否进入学习帮助；
11. 轻记忆是否偶尔出现，不频繁、不追踪、不拉回孩子不想聊的话题；
12. 常驻轻共创是否像陪伴，不像任务、游戏化或学习平台；
13. 同一张图是否最多一个共创入口；
14. 界面重设计是否让孩子更容易理解小白狐在听、想、说、看图、等待或收尾；
15. 家长入口是否弱化但可达；
16. 日志是否仍避免孩子原文、图片内容、音频内容和完整接口响应。
```
