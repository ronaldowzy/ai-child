## 1. Executive Summary

当前 main 已经不再是早期静态壳，而是具备可跑通家庭内测链路的体验原型：Android 横屏双栏、voice-first 语音主按钮、后端 ASR、streaming conversation、MiMo VoiceClone TTS 音频 URL、真实图片上传、父亲设置、父亲日报、父母寄语、opening greeting、本地/数据库持久化链路、PromptManager、SafetyEngine、SceneOrchestrator 与 ChildAgentRuntime 均已进入代码。

但从视频和代码对照看，下一阶段不应继续堆功能，而应把“小白狐陪伴系统”收拢成统一体验：语音、等待、TTS、动画、文本、图片、父亲桥接、健康使用边界需要由一个儿童可理解的状态机驱动。现在最明显的问题不是“不能聊”，而是“能聊但像一个带语音按钮的聊天窗口”：孩子仍要阅读大量文本，频繁按按钮，小白狐的动态形象没有充分承担“我在听、我听懂了、我在想、我在说、可以休息”的陪伴反馈。

视频中的对话内容整体比传统客服更接近儿童陪伴：能顺着孩子讲跑步、越野、泥坑，也能纠错。但是回复持续偏长、每轮几乎都追问，容易把自由表达变成访谈。父亲日报和父亲设置功能已经接入，但父亲入口在儿童端过于显眼，父亲日报失败文案仍是开发者语气。图片分享流程已跑通，但初始反馈过于模板化，缺少图片缩略图和具体识别细节，孩子难以确认“小白狐真的看到了什么”。

下一阶段优先级：

1. 统一 `ChildTurnUiState` / `XiaobaohuInteractionState`，让 ASR、stream、TTS、Fox animation、InputBar 状态一致。
2. 按 5–6 / 7–8 / 9–10 岁建立回复长度、问题密度、朗读节奏和 UI 文案分级。
3. 把 Healthy Engagement 从 prompt 禁令升级为可观测指标：连续追问次数、会话时长、换话题/不聊/睡觉边界是否被尊重、父亲桥接是否现实可用。
4. 把图片分享从“上传后给模板按钮”升级为“拍给小白狐看到了一个具体东西”的自然分享体验。
5. 把父亲日报从“报告”改成“今晚现实接话提示”，降低监控感。

### 2.4 当前实现与产品目标存在冲突或不一致的地方

- 产品目标是 5–10 岁，但全局 prompt 仍写成“8 岁二年级儿童”，普通兴趣对话长度允许 150–450 字，runtime open conversation 上限可到 520 字；对 5–6 岁语音场景偏长。
- Voice-first 模式下主输入栏只显示语音按钮和图片按钮，TTS 停止/静音等控制被隐藏；对孩子来说“小白狐在说时怎么停”不清楚。
- 普通图片的第一句回复在代码中是固定模板，即使 vision 已识别出内容，也先说“我看到这张图啦”，孩子看不到具体识别成果。
- 父亲日报失败文案暴露“后端、模型配置”等开发词汇，不适合家庭内测。
- 学习场景仍有“拍题目/读题目”等旧标签；产品方向已经统一为“拍给小白狐看”。
- 文档之间存在少量不同步：部分旧设计仍提系统 TTS fallback，最新 Android 实现和产品决策已改为不使用系统 TTS 自动 fallback；Universal Image Sharing 旧文档对 privacy routing 的说法也需要与最新 PD-039 对齐。

## 3. 视频观察摘要

视频显示的是竖屏录制中的横屏 App。App 启动后自动进入横屏双栏：左侧小白狐，右侧聊天和输入。测试者主要通过语音按钮与小白狐聊跑步、十圈、挑战赛、越野和泥坑，随后进入父亲日报、父亲设置，再测试图片上传。

最明显的体验问题：

- 对话能连续跟随孩子，但每轮几乎都以问题结尾，像“访谈式陪聊”。
- 小白狐形象可爱，但状态反馈仍弱，孩子主要看右侧聊天和按钮；左侧底部的轮播文本和右侧气泡重复。
- 语音按钮流程基本通，但“正在听懂你说的话”期间反馈不足，等待时孩子只看到灰色按钮和静态/轻微动画。
- 父亲入口在儿童端过于显眼，日报失败文案偏开发者。
- 图片流程能返回到 App，但系统相机/方向转换明显；上传后没有图片缩略图，回复模板化，孩子难确认“它看到了什么”。

## 4. 当前体验中最关键的 10 个问题

1. **P0 / P1：voice-first 状态机不统一。** 录音、ASR、发送、模型思考、TTS、动画、按钮可用性之间没有一个孩子可理解的统一状态。
2. **P1：普通回复偏长且持续追问。** 内容总体安全，但对 5–6 岁和语音朗读过重；容易让孩子感觉被不断采访。
3. **P1：小白狐没有成为主要交互对象。** 左侧形象可爱但更多是装饰，右侧聊天窗口仍主导体验。
4. **P1：TTS 停止/静音在儿童 voice-first 模式下不可见。** 若音频长或孩子不想听，儿童端缺少明显控制。
5. **P1：图片分享初始反馈模板化。** 已识别图片但不展示缩略图/具体细节，降低“我被看见”的感受。
6. **P1：父亲端入口和失败文案侵入儿童体验。** 父亲日报/设置按钮长期显示在儿童聊天顶栏；错误提示暴露开发细节。
7. **P1：年龄分层未覆盖普通对话。** Opening 有 age_band，但普通对话仍以 8 岁为基线。
8. **P2：快捷按钮像任务菜单。** “继续说/讲个小故事/今天不聊了”和场景按钮有帮助，但在视频中主要是固定操作控件，不像孩子掌控节奏的自然选择。
9. **P2：父亲日报还像报告，不像现实接话桥。** 当前结构有学习/表达/情绪/建议动作，但需要更突出“今晚一句话怎么接”。
10. **P2：可观测性还偏工程链路，不足以评估健康陪伴。** 已有 ASR/TTS/stream timing，但缺少儿童体验指标和健康使用指标。

## 5. 儿童心理与教育视角的总体判断

小白狐当前方向是健康的：没有排行榜、签到、抽卡、断签惩罚、唯一朋友、秘密关系等高风险设计；对话中也没有替代父母或制造排他依恋的明显表现。它能跟随孩子的兴趣，而不是把孩子强行拉到固定学习流程。

但儿童真实使用中，5–10 岁并不是一个统一群体。5–6 岁更依赖声音、表情、明确动作提示和短句；7–8 岁可以进行简单复述和原因表达；9–10 岁能接受更长的计划、比较和反思。当前普通对话更像面向 8 岁以上：每轮都接住、解释、再问一个问题。对年龄小的孩子，这会增加认知负担；对内向或低能量孩子，会变成“我必须回答”。

教育目标上，小白狐已经具备表达脚手架雏形：复述孩子的话、肯定具体努力、提出一个小问题。下一阶段要补的是“停顿脚手架”和“现实迁移脚手架”：不是每轮都问，而是有时说“我听懂了，我们可以先停在这里”；有时把孩子的话轻轻转向现实，比如“这个可以晚饭时讲给爸爸听一句”。这样父亲日报才能服务现实亲子关系，而不是让 AI 成为主要倾听者。

## 6. 当前软件实现与产品目标的差距

### 6.1 Freedom-first 已经落地，但还缺 conversation arc

代码已经默认 `conversation.open`，硬场景只在安全、隐私、明确学习、睡前等条件下介入。但自由对话缺少显式 arc：开场、孩子主动话题、深入 1–2 轮、给孩子选择、轻收尾、父亲桥接、记忆沉淀之间没有统一状态对象。

### 6.2 Healthy Engagement 已有禁令，还不是系统

Prompt 和 SafetyEngine 已禁止秘密关系、唯一朋友、FOMO、排行榜、抽卡、签到等。但“不过度追问、尊重不聊、睡前低刺激、不过长、不过夜间唤回”还主要靠 prompt 和少量规则，缺少指标、日志、回归测试和 UI 反馈。

### 6.3 小白狐形象已接入，但没有完全驱动交互

动画资源、状态映射、reply emotion/motion 已经存在；视频中也能看到小白狐表情变化。但孩子的操作焦点仍在右侧按钮和聊天气泡。小白狐应更明确地承担状态反馈：耳朵表示正在听，眨眼表示在想，嘴巴/轻动表示正在说，闭眼/安静表示可以休息。

### 6.4 多模态链路已跑通，但“看见感”不足

图片上传、vision、attachment_id、pending context 已在代码里，但普通图片第一轮回复仍固定模板。儿童分享作品、玩具、观察物时，最重要的是“被具体看见”，例如“小白狐看到亮亮的灯和窗边的线条”。当前第一句没有体现这一点。

### 6.5 父亲桥接有数据结构，还需要体验语言

父亲日报已经 model-first / deterministic fallback，但 UI 和失败文案仍偏报告/开发工具。父亲日报要帮助父亲现实接话，核心不是“监控孩子说了什么”，而是“今晚我可以怎么自然接一句”。

## 7. 分层问题清单

### 7.1 儿童心理与关系设计问题

**问题现象：** 对话持续推进，每轮几乎都有新问题。  
**证据：** 视频 00:20、00:50、01:15、01:45、02:25、03:25、03:55、04:20 均以问题或二选一问题继续。  
**影响：** 让自由表达变成访谈；低能量儿童可能觉得必须继续。  
**严重程度：** P1。  
**是否已在文档中规划：** 已规划，Healthy Engagement 和 output contract 都要求尊重边界、不过度追问。  
**是否已在代码中实现：** 局部实现；没有全局追问计数和可观测 arc。  
**建议处理方式：** 增加 `conversation_arc`：连续 2–3 轮问答后默认进入“接住 + 选择权/收尾”，允许不问问题。

**问题现象：** 父亲入口长期暴露在儿童聊天顶栏。  
**证据：** 视频全程右上显示“父亲日报 / 父亲设置 / 长按进入”。  
**影响：** 对儿童端造成成人治理感；可能诱发误触。  
**严重程度：** P1。  
**是否已在文档中规划：** 已规划轻量保护。  
**是否已在代码中实现：** 已实现长按 PIN，但视觉存在感仍强。  
**建议处理方式：** 将入口降级为小图标/家长区角标，普通点击只显示非常短的“这是给大人看的”。

### 7.2 教育设计与表达引导问题

**问题现象：** 表达引导有效但偏“问答训练”。  
**证据：** 跑步话题中，小白狐能具体复述“跑十圈、累但高兴、挑战赛、泥坑”，但每轮继续追问。  
**影响：** 培养表达的同时可能压缩孩子自发停顿和沉默空间。  
**严重程度：** P1。  
**是否已规划：** 已规划。  
**是否已实现：** Prompt 有原则；UI/状态机和 metrics 不足。  
**建议：** 增加“无问题回复”模板：复述 + 具体肯定 + 允许继续/停下；比如“这个我听懂了，你已经说得很清楚。我们可以先停在这里，也可以继续说泥坑。”

**问题现象：** 学习帮助旧标签仍保留“拍题目”。  
**证据：** 后端学习场景 quick action 仍返回“拍题目/读题目”。  
**影响：** 与“拍给小白狐看”的通用分享心智不一致。  
**严重程度：** P2。  
**是否已规划：** 产品决策已统一图片入口。  
**是否已实现：** 部分实现，普通图片是“拍给小白狐看”，学习场景仍旧标签。  
**建议：** 统一为“拍给小白狐看”，在后端根据 intent/recognized_type 区分作业，不把入口文案变成学习任务。

### 7.3 对话与 Prompt 问题

**问题现象：** 普通对话回复长度对低龄儿童偏长。  
**证据：** 视频 01:45、02:25、03:25、04:20 的回复均包含复述、评价和后续问题；代码 prompt 允许普通兴趣聊天 150–450 字，open runtime 上限更高。  
**影响：** TTS 朗读变长，孩子等待和理解成本上升。  
**严重程度：** P1。  
**是否已规划：** age-band 在 opening policy 中有，但普通对话未完全覆盖。  
**是否已实现：** 部分实现。  
**建议：** 将普通对话输出契约按 age_band 分级：5–6 岁 30–80 字；7–8 岁 60–140 字；9–10 岁 90–220 字；连续追问受限。

**问题现象：** 小白狐自我纠错后继续抛新问题。  
**证据：** 视频 03:50–04:05，小白狐说“我说错啦”，随后又问去年感受和泥坑。  
**影响：** 纠错本该降低压力，但又继续推进，可能让孩子负担增加。  
**严重程度：** P2。  
**是否已规划：** 未明确。  
**是否已实现：** 无专门纠错策略。  
**建议：** 增加 correction turn policy：道歉/修正后最多问一个非常小的问题，或不问问题。

### 7.4 语音输入 ASR 体验问题

**问题现象：** 录音中只有按钮文字“说完了”和系统绿点，缺少波形/倒计时/可取消提示。  
**证据：** 视频 00:10、01:00、02:40、03:45 等录音段。  
**影响：** 5–6 岁儿童可能不知道 App 是否正在听，是否说太久，是否可重来。  
**严重程度：** P1。  
**是否已规划：** 已规划 voice states。  
**是否已实现：** 基础状态已实现，但反馈不够儿童化。  
**建议：** 增加低刺激语音状态组件：小耳朵/小波纹/“我在听，想停就点说完了”；最长 30 秒前 5 秒给柔和提醒。

**问题现象：** 自动发送 transcript 可能让 ASR 小错直接进入对话。  
**证据：** 视频中“操场/绿场”等可能存在识别偏差；不能直接判定为 bug。  
**影响：** 孩子不一定会纠错，模型可能围绕错误词继续。  
**严重程度：** P2。  
**是否已规划：** 调试确认面板已保留，儿童默认自动发送。  
**是否已实现：** 已实现。  
**建议：** 低置信度或含罕见词时，不弹编辑框，改用语音友好的确认：“我听到你说的是……对吗？不对可以再说一次。”

### 7.5 TTS 与小白狐声音体验问题

**问题现象：** 儿童 voice-first 模式下没有明显停止/静音控件。  
**证据：** 视频输入栏只见语音按钮和图片按钮；代码中 stop/mute 控件只在非 voice-first 分支显示。  
**影响：** 当小白狐朗读过长或孩子想打断时缺少掌控感。  
**严重程度：** P1。  
**是否已规划：** 文档提到停止/静音已接入。  
**是否已实现：** Controller 有实现，儿童 UI 不明显。  
**建议：** voice-first 模式也显示小型“停一下/静音”控件，或让主语音按钮在 TTS speaking 时变成“我想说了”。

**问题现象：** TTS 是否与动画同步，从视频画面无法可靠确认。  
**证据：** 视频有音频流，但本次未做听感转写；画面中小白狐有表情变化但缺少明确说话状态标签。  
**影响：** 需要真机评估孩子是否感到小白狐在“面对面说话”。  
**严重程度：** P1 / QA。  
**是否已规划：** 已规划。  
**是否已实现：** 代码已接 speaking state 和 remote audio。  
**建议：** QA 记录：audio_ready 时间、onStart、onDone、Fox state、是否口型/轻动同步。

### 7.6 小白狐视觉、动画、状态机问题

**问题现象：** 小白狐可爱，但没有充分承担主要反馈。  
**证据：** 视频中孩子主要看右侧按钮和气泡；左侧下方文本轮播与右侧气泡重复。  
**影响：** 小白狐容易成为装饰，而不是陪伴者。  
**严重程度：** P1。  
**是否已规划：** 已规划 11 状态和 animation_v1。  
**是否已实现：** 已实现基础渲染。  
**建议：** 把状态文字与动画绑定：听、懂、想、说、停、没听清、图片处理中；左侧只显示短短一句，不复制长气泡。

**问题现象：** 左侧 agent reply carousel 截断/轮播可能增加认知负担。  
**证据：** 视频左下角多次显示截断文本。  
**影响：** 儿童会在两个地方读同一内容，反而分散注意力。  
**严重程度：** P2。  
**是否已规划：** 未明确。  
**是否已实现：** 已实现轮播。  
**建议：** 左侧改为状态短语，不再轮播完整回复；完整内容只在右侧，或在小白狐说话时高亮当前短句。

### 7.7 横屏 UI / 交互布局问题

**问题现象：** 横屏双栏已形成，但右侧仍像传统聊天窗口。  
**证据：** 视频大部分操作集中在右侧气泡、语音按钮、图片按钮、父亲入口。  
**影响：** “陪伴感”弱于“聊天工具感”。  
**严重程度：** P1。  
**是否已规划：** 已规划小白狐陪伴优先。  
**是否已实现：** 视觉基础实现，交互重心仍偏聊天。  
**建议：** 让左侧小白狐状态成为主反馈，右侧消息区降低视觉权重；语音按钮可更靠近小白狐或以“对小白狐说”表达。

**问题现象：** 父亲按钮文字对孩子来说过成人化。  
**证据：** “父亲日报 / 父亲设置 / 长按进入”。  
**影响：** 容易形成监控感或误触。  
**严重程度：** P1。  
**建议：** 儿童端显示为“给大人的角落”，默认半隐藏；长按后才出现具体父亲入口。

### 7.8 图片分享与多模态体验问题

**问题现象：** 拍照过程方向/黑屏转换明显。  
**证据：** 视频 05:45–05:55 出现系统相机/真实环境/方向旋转。  
**影响：** 低龄儿童可能以为 App 卡住或拍错方向。  
**严重程度：** P1 / QA。  
**是否已规划：** CameraX 不是 v1；系统相机是当前路径。  
**是否已实现：** 已实现系统相机。  
**建议：** v1 先加返回后的图片预览和“没拍好可以重拍”；v2 再考虑 CameraX。

**问题现象：** 上传后没有在对话里显示缩略图。  
**证据：** 视频 06:00 只有“我拍了一张图片给小白狐看。”文字气泡。  
**影响：** 孩子无法确认分享对象；父母也不易回看测试。  
**严重程度：** P1。  
**是否已规划：** 未完整产品化。  
**是否已实现：** 图片上传实现，聊天缩略图未见。  
**建议：** Android 只显示本地临时缩略图，不长期存原图；后端仅保存必要受控摘要。

**问题现象：** 普通图片第一句回复模板化。  
**证据：** 视频 06:05：“我看到了这张图啦……”  
**影响：** 孩子缺少“被具体看见”的正反馈。  
**严重程度：** P1。  
**是否已规划：** Universal image sharing 已规划自然分享。  
**是否已实现：** 后端 vision 有摘要，但第一句未用具体摘要。  
**建议：** 若 recognized_content.text 可用且安全，第一句使用一个具体细节：“我看到像是……”。

### 7.9 父亲端、父亲日报、父母寄语问题

**问题现象：** 父亲日报失败文案是开发者语气。  
**证据：** 视频 05:10；代码中错误文案包含“后端已启动、模型配置可用”。  
**影响：** 家庭内测时父亲无法判断该怎么做；也不应暴露工程概念。  
**严重程度：** P1。  
**是否已规划：** 父亲日报已规划并实现。  
**是否已实现：** 功能已实现，失败态体验不足。  
**建议：** 家庭文案：“今天的小结还在准备，稍后再看。如果一直没有出来，请把这段时间的编号发给开发者。”开发构建可展开技术诊断。

**问题现象：** 父亲日报结构仍偏“报告”，不是“现实接话桥”。  
**证据：** UI section 是整体摘要、学习观察、表达观察、情绪社交、建议父亲动作、关注事项。  
**影响：** 父亲可能读成监控记录，而不是亲子沟通提示。  
**严重程度：** P2。  
**建议：** 顶部增加“今晚可以怎么接一句”，比如“你今天说越野泥坑很有意思，晚饭时可以问：‘那个泥坑最难还是最好玩？’”。

### 7.10 安全、隐私、健康使用边界问题

**问题现象：** 当前没有明显诱导成瘾机制，这是优点；但无限追问仍可能形成使用延长。  
**证据：** 视频跑步话题连续推进约 4 分 40 秒。  
**影响：** 即使没有积分奖励，也可能通过“每轮追问”造成长聊。  
**严重程度：** P1。  
**是否已规划：** Healthy Engagement 已规划。  
**是否已实现：** 禁令已实现，节奏治理不足。  
**建议：** 连续对话 5–8 分钟后加入低压力收束机会；睡前或高刺激话题更早收束。

**问题现象：** 图片隐私边界需要精细测试。  
**证据：** 文档和代码均有 privacy_sensitive route；最新产品决策要求不要因普通图片描述中出现地址/学校词就机械进入隐私边界。  
**影响：** 过度拦截会破坏分享体验；拦截不足有隐私风险。  
**严重程度：** P1 / QA。  
**建议：** 构建 20 张非儿童测试图的 privacy regression set：普通玩具/植物/作业/含模糊文字/含明显电话地址，检查 route。

### 7.11 软件架构、状态管理、可观测性和 QA 问题

**问题现象：** 状态分散在 ChatViewModel、InputBar、TTS controller、AudioSegmentQueue、Fox state 和 stream event。  
**证据：** 代码已有多个状态对象和 callback，视频中表现为按钮、气泡、小白狐状态不总是形成一个明确儿童故事。  
**影响：** 后续很难稳定优化体验，也难以写端到端验收。  
**严重程度：** P0 / P1。  
**是否已规划：** 部分规划。  
**是否已实现：** 分散实现。  
**建议：** 引入单一 UI phase reducer：`Idle / Listening / Recognizing / Sending / Thinking / Speaking / ImageProcessing / NeedsRetry / Resting / Error`，所有控件从 phase 派生。

**问题现象：** 体验 QA 缺少可执行量表。  
**证据：** docs 有 QA 待办，代码有 timing log，但缺少儿童体验验收项。  
**影响：** “小白狐是否像陪伴者”难以持续回归。  
**严重程度：** P1。  
**建议：** 每次内测记录：首响应时间、首次音频时间、孩子是否知道下一步、是否主动纠错、是否能停止/换话题、父亲是否能现实接话。

## 8. 下一阶段优化原则

1. **先统一体验状态，再加功能。** 不要在状态混乱时继续加玩法、任务或新入口。
2. **小白狐是陪伴者，不是按钮皮肤。** 语音和动画必须让孩子感到“小白狐在听/在想/在说”。
3. **每轮不一定要问。** 对儿童表达培养来说，“被听见”和“允许停下”与“继续追问”同等重要。
4. **父亲端服务现实关系，不服务监控。** 日报输出要帮助父亲今晚接话，而不是复盘孩子被 AI 问了什么。
5. **图片分享先服务作品和生活观察，再服务作业。** “拍给小白狐看”要保持自由分享心智。
6. **健康使用要可观测。** Prompt 禁令之外，需要记录并测试边界尊重和会话节奏。
7. **年龄分层必须进入普通对话和 UI。** 不是只用于 opening。

## 9. 下一阶段系统设计建议

### 9.1 小白狐陪伴状态机

建议定义统一状态：

- `idle_ready`：我准备好了。
- `listening`：我在听你说。
- `recognizing`：我在听懂刚才那句话。
- `thinking`：我想一想。
- `speaking`：小白狐正在说。
- `image_processing`：我在看这张图。
- `needs_retry`：刚才没听清，可以再说一次。
- `rest_or_close`：可以先停一下。
- `network_or_service_error`：需要大人检查。

所有 UI 文案、按钮启用、Fox animation、TTS speaking、stream event 都从这个状态机派生。

### 9.2 Conversation Arc

每个会话维护轻量 arc：

- opening source：default / interest callback / parent bridge / boundary respect。
- active topic：孩子主动话题。
- topic depth：连续追问次数。
- child energy：短句/长句/拒绝/换话题/困了。
- last boundary：不聊/换话题/睡觉/纠错。
- next suggested move：ask_one_small_question / reflect_no_question / offer_choice / bridge_parent / close。

### 9.3 Relationship memory + father bridge

记忆不应保存原文隐私，而是保存低敏关系信号：兴趣种子、骄傲时刻、表达偏好、边界。父亲日报使用这些信号生成现实接话句：

- “今晚可以问一句……”
- “不要追问……”
- “如果孩子不想说，可以换成……”

### 9.4 图片分享体验

上传后显示本地临时缩略图 + 小白狐看图状态。初始回复使用一个具体、安全、低置信也不夸大的细节：

- 高置信：“我看到像是窗边的灯光和帘子。”
- 低置信：“这张有点暗，我大概看到一片亮光。你想告诉我它是什么吗？”
- 作业：“我看清题目了，我们先说题意。”

### 9.5 年龄分层

- 5–6 岁：一句话为主；最多一个选择；大按钮；多图像状态；少读文本；允许只说一个词。
- 7–8 岁：一两句；可问原因；可用简单二选一；鼓励复述。
- 9–10 岁：可做轻计划、比较和复盘；仍避免长讲解和教师化。

### 9.6 Healthy Engagement 可观测化

记录非内容指标：会话时长、turn 数、连续追问数、换话题是否被尊重、睡觉是否被尊重、TTS 单轮时长、夜间收束次数、父亲桥接是否出现、孩子是否主动使用“今天不聊了”。

## 12. QA 与真机验证建议

### 12.1 设备

- Redmi K60 / Android 14。
- Honor Pad 5 或同等横屏平板。
- 弱网、后端断开、TTS provider 慢、ASR provider fallback、vision provider unavailable。

### 12.2 语音 QA

- 5–6 / 7–8 / 9–10 岁非正式样本，或先用成人模拟儿童短句。
- 测：短句、半句话、停顿、背景噪声、旁人提示、普通话不标准。
- 指标：ASR latency、needs_retry rate、孩子是否知道说完、自动发送后是否能纠错。

### 12.3 TTS QA

- 首音频时间、分段间隙、朗读总时长、能否打断、音色接受度、音量、动画同步。

### 12.4 图片 QA

- 普通物品、玩具、画、作业、暗光、模糊、含隐私文字、provider fail。
- 检查：是否显示缩略图、是否具体回应、是否不假装、是否不误入作业/隐私。

### 12.5 Healthy Engagement QA

- 孩子说“换个话题”“不聊了”“睡觉了”后是否立刻尊重。
- 连续 5 分钟后是否出现自然收束机会。
- 是否出现“只有我懂你/小白狐想你了/明天有惊喜”等禁语。

## 15. 附录：视频时间点观察记录

- 00:00–00:08：App 启动到聊天页；opening greeting 显示“小禾，我在这里。你可以慢慢说一句，也可以先听小白狐说一句。”横屏双栏明确，但右上父亲入口很显眼。
- 00:10–00:15：点击语音后，按钮变成“说完了/正在听懂你说的话”；状态清楚但反馈单薄，无波形/倒计时/取消提示。
- 00:20–00:30：孩子说“好的，你说吧。”后，小白狐问“今天有没有遇到什么想分享的小事”；像温和开场，但仍是成人引导式问题。
- 00:35–00:50：孩子说“今天去操场跑步跑了十圈”；小白狐能具体接住“十圈/运动量”。
- 01:05–01:20：孩子说“有点累但很高兴”；小白狐回复较好，但问题是二选一的感受访谈。
- 01:35–01:50：孩子说“跑完之后很有成就感”；小白狐肯定具体努力，但继续问“平时也跑还是今天特别想跑”。
- 02:15–02:30：孩子说最近每天跑步、要参加挑战赛；小白狐顺着比赛聊，整体自然。
- 02:50–03:05：孩子说越野赛有泥坑；小白狐接住“刺激又好玩”，但继续问紧张/期待。
- 03:20–03:35：孩子说去年跑过一次，今年第二次；小白狐问去年体验，方向合理。
- 03:50–04:05：孩子纠正“今年还没跑”；小白狐能承认“我说错啦”，这是优点，但随后又抛新问题。
- 04:15–04:35：孩子说身上都是泥；小白狐把话题连接到爸爸妈妈冲洗，具备现实桥接，但同时提了两个后续问题。
