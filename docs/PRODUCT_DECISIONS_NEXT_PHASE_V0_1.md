## 2. 执行约束

```text
1. 所有下一阶段任务必须先阅读 docs/下一阶段产品进化计划_V0_1.md。
2. 所有涉及儿童端文案、提示语、共创玩法、轻记忆回访和界面结构的任务，必须由主控会话先出设计和提示语。
3. 开发端不得自行设计小白狐提示语、儿童端文案、回访策略、共创玩法或界面方向。
4. 任何实现都必须说明孩子能直接感受到什么变化。
5. 如果任务不能提升孩子真实体验，不应进入下一阶段开发。
```

## 3. 下一步推荐启动

```text
第 1 轮：连续陪伴与低压回访。
```

启动顺序：

```text
1. 主控会话生成第 1 轮产品设计提示语；
2. 必要时新开 ChatGPT 子会话细化低压回访体验；
3. 主控会话收口并生成开发端执行提示语；
4. 开发端按提示语实现；
5. 主控会话验收是否真正提升孩子亲近感。
```
