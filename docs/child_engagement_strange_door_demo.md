# ai-child 儿童吸引力试玩原型：奇怪小门 Demo

## 1. 背景

当前会话不是主控会话，本文件用于把本轮产品设计沉淀到 GitHub，供主控方读取、评估和拆解。

此前 ai-child 已完成「小白狐的小展台」及相关成长事件、家长报告轻量能力。但当前阶段不应继续扩展后台成长系统，而应回到儿童端核心吸引力：

> 孩子第一次打开 App 后，是否会被一件事马上吸引，并愿意继续玩、继续拍、明天再来。

本文件提出一个可试玩原型：**小白狐的现实变身闯关：奇怪小门 Demo**。

---

## 2. 核心判断

儿童端早期吸引力不能只依赖：

- 温柔陪伴
- 小白狐问候
- 拍照识别
- 小展台保存
- 成长记录
- 家长报告

这些能力更适合在孩子已经建立使用习惯后发挥价值。

初期更需要一个强钩子：

> 孩子一打开 App，就看到小白狐遇到一个可参与的小麻烦；孩子可以用现实物品或语言小游戏马上帮它过关；孩子的输入会立即改变小白狐的小世界。

核心不是「教育训练」，也不是「任务奖励」，而是：

> 现实世界被 AI 变成游戏道具，孩子用它帮小白狐闯过一个小关卡。

---

## 3. 产品定位建议

儿童侧不要只表达为：

> 儿童 AI 成长陪伴 App

更适合儿童心智的表达是：

> 小白狐每天都会遇到一个小麻烦。孩子可以用身边的东西，或者用脑袋里的答案，帮它过关。

孩子视角：

- 我不是来聊天的。
- 我是来帮小白狐闯关的。
- 我拍的东西会变厉害。
- 我说的话会让机关动起来。
- 我还想再试一个。

---

## 4. Demo 名称

建议内部名称：

```text
strange_door_demo
```

儿童侧可展示为：

```text
奇怪小门挡住了小白狐
```

或：

```text
小白狐今天卡住了
```

---

## 5. Demo 核心体验

孩子打开 App 后，首页直接出现一个正在发生的小事件：

```text
小白狐被一扇奇怪小门挡住了。

小白狐：
“你来得正好！
我被这扇奇怪小门挡住了。

它说：
要么给它看一个圆圆的东西，
要么答对一个怪问题。

你想用哪种办法帮我？”
```

页面提供两个选择：

```text
找圆圆的东西
答怪问题
```

两种方式都服务同一关卡，不要做成两个割裂功能。

---

## 6. 两种模式

### 6.1 模式 A：拍照变身闯关

孩子点击「找圆圆的东西」。

小白狐提示：

```text
找一个有点圆的东西就行。
瓶盖、杯子、球、纽扣……奇怪一点也可以。
拍给我看看！
```

孩子拍照或选图。

系统拿到图片识别结果后，生成三段反馈：

1. 现实识别
2. 变身道具
3. 小门反馈

示例：

```text
我看见了：蓝色瓶盖。

它现在不是普通的瓶盖了。
它变成了——小月亮盾牌！

小白狐把小月亮盾牌轻轻一转，
门上的圆锁“咔哒”一声打开了一点。
```

如果孩子拍到的不是圆形物品，也不要失败，要接住并给搞笑反馈：

```text
嗯……它不是很圆。
但它有一点弯弯的。

我把它变成了「半圆冲撞器」。
小白狐试了一下，门没有完全打开，
但是它打了个喷嚏，露出一条小缝。
```

关键原则：

- 不要像考试判错。
- 不要说「不符合要求，不能用」。
- 允许乱拍。
- 乱拍也要有反应。
- 孩子应该觉得“我还想再拍一个试试”。

---

### 6.2 模式 B：语言小游戏闯关

孩子点击「答怪问题」。

第一版固定题目即可：

```text
小门问：
“什么东西越洗越脏？”
```

标准答案：

```text
水
```

答对反馈：

```text
对，是水！

小门被你说得愣住了。
它低头想了三秒，
然后咔哒一下打开了。
```

答错反馈：

```text
这个答案有点勇敢。
小门差点相信了。

我给你一个提示：
它常常在杯子里、河里、盆里。
```

答错后允许：

- 再答一次
- 切换到「找东西帮忙」

语言小游戏不要包装成学习练习，而要包装成机关互动。

## 7. 关卡状态建议

本阶段只做一个 Demo 关卡。

建议状态：

```kotlin
enum class StrangeDoorDemoState {
    NotStarted,
    ChoosingMethod,
    PhotoSolving,
    RiddleSolving,
    PartialSuccess,
    Completed,
}
```

说明：

- 本地状态即可。
- App 重启后可以重置。
- 不要求长期保存关卡进度。
- 不要引入复杂任务、关卡地图、积分、等级。

## 8. 输出结构建议

### 8.1 拍照变身 JSON

图片识别后，可由本地模板或模型生成以下结构：

```json
{
  "objectName": "瓶盖",
  "transformedName": "小月亮盾牌",
  "transformedAbility": "可以让圆圆的门锁轻轻转起来",
  "doorEffect": "小白狐把它贴到门锁上，门锁咔哒一声转开了一点。",
  "foxLine": "这个可以用！它看起来很像一小片月亮。",
  "isGoodMatch": true
}
```

如果识别结果为空：

```text
objectName = "这个小东西"
```

### 8.2 道具命名风格

道具名要短、怪、儿童能懂。

推荐：

- 小月亮盾牌
- 半圆冲撞器
- 咕噜圆盘
- 蓝盖盖转轮
- 软软开门垫
- 星星小钥匙
- 眨眼门铃
- 圆滚滚按钮

避免：

- 稀有
- 史诗
- S 级
- 战斗力
- 装备等级
- 宝物
- 奖励
- 通关神器

## 9. 首页结构建议

首页不要先摆多个功能入口，而是优先展示今日事件。

结构：

```text
顶部：今日事件标题
中间：奇怪小门 + 小白狐当前困境
底部：两种解法按钮
角落：小展台入口 / 家长入口等弱入口
```

示例：

```text
标题：
奇怪小门挡住了小白狐

小白狐气泡：
“它说要么找一个圆圆的东西，
要么答对一个怪问题。”

主按钮：
找东西帮忙

副按钮：
动脑试试
```

孩子 3 秒内应该知道：

- 小白狐遇到事了。
- 我能帮它。
- 我可以拍东西，也可以答问题。

---

## 10. 与小展台的关系

小展台不是主玩法入口，而是结果承接。

正确关系：

```text
奇怪小门 Demo 吸引孩子参与
→ 孩子拍东西
→ 现实物品变成奇怪道具
→ 道具帮助小白狐过门
→ 孩子愿意保存这个道具 / 小发现
→ 放进小展台
```

不要让小展台继续膨胀成复杂成长档案系统。

## 11. 素材需求

### 11.1 奇怪小门

一扇绘本感、圆润、柔软的奇怪小门。

要求：

- 门上可以有圆形机关或圆锁。
- 有点神秘但不吓人。
- 不要宝箱感。
- 不要副本门。
- 不要解锁奖励门。

尺寸：

```text
1024x1024 透明 PNG
```

### 11.2 小门半开状态

同一扇奇怪小门，打开一条缝。

要求：

- 门缝里可以有柔和暖光。
- 不要强光。
- 不要奖励闪光。

尺寸：

```text
1024x1024 透明 PNG
```

### 11.3 圆形机关 / 圆锁

一个可单独放在门上的圆形机关。

尺寸：

```text
512x512 透明 PNG
```

### 11.4 道具变身光效

用于现实物品变成奇怪道具。

要求：

- 柔和云朵光晕
- 少量小星屑
- 不要爆炸
- 不要金币
- 不要强闪光
- 不要抽卡感

尺寸：

```text
512x512 透明 PNG
```

### 11.5 小门成功反馈光效

门打开一点时出现的轻柔反馈。

尺寸：

```text
512x512 透明 PNG
```

### 11.6 怪问题气泡底板

用于展示小门问题。

要求：

- 圆角
- 柔软
- 奶白 / 浅蓝 / 浅暖黄
- 中间留足文字区域
- 不自带文字

尺寸：

```text
1024x512 透明 PNG
```

### 11.7 变身道具卡底板

用于展示现实物品名、变身道具名、使用效果。

要求：

- 只做底板
- 不要文字
- 柔软小卡片感
- 不要卡牌抽奖感

尺寸：

```text
1024x768 透明 PNG
```

### 11.8 小门地面阴影

用于把门自然放到小屋背景中。

尺寸：

```text
1024x512 透明 PNG
```

## 12. 开发范围建议

### 12.1 本阶段要做

- 首页展示「奇怪小门」事件。
- 两种解法：拍照 / 答题。
- 拍照后生成现实物品变身反馈。
- 答题后生成小门反应。
- 支持再试一次。
- 支持把拍照产生的小发现放进现有小展台。

### 12.2 本阶段不要做

- 多关卡地图
- 积分
- 等级
- 排行榜
- 连续签到
- 复杂背包
- 复杂道具系统
- 云同步
- 新家长报告
- 新 GrowthEvent / GrowthInsight 类型
- 大规模素材系统

## 13. AI 话术原则

小白狐在这个 Demo 中不是老师，而是游戏主持人。

人格方向：

- 有点紧张
- 有点兴奋
- 有点好笑
- 会认真承认孩子的奇怪输入
- 不说教
- 不长篇大论

每次反馈最多 3–5 句。

### 拍照反馈模板

```text
我看见了：{现实物品}。

在小白狐的世界里，
它变成了：{怪道具名}。

小白狐用它{动作}，
于是{关卡变化}。
```

### 语言反馈模板

```text
你说的是：{孩子答案}。

{机关/小门}听完以后，
{反应}。

小白狐趁这个机会，
{关卡推进}。
```

## 14. 验收标准

这个 Demo 的验收不看后台复杂度，只看儿童吸引力。

关键标准：

1. 孩子打开 App 后，第一眼能看到“小白狐遇到事了”。
2. 孩子能立刻选择“拍东西”或“答怪问题”。
3. 拍照后能看到“现实物品变成奇怪道具”。
4. 道具会影响小门状态。
5. 答题后小门会有反应。
6. 答错不挫败，有提示和搞笑反馈。
7. 孩子能再试一次。
8. 小展台只是结果承接，不是主玩法。
9. 不继续扩展后台成长系统。
10. 目标是让孩子愿意再拍一个 / 再答一个。

更直接的验证问题：

```text
孩子拍完第一个东西后，是否愿意再拍第二个？
孩子答完一个怪问题后，是否愿意再试一次？
孩子第二天是否愿意看看小白狐又遇到什么事？
```

## 15. 后续扩展方向

如果「奇怪小门 Demo」验证有效，可以继续扩展：

### 15.1 现实合成

拍两个东西，合成一个奇怪道具。

示例：

```text
杯子 + 铅笔 = 咕噜雷达塔
橡皮 + 尺子 = 错字测量刀
书包 + 袜子 = 大肚保暖怪
```

### 15.2 孩子造关

孩子拍 2–3 个东西，小白狐用它们生成一个小关卡。

### 15.3 骗骗小白狐

允许孩子故意拍奇怪东西，看小白狐怎么解释。

例如：

- 墙
- 黑屏
- 手指
- 脚
- 作业本

### 15.4 小怪物生成器

拍一个东西，把它变成小怪。

示例：

```text
尺子 → 直直龙
橡皮 → 错字吞吞兽
书包 → 吞吞背包怪
```

### 15.5 作业怪模式

把作业压力包装成一个可拆解的小怪。

```text
这页作业太大了。
我们把它拆成三只小怪。
先解决最小的一只。
```

这些都不要在首版 Demo 中展开，先验证第一关是否有吸引力。
