# 多会话协作总则 v0.4

用途：定义本项目如何拆分主控会话、ChatGPT 子会话和开发端执行会话，避免多个会话互相覆盖、重复设计或越权改变儿童安全原则。

当前阶段已经完成孩子体验五轮收尾。下一阶段主方向为：

```text
小白狐连续陪伴与亲近感收敛
```

下一阶段不再按旧的家长日报、合规治理、复杂学习、长期成长档案优先级推进，也不再开笼统的“第六轮体验大改”。

---

## 0. 子会话启动前必须做

所有执行会话启动前必须阅读：

```text
AGENTS.md
README.md
docs/下一阶段产品进化计划_V0_1.md
docs/孩子体验五轮收尾计划_V0_1.md
docs/项目现状与后续路线图_V0_2.md
docs/小白狐提示语与文案规范_V0_1.md
docs/PRODUCT_DECISIONS_V0_1.md
docs/CODEX_PROGRESS_BOARD_V0_1.md
docs/session_process/SHARED_CONTEXT_V0_1.md
```

涉及 prompt、儿童端文案、小白狐人格表达、共创互动、界面设计或家长日报时，还必须遵守：

```text
docs/提示词与文案归属规则_V0_1.md
```

执行会话启动前先运行：

```bash
bash scripts/doctor_local_env.sh
```

规则：

```text
1. 不要只根据裸命令失败就判断本机缺少依赖。
2. 后端验证优先使用 scripts/test_backend.sh、scripts/lint_backend.sh。
3. 后台服务启停必须使用 scripts/start_backend_services.sh、scripts/status_backend_services.sh、scripts/stop_backend_services.sh。
4. 安卓验证优先使用 scripts/android_gradle.sh，而不是裸 ./gradlew。
5. 遇到本机环境、设备、局域网地址、基础地址等问题，先查 SHARED_CONTEXT_V0_1.md。
6. 新发现的共性坑必须在交接摘要里提出，由主控会话确认后写入共享上下文。
```

## 1. 当前主线

当前主线不是继续堆内部能力，而是把已完成的五轮能力收束成孩子能感受到的亲近感。

已完成五轮：

```text
第 1 轮：界面和小白狐形象丰富。
第 2 轮：语音朗读延迟收敛。
第 3 轮：自然说话机制。
第 4 轮：拍图分享体验。
第 5 轮：轻量记忆和连续陪伴。
```

下一阶段四个方向：

```text
第 1 轮：连续陪伴与低压回访。
第 2 轮：主动回应与等待感优化。
第 3 轮：常驻轻共创。
第 4 轮：App 界面重设计与落地。
```

当前暂不作为主线：

```text

2. 合规和家长治理重设计；
3. 多孩子、多家长、正式账号体系；
4. 长期成长档案；
5. 大范围提示语再优化；
6. 为测试而测试的完整回归；
7. 积分、徽章、打卡、排行、抽卡、宠物饥饿或连续使用奖励。
```

## 2. 会话角色

### 2.1 主控会话：产品设计者、规划者和体验治理者

主控会话承担下一阶段绝大部分产品设计工作，拥有最高产品决定权。

职责：

```text
1. 维护 docs/下一阶段产品进化计划_V0_1.md。
2. 拆分每一轮任务、定义边界、依赖顺序和验收标准。
3. 设计小白狐主动回应、轻记忆回访、共创互动和界面重设计方向。
4. 生成给 ChatGPT 子会话和开发端的执行提示语。
5. 审查执行会话产出是否真正提升孩子可感知体验。
6. 审查是否违反 AGENTS.md、PRODUCT_DECISIONS 和儿童安全原则。
7. 汇总测试结果、阻塞事项和下一轮任务。
8. 处理跨模块接口变化。
9. 维护文档事实源，删除或降级过期过程文档。
```

主控会话必须亲自设计或明确批准：

```text
1. 小白狐提示语；
2. 儿童端产品文案；
3. 主动回应策略；
4. 轻记忆出现方式；
5. 共创场景设计；
6. 界面核心结构方向；
7. ChatGPT image-2 生图提示语；
8. 开发端验收标准。
```

主控会话可以修改：

```text
README.md
AGENTS.md
docs/下一阶段产品进化计划_V0_1.md
docs/孩子体验五轮收尾计划_V0_1.md
docs/项目现状与后续路线图_V0_2.md
docs/PRODUCT_DECISIONS_V0_1.md
docs/CODEX_PROGRESS_BOARD_V0_1.md
docs/session_process/
scripts/
跨模块接口文档
```

### 2.2 ChatGPT 子会话：专项设计、专项生成或专项验证

ChatGPT 子会话不拥有最终产品决定权，只按主控会话生成的提示语完成单项任务。

职责：

```text
1. 按主控提示词完成单项设计、梳理、评审或生成任务。
2. 产出产品草案、界面草案、文案草案、状态说明、测试脚本等。
3. 不自行改变产品方向。
4. 不擅自扩展系统范围。
5. 不直接把结果交给开发端执行，必须先回到主控会话评审。
```

典型子会话：

```text
1. 连续陪伴设计子会话；
2. 主动回应与等待感设计子会话；
3. 常驻轻共创设计子会话；
4. App 界面 image-2 生图子会话；
5. 真机验收标准整理子会话。
```

界面生图特殊规则：

```text
1. 主控会话先生成 image-2 生图提示语。
2. 产品负责人新开 ChatGPT 会话调用 image-2 出图。
3. 产品负责人把出图结果发回主控会话。
4. 主控会话筛选、取舍、修订并转译成开发可执行结构说明。
5. 开发端不得直接根据未评审图片自由发挥。
```

### 2.3 开发端：实现执行者

开发端负责一个清晰方向的实现，不承担主要产品设计职责。

每个开发执行会话必须：

```text
1. 先阅读 AGENTS.md、README、下一阶段产品进化计划、进度看板和任务指定文档。
2. 先输出实现计划。
3. 列出会修改和不会修改的文件。
4. 等待确认或按提示明确授权后实现。
5. 添加或更新必要测试。
6. 运行相关测试。
7. 输出交接摘要。
8. 报告是否发现新的共性坑。
```

开发端不得：

```text
1. 自行设计小白狐提示语；
2. 自行设计儿童端产品文案；
3. 自行扩展新功能方向；
4. 把模糊理解替代为产品方案；
5. 借机做大系统重构；
6. 直接根据未评审的生图结果改界面；
7. 把假路径、mock 路径或兜底路径写成家庭内测真实能力通过。
```

## 3. 当前可并行会话划分

| 会话 | 主要目标 | 默认范围 | 依赖 | 备注 |
|---|---|---|---|---|
| 第 1 轮设计：连续陪伴与低压回访 | 让轻记忆真正变成陪伴体验 | 主控产品规则、Prompt 局部规则、记忆召回边界 | 第 0 轮完成 | 开发端不得自行设计回访话术 |
| 第 1 轮开发：连续陪伴实现 | 按主控规则实现低压回访和放下机制 | `backend/app/services/`、相关 tests | 第 1 轮设计确认后 | 不做成长档案 |
| 第 2 轮设计：主动回应与等待感 | 定义轮流说话、等待、打断、收回体验 | 主控产品规则、Android 状态说明 | 第 1 轮方向稳定后 | 必须服务真机体感 |
| 第 2 轮开发：等待感优化 | 落地语音队列、等待状态、打断停止 | `android/`、必要后端接口 | 第 2 轮设计确认后 | 不做常开麦克风 |
| 第 3 轮设计：常驻轻共创 | 定义故事接龙和拍图轻共创 | 主控文案、Prompt 局部规则、图片后续动作 | 第 1-2 轮稳定后 | 不做游戏系统 |
| 第 3 轮开发：轻共创实现 | 接入共创入口和最小互动链路 | 后端对话、图片后续动作、Android 呈现 | 第 3 轮设计确认后 | 不做积分、任务、作品库 |
| 第 4 轮设计：App 界面重设计 | 生成 image-2 提示语并评审出图 | 主控设计、生图子会话、结构说明 | 可在第 1 轮后启动 | 出图不直接进入开发 |
| 第 4 轮开发：界面落地 | 按主控结构说明实现 Android 页面调整 | `android/` 儿童主界面、小白狐状态、输入区 | 第 4 轮设计确认后 | 不做大而全 UI 重构 |

## 4. 文件所有权规则

同一时间只允许一个执行会话拥有某个模块的主要写入权。

```text
android/                      Android 视觉、语音等待、界面重设计开发会话拥有
backend/app/services/         对应后端体验任务会话拥有
backend/app/providers/        供应商相关任务会话拥有，下一阶段原则上少改
backend/app/repositories/     数据访问任务会话拥有，下一阶段原则上少改
docs/session_process/         主控会话拥有
README.md / AGENTS.md         主控会话拥有
docs/下一阶段产品进化计划_V0_1.md  主控会话拥有
```

如果执行会话必须改不属于自己的文件，应先在计划里说明原因。

## 5. 禁止事项

```text
1. 不得擅自改变儿童安全原则。
2. 不得引入真实模型调用作为自动化测试默认行为。
3. 不得把业务逻辑写进接口层。
4. 不得删除其他会话新增的测试来让自己通过。
5. 不得在未说明的情况下跨模块大改。
6. 不得写入真实儿童身份信息、真实照片、真实音频或真实密钥。
7. 不得让执行会话自行设计新的产品方向。
8. 不得把假路径或兜底路径写成家庭内测真实能力通过。
9. 不得加入积分、打卡、奖励、排行、抽卡、宠物饥饿或依赖话术。
10. 不得启动合规重设计、正式账号体系、多孩子多家长或成长档案。
11. 不得把常驻轻共创做成游戏中心、任务中心或学习平台。
12. 不得让小白狐变成唯一朋友、秘密朋友、监控者、老师、客服或任务系统。
```

## 6. 交接机制

每个执行会话完成后，必须给主控会话交接：

```text
1. 修改文件列表。
2. 孩子能感受到什么变化。
3. 测试命令和结果。
4. 安全影响。
5. 文档更新。
6. 未完成事项。
7. 下一个会话需要知道的接口或约束。
8. 共享上下文是否需要更新。
9. 是否严格遵守主控提示语，没有自行设计产品方向。
```

主控会话收到交接后：

```text
1. 检查是否符合 AGENTS.md。
2. 检查是否越过任务边界。
3. 检查是否真正提升孩子可感知体验。
4. 检查是否与 docs/下一阶段产品进化计划_V0_1.md 一致。
5. 更新进度板或当前路线图。
6. 决定是否进入下一轮。
```

## 7. 本机环境规则

环境诊断：

```bash
bash scripts/doctor_local_env.sh
```

后端验证：

```bash
bash scripts/test_backend.sh
bash scripts/lint_backend.sh
```

后台服务：

```bash
bash scripts/start_backend_services.sh --agent main --port 8000
bash scripts/status_backend_services.sh
bash scripts/stop_backend_services.sh --agent main
```

安卓验证：

```bash
bash scripts/android_gradle.sh test
bash scripts/android_gradle.sh assembleDebug
bash scripts/android_gradle.sh lintDebug
```

如果命令不存在或失败，执行会话必须如实报告，不能假装通过；但报告阻塞前必须先尝试共享上下文里的标准入口。

## 8. 当前推荐启动

当前已经完成第 0 轮主控定标与文档落地。下一步建议启动：

```text
第 1 轮：连续陪伴与低压回访。
```

启动方式：

```text
1. 主控会话先生成第 1 轮产品设计提示语；
2. 必要时新开 ChatGPT 子会话细化低压回访体验；
3. 主控会话收口并生成开发端执行提示语；
4. 开发端按提示语实现；
5. 主控会话验收。
```

界面重设计暂不直接进入开发。等主控生成 image-2 生图提示语后，产品负责人可新开 ChatGPT 生图会话，产出界面方向图后发回主控会话评审。
