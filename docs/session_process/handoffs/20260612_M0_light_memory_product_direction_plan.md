## 4. 5 行摘要

```text
1. 轻记忆定位为“轻轻记得一点点”，不是成长档案、学习画像、聊天记录检索或家长监控。
2. 允许来源限定为奇怪小门、小展台、语言游戏的低敏摘要，不保存原始语音、原始照片、原始聊天全文或错误答案。
3. opening 是首版最适合的低频召回位置，普通聊天只在孩子主动相关时轻接，不强拉旧话题。
4. 建议采用两阶段：先用 Android 本地轻状态和既有小展台数据验证体验，再接后端白名单轻摘要。
5. MVP 不做积分、等级、奖励、打卡、排行榜、学习报告、家长监控面板或复杂记忆管理页。
```

7. 小展台 item 的图片引用是否可被轻记忆读取展示，还是轻记忆只能读取名字和摘要。
8. opening 轻记忆的最终儿童端文案由哪个 master-copy 文档承接。
9. 是否需要新增专门的轻记忆真机验收脚本。
10. 是否需要先做 public repo 数据扫描规则，防止误提交真实家庭材料。

## 8. 风险点

```text
1. 轻记忆如果频繁提起，容易形成监控感。
2. 轻记忆如果强调等待、唯一关系或秘密关系，会制造依赖感。
3. 如果保存原文、原图、语音或游戏回答，会越过数据最小化边界。
4. 语言游戏轻记忆容易滑向成绩化，必须只记偏好不记表现。
5. 家长端如果展示逐字证据或跳过记录，会破坏儿童陪伴信任。
```
