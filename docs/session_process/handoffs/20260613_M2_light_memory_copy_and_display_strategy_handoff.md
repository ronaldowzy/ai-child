## 5. opening 策略

opening 中允许展示轻记忆的条件：

```text
1. ChatUiState.lightMemory.openingRecallCandidateId != null；
2. strangeDoorDemo == null；
3. languageGame == null；
4. 本 ChatViewModel 生命周期内没有 recall 过；
5. lightMemory 没有 mutedForCurrentLifecycle；
6. 当前不是拍照、上传、小展台选择、语言游戏、奇怪小门进行中；
7. 当前没有安全 / 隐私 / 学习 / 睡前收尾等更高优先级场景。
```

opening 中不允许：

```text
1. 每次打开都提；
2. 连续提同一个小发现；
3. 孩子先聊别的后继续提；
4. 孩子短答或跳过后追问；
5. 在奇怪小门进行中插入；
6. 在语言游戏进行中插入。
```

---

## 6. 普通聊天策略

普通聊天只在孩子主动相关时轻接。

允许触发关键词沿用 M1：

```text
小门
奇怪小门
小展台
小发现
刚才那个
我放进去的
帮小白狐
```

普通聊天中禁止：

```text
1. 孩子说新话题时强拉回旧记忆；
2. 孩子负面情绪时提娱乐记忆；
3. 睡前提兴奋玩法；
4. 孩子短答后继续追问；
5. 引导孩子必须回去玩。
```

---

## 7. 奇怪小门策略

奇怪小门中只允许很轻地承接既有 S5 “用小展台里的”路径。

允许：

```text
1. 如果 S5 已显示“用小展台里的”，可让最近小发现排序靠前；
2. 选择页仍使用“选一个小发现”；
3. 点击旧物后的反馈仍使用 S5 已确认文案；
4. 不新增“我记得这个”的专门文案。
```

禁止：

```text
1. 不新增奇怪小门轻记忆提示；
2. 不说“上次你用过”；
3. 不说“它又等着你”；
4. 不新增任务、关卡、奖励；
5. 不改变门状态节奏。
```

## 12. 真机验收标准

后续真机验收判断“不像监控、不像复盘”：

```text
1. 孩子第一感受是小白狐轻轻想起，而不是系统汇报。
2. 文案不复述照片、语音、聊天全文或孩子回答。
3. 文案不出现次数、成绩、等级、奖励、任务或连续记录。
4. 小白狐不催孩子继续旧流程。
5. 孩子跳过或先聊别的后，小白狐自然放下。
6. 轻记忆不抢奇怪小门、语言游戏或普通聊天主线。
7. 家长端没有新增监控感入口或报告项。
8. 仓库、日志和测试材料不包含真实儿童照片、音频、聊天转录或家庭测试材料。
```
