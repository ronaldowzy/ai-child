# M2：小白狐轻记忆儿童端文案与展示策略计划

日期：2026-06-13

执行角色：主控会话

状态：PLAN / 待开发执行会话整理

---

## 1. 背景

M0 / M0-A 已确认“小白狐轻记忆”为下一阶段产品方向。

M1 已完成 Android 本地轻状态与数据合同：

```text
1. LightMemorySnapshot；
2. LightMemorySource；
3. LightMemoryCandidate；
4. LightMemoryStatus；
5. LightMemoryReducer；
6. LightMemorySafetyGate；
7. ChatUiState.lightMemory；
8. openingRecallCandidateId；
9. relatedChatCandidateId。
```

M1 当前只提供状态条件，不提供儿童端最终文案，不真正展示“记得一点点”的体验。

M2 目标是补齐儿童端文案和展示策略，让后续开发有明确 master-copy 和显示边界。

---

## 2. M2 目标

M2 只做产品 / 文案 / 展示策略设计，不编码。

目标：

```text
1. 定义轻记忆儿童端 master-copy；
2. 定义 opening 中什么时候可以提；
3. 定义普通聊天中什么时候可以接；
4. 定义奇怪小门中是否可以轻轻提示旧小发现；
5. 定义哪些情况下必须不提；
6. 定义孩子跳过、换题、先聊别的后的处理；
7. 定义真机验收时如何判断“不像监控、不像复盘”。
```

## 4. 总原则

轻记忆文案必须像“小白狐轻轻想起来”，不能像系统记录。

必须符合：

```text
1. 低频；
2. 短句；
3. 可跳过；
4. 不追问；
5. 不复盘细节；
6. 不评价孩子；
7. 不制造亏欠；
8. 不暗示唯一关系；
9. 不说一直等你；
10. 不说我一直记着你。
```

核心感觉：

```text
小白狐好像轻轻想起一点点，
然后马上把选择权还给孩子。
```

## 6. opening 展示策略

opening 是 M2 首要展示位置。

允许条件：

```text
1. ChatUiState.lightMemory.openingRecallCandidateId != null；
2. strangeDoorDemo == null；
3. languageGame == null；
4. 本 ChatViewModel 生命周期内没有 recall 过；
5. lightMemory 没有 mutedForCurrentLifecycle；
6. 当前不是拍照、上传、小展台选择、语言游戏、奇怪小门进行中；
7. 当前没有安全 / 隐私 / 学习 / 睡前收尾等更高优先级场景。
```

不允许：

```text
1. 每次打开都提；
2. 连续提同一个小发现；
3. 孩子先聊别的后继续提；
4. 孩子短答或跳过后追问；
5. 在奇怪小门进行中插入；
6. 在语言游戏进行中插入。
```

## 7. opening master-copy 候选

### 7.1 小展台小发现来源

适用来源：

```text
ShowcaseItem
ShowcaseAssist
```

允许文案：

```text
我刚才想起一个小发现
{name} 好像还在小展台里安安静静地待着

今天我们也可以聊新的东西
```

更轻版本：

```text
我想起了小展台里的 {name}
它好像轻轻动了一下

不过今天想聊新的也可以
```

注意：

```text
1. 不说“你上次”；
2. 不说“我一直记得”；
3. 不说“它在等你”；
4. 不要求孩子去小展台；
5. 不自动跳转小展台。
```

### 7.2 奇怪小门完成来源

适用来源：

```text
StrangeDoorCompleted
```

允许文案：

```text
我好像想起一扇奇怪小门
有个小东西曾经帮过忙

今天也可以从新的事情开始
```

如果有安全 displayName：

```text
我好像想起 {name}
它曾经帮过一扇奇怪小门

今天也可以从新的事情开始
```

注意：

```text
1. 不说通关；
2. 不说你完成了几次；
3. 不说继续任务；
4. 不显示开门次数；
5. 不做连续回访。
```

### 7.3 最近机关类型来源

适用来源：

```text
StrangeDoorMechanism
```

Round 可用：

```text
我好像想起一个圆圆的小机关
它轻轻咔哒了一下

今天想玩什么都可以
```

Soft 可用：

```text
我好像想起一个软软的小机关
它轻轻挪了一点点

今天想玩什么都可以
```

Shiny 可用：

```text
我好像想起一个亮亮的小机关
它闪了一下又安静了

今天想玩什么都可以
```

注意：

```text
1. 机关文案不暗示任务；
2. 不说还差一步；
3. 不催孩子继续找东西；
4. 不说奖励。
```

## 8. opening 按钮策略

M2 不新增按钮。

opening 轻记忆出现时，仍复用当前普通聊天 / 游戏入口已有选择。

不新增：

```text
1. 去看看；
2. 继续上次；
3. 打开小展台；
4. 再帮一次；
5. 不要记了；
6. 忘掉这个。
```

原因：M1 / M2 只做轻召回，不做新流程入口，不做孩子端显式记忆管理。

## 9. 普通聊天展示策略

普通聊天只在孩子主动相关时轻接。

允许触发关键词沿用 M1：

```text
小门
奇怪小门
小展台
小发现
刚才那个
我放进去的
帮小白狐
```

允许文案结构：

```text
轻轻接住孩子的话
+ 提一个已有安全摘要
+ 不强拉流程
```

示例：

孩子：小展台里有什么？

可回应：

```text
我记得小展台里有一个小发现
{name} 还在那里

你想说说它，或者聊新的都可以
```

孩子：那个小门呢？

可回应：

```text
那扇奇怪小门我也有一点点印象
有个小东西曾经帮过忙

不过我们也可以先聊别的
```

禁止：

```text
1. 孩子说新话题时强拉回旧记忆；
2. 孩子负面情绪时提娱乐记忆；
3. 睡前提兴奋玩法；
4. 孩子短答后继续追问；
5. 引导孩子必须回去玩。
```

## 10. 奇怪小门展示策略

奇怪小门中可以很轻地承接既有 S5 “用小展台里的”路径，但 M2 不新增小门主流程文案。

允许：

```text
1. 如果 S5 已显示“用小展台里的”，可让最近小发现排序靠前；
2. 选择页仍使用“选一个小发现”；
3. 点击旧物后的反馈仍使用 S5 已确认文案；
4. 不新增“我记得这个”的专门文案。
```

禁止：

```text
1. 不新增奇怪小门轻记忆提示；
2. 不说“上次你用过”；
3. 不说“它又等着你”；
4. 不新增任务、关卡、奖励；
5. 不改变门状态节奏。
```

```text

我一直在等你
你终于回来了
你怎么才回来
只有我记得
这是我们的秘密
不要告诉爸爸妈妈

你上次答错了
你已经完成了几关
你连续来了几天
你获得了奖励
你升级了
你应该继续练习
你最近不太会
你比以前厉害
```

也禁止任何：


## 14. 测试与验收策略

M2 是文案计划，不跑 Android 测试。

后续 M3 实现时需要测试：

```text
1. openingRecallCandidateId 为空时不显示轻记忆文案；
2. strangeDoorDemo != null 时不显示；
3. languageGame != null 时不显示；
4. mutedForCurrentLifecycle == true 时不显示；
5. recalledInCurrentLifecycle == true 时不显示；
6. ShowcaseItem 文案只使用安全 name；
7. StrangeDoorCompleted 文案不出现通关、奖励、次数；
8. Round / Soft / Shiny 文案匹配正确；
9. 普通聊天只有主动关键词才接；
10. 先聊别的后本生命周期不再提；
11. 禁止文案扫描通过。
```

真机验收看：

```text
1. 是否像小白狐轻轻想起；
2. 是否没有系统复盘感；
3. 是否没有监控感；
4. 是否没有催促继续；
5. 是否没有学习评价或成绩感；
6. 孩子跳过后是否自然放下。
```
