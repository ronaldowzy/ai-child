# 轻量交接协议 v0.1

项目：`ai-child`

用途：降低主控、监理、ChatGPT 子会话、Codex / Claude Code 开发会话之间的大段文本复制，避免浏览器会话变卡、上下文过长和信息丢失。本文定义后续所有执行会话的交接方式。

更新时间：2026-05-30

---

## 0. 主控结论

后续不再把开发计划、长 diff、长测试日志、完整交接摘要直接粘贴到主控聊天窗口。

默认交接方式改为：

```text
开发方把完整计划 / 完整交接写入仓库文档；
监理和配置方只把文档路径、提交号和 5-10 行摘要发给主控；
主控读取仓库文档后审核。
```

---

## 1. 为什么要改

原先方式的问题：

```text
1. 单次消息过长，浏览器容易卡。
2. 几轮后主控会话上下文膨胀。
3. 大段文本复制容易漏字段。
4. 主控难以追踪哪个版本是最终计划。
5. 开发方交接不沉淀在仓库事实源中。
```

新方式目标：

```text
1. 主控聊天窗口只传短摘要。
2. 完整内容沉淀到仓库。
3. 每次交接有路径、有提交号、有状态。
4. 主控可以按需读取，不靠用户大段复制。
```

---

## 2. 适用范围

以下内容都必须使用轻量交接方式：

```text
1. 开发会话计划。
2. 开发会话交接摘要。
3. 长测试日志。
4. 真机 QA 记录。
5. 子会话专项设计草案。
6. 家庭测试记录汇总。
7. 需要主控审核的长文档。
```

以下内容可以直接发主控：

```text
1. 一句话状态。
2. 文档路径。
3. 提交号。
4. 5-10 行摘要。
5. 需要主控确认的 3-8 个问题。
6. 截图或录屏文件名 / 路径。
```

---

## 3. 文件存放规则

开发会话计划和交接统一放到：

```text
docs/session_process/handoffs/
```

文件命名：

```text
YYYYMMDD_Dx_简短任务名_plan.md
YYYYMMDD_Dx_简短任务名_handoff.md
```

示例：

```text
docs/session_process/handoffs/20260530_D1_companion_object_plan.md
docs/session_process/handoffs/20260530_D1_companion_object_handoff.md
```

如果目录不存在，开发方可以创建。

---

## 4. 监理和配置方给主控的最小消息格式

以后传回主控时，只发：

```text
会话：D1 后端小屋小客人与轻记忆
状态：计划已写入仓库 / 实现已交接 / 需要主控确认
文档：docs/session_process/handoffs/20260530_D1_companion_object_plan.md
提交：abcdef1
摘要：
1. 会新增 companion_object 领域模型、repository、service、迁移和测试。
2. 一个孩子最多一个 active 小客人。
3. 不涉及儿童端文案和家长端文案。
4. 需要主控确认 6 个产品规则问题。
需要确认：
1. ……
2. ……
```

原则：

```text
不要再粘完整计划。
不要再粘完整测试日志。
不要再粘完整 diff。
```

## 5. 开发方计划文档模板

开发方计划文档必须包含：

```text
# Dx 任务计划

## 1. 已阅读文档

## 2. 任务范围理解

## 3. 会修改的文件

## 4. 不会修改的文件

## 5. 是否涉及文案 / prompt / 家长端表达

## 6. 现有代码复用点

## 7. 数据结构 / 接口建议

## 8. 状态流转建议

## 9. 测试策略

## 10. 风险点

## 11. 发现的文档与代码冲突

## 12. 需要主控确认的问题
```

## 6. 开发方交接文档模板

开发方完成实现后，交接文档必须包含：

```text
# Dx 任务交接

## Summary

## Files

## Child-visible change

## Tests

## Safety

## Docs

## Product boundary

## Known issues

## Shared context

## Commit
```

## 7. 长测试日志处理

长日志不要粘贴到聊天。

处理方式：

```text
1. 开发方把完整日志保存为文件。
2. 在交接文档中只摘关键命令和结果。
3. 如果失败，只贴关键错误 20 行以内。
4. 主控需要时再读取具体文件。
```

建议路径：

```text
docs/session_process/handoffs/logs/
```

或开发方本地 artifacts 路径。

---

## 8. 主控审核方式

主控收到短摘要后：

```text
1. 读取仓库中的完整计划或交接文档。
2. 审核范围、文案、数据、安全和测试。
3. 直接给出通过 / 返修 / 阻塞结论。
4. 必要时把审核结论写入对应 review 文档。
```

---

## 9. 立即生效规则

从 D1 起立即执行：

```text
1. D1 需要把完整计划写入 docs/session_process/handoffs/20260530_D1_companion_object_plan.md。
2. 监理和配置方不再向主控粘贴完整计划。
3. 主控只需要文档路径、提交号、摘要和待确认问题。
4. 后续 D2-D5 同样执行本协议。
```

---

## 10. 例外

以下情况可以直接粘贴：

```text
1. 只有 5 行以内的小问题。
2. 主控明确要求粘贴某一段。
3. 工具暂时无法写入仓库文档。
4. 紧急安全问题，需要马上说明。
```

即使例外，也尽量控制在 800 字以内。
