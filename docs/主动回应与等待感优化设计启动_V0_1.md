## 1. 第 2 轮方向

第 2 轮名称：

```text
主动回应与等待感优化
```

核心问题：

```text
孩子与小白狐的对话还不够像自然轮流说话。
```

本轮不是重做语音系统，不是做常开麦克风，不是做情绪识别大系统，而是设计孩子能直接感受到的：

```text
小白狐说完会自然等一下；
孩子想接就接，不想接也没关系；
孩子打断时旧语音能停；
孩子沉默时小白狐能自然收回；
等待、听、想、说的状态更清楚。
```

```text

```

## 三、本轮只设计 5 个核心场景

### 场景 1：小白狐说完后的自然等待

孩子听完小白狐说话后，小白狐短时间等待孩子接话。

重点判断：

```text
等待应该像“我在听你”，不是像“我在催你”；
等待时孩子应该看得懂当前状态；
孩子不说时要自然收回。
```

### 场景 2：孩子打断小白狐

孩子在小白狐说话时开口或点击说话，小白狐应及时停止旧语音，进入听孩子说的状态。

重点判断：

```text
怎样让孩子感觉“它听见我了”；
怎样避免旧语音继续播造成抢话；
打断后是否需要一句可见反馈。
```

### 场景 3：孩子沉默或短答

孩子没有继续说，或只说“嗯”“哦”“不知道”“随便”。

重点判断：

```text
小白狐如何收回等待；
如何不追问；
如何不显得失落、委屈或催促；
什么时候可以轻轻换话题，什么时候应该安静结束。
```

### 场景 4：首段语音等待感

当前真实体验仍可能暴露首段语音慢的问题。文字可以先出现，但孩子仍会感受到小白狐“卡住了”。

重点判断：

```text
文字先出现时，视觉状态如何承接；
首段语音来之前，小白狐应该呈现什么状态；
等待超过一定时间时，是否需要低压力提示；
怎样避免技术感、错误感或焦虑感。
```

### 场景 5：没听清 / 没传好 / 停一下

语音识别失败、网络慢、孩子说“停一下”、孩子取消时，小白狐要有温和、短句、非技术化反馈。

重点判断：

```text
哪些反馈必须儿童可见；
哪些不应显示给孩子；
如何避免 error、ASR、TTS、debug 等技术词；
如何让孩子知道可以重说、先不说或稍后再来。
```
