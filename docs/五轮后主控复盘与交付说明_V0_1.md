# 五轮后主控复盘与交付说明 v0.1

项目：`ai-child`

用途：给家长/产品负责人、后续主控会话、执行会话和开发方说明当前版本真实状态。本文不是新的大规划，而是五轮孩子体验收尾后的交付说明。

更新时间：2026-05-27

---

## 1. 一句话结论

当前版本已经完成五轮孩子可感知体验收尾。后续不要继续开“第六轮大改”，而应进入真机体验微调、小范围问题修复和交付准备。

当前产品重点已经从“内部能力很多”推进到：

```text
孩子打开后先看到小白狐；
孩子可以用语音自然说话；
小白狐文字先回应、语音分段跟上；
小白狐说完后短时间等孩子继续说；
孩子可以拍图给小白狐看；
小白狐偶尔轻轻记得孩子近期兴趣或作品。
```

## 2. 五轮实际完成情况

### 第一轮：界面和小白狐形象丰富

远程提交：

```text
d0e42ff2f8ad6ce3a924237cd70298218d4ba745
ef44e37d89d51853cf7dc84fd29affb5f5753299
```

孩子能感受到的变化：

```text
1. 小白狐不再只是聊天框旁边的形象，而是儿童页主视觉。
2. 竖屏下小白狐在上方更突出。
3. 横屏下小白狐和聊天区权重更接近。
4. 听、想、说、看图、困了、开心等状态更清楚。
```

仍需真机观察：

```text
1. 低配横屏是否仍挤压消息区；
2. 小白狐视觉区域是否足够大；
3. 状态变化是否舒适、不闪烁。
```

### 第二轮：语音朗读延迟收敛

远程提交：

```text
4fbfd49288387a9e4dd009ab36a3ae1a4cfba4c5
8091353cbbecacd1198a3a4f9d17fd9e24520c49
```

孩子能感受到的变化：

```text
1. 回复文字优先出现，不等完整语音。
2. 多段语音并行生成，减少等待完整音频的时间。
3. 安卓端分段播放队列更稳，停止和静音行为更明确。
4. 常见短句开始预热，降低首段等待感。
```

仍需真机观察：

```text
1. 小白狐首段语音是否仍慢；
2. 分段音频是否乱序、重复或漏播；
3. 停一下是否真的立刻停止；
4. 静音后是否只影响后续朗读，不影响文字。
```

### 第三轮：自然说话机制

远程提交：

```text
0bd5a3f031b8e9da3f5686271f4221c3dbe1c7f9
```

孩子能感受到的变化：

```text
1. 小白狐说完后会短时间等孩子继续说。
2. 孩子不用完全理解按钮逻辑，也可以更像轮流对话。
3. 孩子不说时自动收回等待，不长期占用麦克风。
4. 可关闭自然等待，退回点击说话。
```

仍需真机观察：

```text
1. 5 秒等待是否太短或太长；
2. 普通噪声下是否误触发；
3. 孩子打断小白狐时旧语音是否立即停；
4. 家长是否需要更明显的关闭入口。
```

### 第四轮：拍图分享体验

远程提交：

```text
efbfe5c21ada2d069c66e1a162368a141542e155
```

孩子能感受到的变化：

```text
1. 图片入口更像“给小白狐看看”，而不是上传文件。
2. 图片确认卡更自然，不显示格式和大小等技术信息。
3. 小白狐看到图片后优先回应安全具体细节。
4. 普通图片进入起名字、讲故事、说发生了什么等轻创作入口。
5. 明确题目图片仍进入学习分支，但不直接给答案。
```

仍需真机观察：

```text
1. 图片预览高度是否合适；
2. 拍照、相册、取消、失败、成功路径是否顺；
3. 普通玩具、画、积木、书本是否被误判成作业；
4. 看不清时，小白狐是否诚实承认。
```

### 第五轮：轻量记忆和连续陪伴

远程提交：

```text
ad9590ba25b2ca5d4fdf169c854950edc1d2110b
```

孩子能感受到的变化：

```text
1. 小白狐可以偶尔轻轻记得孩子近期兴趣或作品。
2. 同一话题每个会话最多召回一次，避免反复拉旧话题。
3. 孩子短答、换题、表达边界后，系统会暂时停止记忆召回。
4. 睡前不过度召回比赛、游戏、冒险、挑战等高刺激话题。
```

仍需真机观察：

```text
1. 记忆出现频率是否自然；
2. 孩子换题后是否真的放下；
3. 睡前是否足够短、低刺激；
4. 小白狐是否有“盯着孩子”的感觉。
```

## 3. 当前不要继续大改的方向

以下方向后续可以做，但当前不要作为下一阶段主线：

```text
1. 不继续开第六轮体验大改。
2. 不继续全局提示语大改。
3. 不做长期成长档案。
4. 不做积分、徽章、打卡、排行、抽卡、宠物饥饿等机制。
5. 不做复杂学习脚手架。
6. 不做多孩子、多家长、正式账号体系。
7. 不做重型合规和家长治理重设计。
8. 不反复做完整功能测试；只围绕真实问题做验证。
```

## 4. 后续只做三类事

### 4.1 真机体验微调

只处理孩子或家长真实感受到的问题，例如：

```text
小白狐太小；
横屏消息区挤；
小白狐状态变化太快；
首段语音仍慢；
自然说话等待时间不合适；
拍图确认卡太占地方；
图片后续动作看起来还是像功能菜单；
轻记忆出现太频繁；
孩子换题后小白狐仍拉回旧话题。
```

### 4.2 小范围问题修复

只修明确问题，不借机扩展新系统。

每个小修任务都必须写清楚：

```text
当前孩子感受到什么问题；
改完后孩子会感受到什么变化；
修改范围；
不改什么；
如何确认不是原地打转。
```

### 4.3 打包交付准备

整理当前版本给开发方或测试方使用，包括：

```text
当前能力说明；
本地启动和打包命令；
真机测试重点；
已知风险；
不要误改的产品边界；
如果出现问题，应如何写小修任务。
```

## 5. 真机测试重点

后续真机不需要再做无目的全量回归，而是围绕五轮体验结果验证。

建议只看以下链路：

```text
1. 打开儿童页：小白狐是否第一眼可见；
2. 点击说话：录音、识别、自动发送是否顺；
3. 小白狐回复：文字是否先出现，语音是否及时跟上；
4. 停一下：旧语音是否立刻停止；
5. 静音：文字是否保留，后续是否不自动朗读；
6. 自然说话：小白狐说完后等待是否自然；
7. 拍图：拍照、相册、取消、失败、成功是否顺；
8. 图片回应：普通图片是否进入创作分享，题目图片是否进入学习帮助；
9. 轻记忆：是否偶尔出现，不反复、不追踪、不让孩子有压力；
10. 横屏：小白狐、消息区、语音按钮是否互相挤压。
```

---

## 6. 开发方接入注意事项

```text
1. 不要把当前五轮收尾理解为“所有体验已经完美”，它只是完成了孩子可感知体验的第一轮产品化。
2. 后续问题优先从真机录屏、日志和具体用户感受出发，不要抽象成大系统重构。
3. 日志只记录状态、时长、请求标识、字数、数量、成功失败；不要记录孩子原文、图片内容、音频内容和完整接口响应。
4. 不要把安卓系统语音作为正式小白狐声音兜底。
5. 不要把普通图片默认成作业。
6. 不要把记忆做成成长档案或成就系统。
7. 不要把家长日报改成监控报告。
8. 不要用积分、奖励、打卡或连续使用机制提升活跃。
```

## 8. 当前建议

下一步建议只做一次小范围真机体验检查，重点看：

```text
1. 小白狐是否足够吸引孩子第一眼注意；
2. 首段语音是否仍慢；
3. 自然说话 5 秒等待是否合适；
4. 拍图确认卡是否舒服；
5. 轻记忆是否自然。
```

如果其中某一项明显不好，再开对应小修任务。没有明确问题，不继续开新一轮大改。
