# 奇怪小门 Demo 吸引力增强设计 v0.1

项目：`ai-child`

日期：2026-06-05

本文用于 Q1 真机试玩后的下一轮优化。目标不是增加新系统，而是把“奇怪小门”这一关本身做得更有吸引力。

---

## 0. 主控结论

奇怪小门 Demo 首版已经形成闭环：

```text
首页事件 -> 拍照变身 -> 怪问题 -> 小门变化 -> 小展台承接
```

下一步不应继续加新关卡、新地图、新奖励、新成长系统，而应先把第一关做得更好玩。

本轮核心目标：

```text
孩子第一次拍完或答完后，会不会主动想再试一次。
```

本轮优化名：

```text
R1：奇怪小门试玩吸引力增强
```

---

## 1. 当前问题判断

Q1 之后需要重点优化的不是功能数量，而是三件事：

```text
1. 首屏事件感是否足够强；
2. 拍照后“现实物品变道具”的感受是否足够明显；
3. 小门变化是否制造“我再试一个”的动力。
```

如果孩子只觉得“拍完后出现几行字”，这个 Demo 就还不够。

孩子应该感觉到：

```text
我拍的东西真的变了；
小白狐真的用了它；
小门真的动了一下；
我想再找一个试试。
```

## 3. 门状态节奏调整

### 3.1 当前问题

如果一个圆形物品直接让门打开，孩子可能很快结束，缺少“再拍一个”的动力。

### 3.2 调整原则

拍照路径不应一次直接完成，除非是特殊强匹配或已经接近打开。

建议节奏：

```text
第一次有效拍照 -> cracked
第二次有效拍照 -> almost_open
第三次有效拍照 -> open
```

怪问题答对可以直接 open，因为它是另一条快速路径。

### 3.3 规则建议

```text
拍照 round：AdvanceOneStep
拍照 partial：AdvanceOneStep
拍照 unknown：AdvanceOneStep，但反馈更轻
拍照 blocked：None
怪问题答对：Open
怪问题答错：None
```

如当前门已经 almost_open，下一次有效拍照可以 open。

### 3.4 体验目标

孩子拍第一个东西后看到门开了一点，应该产生：

```text
还差一点，我再找一个。
```

## 4. 拍照结果页增强

### 4.1 当前问题

如果拍照后只是文字反馈，儿童感知弱。

### 4.2 结果页应该呈现三层

```text
第一层：孩子拍的东西
第二层：它变成的奇怪道具
第三层：小门发生的变化
```

### 4.3 页面结构建议

拍照成功后页面应包含：

```text
上方：小门当前状态
中间：变身道具卡
下方：操作按钮
```

变身道具卡内容：

```text
我看见了：{现实物品}

在小白狐的世界里
它变成了：{怪道具名}

小白狐用它{动作}
{小门反应}
```

如果可能，卡片左侧或上方显示孩子拍摄图片的小缩略图。

注意：缩略图不应变成相册或展台主视觉，只用于让孩子确认“这是我刚拍的东西”。

---

## 5. 小门变化要更明显

### 5.1 视觉节奏

每次状态变化时，应有轻微反馈：

```text
closed -> cracked：门缝出现一点暖光
cracked -> almost_open：门缝变大，圆锁轻轻松动
almost_open -> open：门打开，暖风出来
```

### 5.2 使用现有素材

优先使用 S1 已接入素材：

```text
strange_door_cracked.webp
strange_door_almost_open.webp
strange_door_open.webp
strange_door_success_glow.webp
```

不新增素材，先用现有成功光效做轻叠加。

### 5.3 动效原则

允许：

```text
淡入
轻微缩放
轻微晃动
柔和光效出现
```

禁止：

```text
爆闪
金币
胜利光
通关动画
彩带
领奖感
```

## 7. 小门反应词池补充

开发方只能使用本词池。


```text

```


## 8. 乱拍也要好玩

### 8.1 原则

乱拍不是失败，而是“奇怪解释”。

孩子拍到不太相关的东西时，不要说“不符合要求”。

应该说：

```text
这个东西有点奇怪
小门看了好久
小白狐说：也许可以试试
```

### 8.2 但安全内容不能乱接

以下仍必须拒绝：

```text
人脸
学校
地址
证件
隐私文字
作业题
医疗/暴力/惊吓内容
```

拒绝文案仍用：

```text
这张图不太适合变成开门道具
我们换一个小东西试试
```

## 9. 小展台出现时机调整

### 9.1 当前问题

拍照结果后立刻显示“放进小展台”，可能会分散主玩法。

### 9.2 建议

拍照结果后仍可显示，但视觉优先级降低。

按钮排序：

```text
再找一个
动脑试试
放进小展台
```

原因：

```text
先鼓励继续玩，再承接保存。
```

门打开后，可以更自然地提示保存：

```text
这个小发现帮了大忙
要不要放进小展台？
```

如果本轮不新增这句话，则只调整按钮优先级，不改保存流程。

## 10. 首屏事件感增强

首屏如果显得像普通聊天页，应调整：

```text
1. 小门更大；
2. 小白狐和小门距离更近；
3. 圆锁更明显；
4. “找东西帮忙”按钮更突出；
5. 普通输入区域继续隐藏；
6. 小展台和家长入口保持弱入口。
```

儿童第一眼目标：

```text
小白狐过不去了，我要帮它。
```

```text

3. 孩子是否愿意再拍；
4. 拍错是否仍好玩。
```

## 12. R1 验收标准

不是看代码是否通过，而是看试玩效果。

必须回答：

```text
1. 第一眼是否能看出“小白狐被门挡住”；
2. 拍照后是否能看出“我的东西变成道具”；
3. 门是否真的发生明显变化；
4. 孩子是否愿意再找一个东西；
5. 小展台是否只是承接，没有抢主玩法。
```

最终验收题：

```text
孩子是否主动说：我再找一个给它看看。
```
