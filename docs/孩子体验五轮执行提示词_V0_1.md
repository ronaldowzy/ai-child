# 孩子体验五轮执行提示词 v0.2

项目：`ai-child`

用途：给多个干净执行会话使用。每个执行会话只负责一个清晰方向，避免互相覆盖，也避免继续把简单问题复杂化。

总原则：当前优先提升孩子实际体验。尽量少讨论内部结构，多做孩子能直接感受到的改进。

---

## 0. 所有执行会话共同要求

复制任一提示词前，先附上以下共同要求。

```text
你正在参与儿童 AI 成长陪伴 App ai-child 的下一阶段体验收尾。GitHub 仓库是 ronaldowzy/ai-child。

请先同步远程 main 最新状态，阅读：

README.md
AGENTS.md
docs/孩子体验五轮收尾计划_V0_1.md
docs/孩子体验五轮执行提示词_V0_1.md
docs/小白狐提示语与文案规范_V0_1.md
docs/项目现状与后续路线图_V0_3.md
docs/PRODUCT_DECISIONS_V0_1.md
docs/session_process/SHARED_CONTEXT_V0_1.md

本轮目标是孩子能直接感受到体验提升，不是继续堆内部规划。

全程尽量使用中文说明。代码路径、类名、函数名可以保持仓库原名。

不得做：
1. 不改学习深度脚手架；
2. 不做合规重设计；
3. 不做多孩子、多家长、正式账号体系；
4. 不新增积分、打卡、奖励、宠物饥饿、排行、抽卡或上瘾机制；
5. 不让小白狐说“我一直等你”“只有我懂你”“这是我们的小秘密”；
6. 不把假路径或兜底路径写成真实体验通过；
7. 不跨范围大改。

日志要求：
1. 可以记录状态、时长、段序号、请求标识、队列长度、缓存命中、成功失败数量；
2. 不要记录孩子原文、图片内容、音频内容、家长密码、完整接口响应正文；
3. 如果现有日志已经记录孩子原文片段，本轮涉及到该文件时应顺手脱敏。

提交要求：
1. 任务完成后必须提交并推送到远程 main；
2. 最终报告必须给出远程 main 上可查到的提交编号；
3. 如果没有远程提交编号，视为任务没有完成；
4. 如果因为权限或冲突不能推送，必须明确说出原因，并列出本地提交编号或未提交修改。

完成后必须说明：
1. 远程 main 提交编号；
2. 修改文件；
3. 孩子能感受到什么变化；
4. 运行了哪些测试；
5. 未运行测试的真实原因；
6. 是否影响安全、语音、图片、家长入口或数据保存；
7. 是否需要后续会话接手。
```

## 5. 第三轮会话：自然说话机制

当前推荐启动本会话。建议先用一个会话做最小闭环，不要一开始拆太多并行。

```text
任务：在点击说话基础上增加轻量自然说话机制，让孩子更像在和小白狐轮流说话。

范围：
安卓录音状态、说话等待窗口、自动收回、家长/调试开关、播放打断、必要后端接口适配。

当前基础：
1. 第一轮已经把小白狐做成主视觉。
2. 第二轮已经优化后端语音并行生成、常见短句预热、安卓播放队列和停止/静音行为。
3. 现在可以开始做轻量自然说话，但不能做永久监听。

先做的事：
1. 检查安卓播放队列和录音相关日志，去掉孩子原文片段，只保留字数、段序号、请求标识、队列长度。
2. 梳理当前点击说话流程：开始录音、停止录音、上传识别、自动发送、播放回复、停止朗读。
3. 找到最小改动点，不重写整套语音系统。

目标：
1. 小白狐说完后，可以短时间等待孩子继续说。
2. 孩子开始说话时旧语音停止。
3. 没听清时温和提示重说。
4. 孩子不说时自动收回等待，不长期开麦。
5. 可以通过设置关闭，退回点击说话。
6. 不保存不必要的原始音频。
7. 儿童可见文案不出现技术词。

不要做：
1. 不做永久监听；
2. 不做复杂唤醒词；
3. 不绕过现有安全和语音上传规则；
4. 不保存额外原始音频；
5. 不改学习逻辑、家长日报、图片分享、账号体系；
6. 不新增重型音频库，除非非常必要并先说明。

验收：
1. 安静环境、普通噪声、孩子停顿、打断小白狐说话四类场景可验证。
2. 关闭开关后恢复点击说话。
3. 自动等待不会长时间占用麦克风。
4. 孩子开始说话时，旧语音立即停止。
5. 没听清时提示自然，不出现技术词。
6. 有测试覆盖状态转换或关键 ViewModel 行为。
7. 最终报告必须给出远程 main 提交编号；没有远程提交编号视为未完成。
```

2. 低配横屏不挤压主内容。
3. 儿童文案不出现技术词。
4. 最终报告必须给出远程 main 提交编号；没有远程提交编号视为未完成。

## 7. 第四轮会话 B：后端图片回应和创作入口

待第三轮完成后，可与第四轮会话 A 并行。

```text
任务：优化后端图片回应，让普通图片优先成为表达和创作入口，而不是默认作业或隐私风险。

范围：
后端图片上下文、图片回应文案、后续动作、相关测试。

目标：
1. 普通图片优先回应一个安全具体细节。
2. 提供轻量创作入口：起名字、讲故事、说发生了什么。
3. 只有明确是作业求助时才进入学习分支。
4. 看不清时承认看不清。
5. 已有图片上下文时不能说“我看不到图片”。

验收：
1. 用画、玩具、积木、书本、模糊图、题目图等测试。
2. 普通图片不默认进入作业模式。
3. 有测试覆盖“看不清”和“已有图片上下文”。
4. 日志不记录图片内容或孩子原文。
5. 最终报告必须给出远程 main 提交编号；没有远程提交编号视为未完成。
```

## 8. 第五轮会话：轻量记忆和连续陪伴

待第三、四轮稳定后启动。

```text
任务：让小白狐能轻轻记得孩子近期兴趣或展示过的小东西，但不追踪、不施压、不制造依赖。

范围：
后端记忆召回、兴趣种子、图片作品记忆、对话上下文注入、测试。

目标：
1. 小白狐偶尔提起孩子近期兴趣或作品。
2. 召回频率低，不连续追旧话题。
3. 孩子换题、短答、不想聊时立即放下。
4. 睡前不召回兴奋话题。
5. 不说“我一直等你”“只有我懂你”“我最懂你”。
6. 儿童端不暴露敏感内容。

不要做：
1. 不做长期成长档案界面；
2. 不做成就、徽章、连续打卡；
3. 不让小白狐表现得像一直盯着孩子。

验收：
1. 同一兴趣不会高频重复出现。
2. 孩子拒绝或换题后不继续拉回旧记忆。
3. 覆盖普通兴趣、图片作品、未完成小话题、睡前场景。
4. 有依赖表达禁用测试。
5. 日志不记录孩子原文。
6. 最终报告必须给出远程 main 提交编号；没有远程提交编号视为未完成。
```

## 9. 主控合并检查

每轮结束后，主控会话按以下顺序检查：

```text
1. 是否已经推送到远程 main；
2. 是否给出远程 main 提交编号；
3. 是否真正提升孩子可感知体验；
4. 是否没有引入上瘾、依赖、监控或过重治理；
5. 是否没有跨范围大改；
6. 是否有测试或明确手工验证；
7. 是否存在记录孩子原文、图片内容、音频内容或家长敏感信息的日志；
8. 是否需要更新 docs/孩子体验五轮收尾计划_V0_1.md；
9. 是否可以进入下一轮。
```
