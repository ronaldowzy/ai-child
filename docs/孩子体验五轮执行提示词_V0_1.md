# 孩子体验五轮执行提示词 v0.1

项目：`ai-child`

用途：给多个干净执行会话使用。每个执行会话只负责一个清晰方向，避免互相覆盖，也避免继续把简单问题复杂化。

总原则：当前优先提升孩子实际体验。尽量少讨论内部结构，多做孩子能直接感受到的改进。

---

## 0. 所有执行会话共同要求

复制任一提示词前，先附上以下共同要求。

```text
你正在参与儿童 AI 成长陪伴 App ai-child 的下一阶段体验收尾。GitHub 仓库是 ronaldowzy/ai-child。

请先同步 main 最新状态，阅读：

README.md
AGENTS.md
docs/孩子体验五轮收尾计划_V0_1.md
docs/小白狐提示语与文案规范_V0_1.md
docs/项目现状与后续路线图_V0_2.md
docs/PRODUCT_DECISIONS_V0_1.md
docs/session_process/SHARED_CONTEXT_V0_1.md

本轮目标是孩子能直接感受到体验提升，不是继续堆内部规划。

全程尽量使用中文说明。代码路径、类名、函数名可以保持仓库原名。

不得做：
1. 不改学习深度脚手架；
2. 不做合规重设计；
3. 不做多孩子、多家长、正式账号体系；
4. 不新增积分、打卡、奖励、宠物饥饿、排行、抽卡或上瘾机制；
5. 不让小白狐说“我一直等你”“只有我懂你”“这是我们的小秘密”；
6. 不把 mock 或兜底路径写成真实体验通过；
7. 不跨范围大改。

完成后必须说明：
1. 修改文件；
2. 孩子能感受到什么变化；
3. 运行了哪些测试；
4. 未运行测试的真实原因；
5. 是否影响安全、语音、图片、家长入口或数据保存；
6. 是否需要后续会话接手。
```

## 1. 第一轮会话 A：安卓界面和小白狐主视觉

适合并行执行，主要负责安卓界面布局和状态呈现。

```text
任务：优化儿童主界面，让小白狐成为孩子打开页面后第一眼看到的中心，而不是普通聊天框附带一个形象。

范围：
android/ 下儿童聊天主界面、小白狐展示组件、状态呈现、横屏布局、输入区布局。

目标：
1. 小白狐在儿童页更突出。
2. 听、想、说、拍图处理中、出错、困了等状态更清楚。
3. 语音按钮、拍图按钮、停止/静音按钮围绕孩子使用路径组织，不挤压小白狐。
4. 横屏下更像“小白狐陪我说话”的空间。
5. 不加入积分、奖励、任务、打卡、宠物饥饿等机制。

请先盘点当前相关文件，再给最小实现方案。优先改界面结构和状态呈现，不改后端接口。

验收：
1. 打开儿童页 3 秒内能明确看到小白狐是核心。
2. 录音、识别、思考、说话、图片处理中，小白狐状态变化清楚。
3. 横屏不挤压主内容。
4. 关键状态有测试或可复核的截图/说明。
```

---

## 2. 第一轮会话 B：小白狐资源和状态丰富接入

适合与第一轮会话 A 并行。主要负责资源映射和状态细化，不重做界面大布局。

```text
任务：检查并丰富小白狐现有状态资源接入，让已有状态图真正服务儿童体验。

范围：
小白狐资源清单、状态映射、安卓状态解析、资源加载、降级显示。

目标：
1. 确认当前已有多少状态资源，哪些已接入，哪些未触发。
2. 补齐听、想、说、图片处理中、网络错误、安全提醒、睡前、开心等状态映射。
3. 状态变化温和，不闪烁、不刺激。
4. 没有资源时使用稳定降级，不显示破图或空白。

不要做：
1. 不重新设计整套美术；
2. 不引入重型动画引擎；
3. 不把开心状态绑定奖励、打卡、正确答案。

验收：
1. 状态映射表清晰。
2. 关键状态有测试覆盖。
3. 资源缺失时有降级。
4. 与界面会话不互相覆盖；如要改同一文件，先说明。
```

## 3. 第二轮会话 A：后端语音延迟收敛

需在第一轮后执行，也可与第二轮会话 B 并行。

```text
任务：降低小白狐语音等待感，重点处理后端语音生成、缓存、分段返回和日志。

范围：
后端语音生成、语音缓存、分段朗读、流式返回、时间日志。

目标：
1. 文字先返回，不等待完整语音。
2. 第一段语音尽快生成并可播放。
3. 常见短句可以预热或缓存。
4. 日志能看清：文字出现、语音请求、第一段语音完成、返回安卓的时间。
5. 语音失败时不冒充成功，不要求安卓回退系统音色。

不要做：
1. 不改模型供应商核心抽象；
2. 不恢复系统语音作为正式小白狐声音；
3. 不用动画掩盖长时间无响应。

验收：
1. 至少提供一个本地或后端脚本验证路径。
2. 能看到关键时间日志。
3. 停止或失败不造成旧语音继续播放。
4. 说明当前最慢点在哪里。
```

## 4. 第二轮会话 B：安卓语音播放队列和打断

可与后端语音会话并行。

```text
任务：优化安卓端小白狐语音播放队列、停止、静音和打断体验。

范围：
安卓端远程音频播放、分段队列、停止朗读、静音、录音打断。

目标：
1. 分段语音按顺序播放，不混播、不乱序。
2. 孩子点击停止后，旧音频立即停止。
3. 孩子开始录音时，当前小白狐语音自动停止。
4. 静音只影响后续自动朗读，不隐藏文字内容。
5. 语音失败时显示儿童能懂的温和提示，不出现技术词。

验收：
1. 有 ViewModel 或播放队列测试。
2. 有手工验证步骤。
3. 覆盖停止、静音、录音打断、连续分段四类场景。
```

## 5. 第三轮会话：自然说话机制

必须等第二轮确认语音等待感已经改善后再启动。

```text
任务：在点击说话基础上增加轻量自然说话机制，让孩子更像在和小白狐轮流说话。

范围：
安卓录音状态、说话等待窗口、自动收回、家长/调试开关、后端接口适配。

目标：
1. 小白狐说完后，可以短时间等待孩子继续说。
2. 孩子开始说话时旧语音停止。
3. 没听清时温和提示重说。
4. 孩子不说时自动收回等待，不长期开麦。
5. 可以通过设置关闭，退回点击说话。
6. 不保存不必要的原始音频。

不要做：
1. 不做永久监听；
2. 不做复杂唤醒词；
3. 不绕过现有安全和语音上传规则。

验收：
1. 安静环境、普通噪声、孩子停顿、打断小白狐说话四类场景可验证。
2. 关闭开关后恢复点击说话。
3. 儿童可见文案没有技术词。
```

## 6. 第四轮会话 A：安卓拍图分享体验

可与第四轮会话 B 并行。

```text
任务：优化安卓端“拍给小白狐看”的儿童体验，让它更像分享和创作入口。

范围：
安卓图片入口、本地图片确认卡、图片处理中状态、后续动作展示。

目标：
1. 拍图入口更自然，不像上传文件。
2. 图片发送后有清楚的本地确认卡。
3. 小白狐进入图片处理中状态。
4. 后续轻动作不挤压语音按钮和小白狐区域。
5. 不默认作业，不做长期图库。

验收：
1. 拍照、相册、取消、上传失败、处理中、成功六类路径可验证。
2. 低配横屏不挤压主内容。
3. 儿童文案不出现技术词。
```

---

## 7. 第四轮会话 B：后端图片回应和创作入口

可与安卓拍图体验并行。

```text
任务：优化后端图片回应，让普通图片优先成为表达和创作入口，而不是默认作业或隐私风险。

范围：
后端图片上下文、图片回应文案、后续动作、相关测试。

目标：
1. 普通图片优先回应一个安全具体细节。
2. 提供轻量创作入口：起名字、讲故事、说发生了什么。
3. 只有明确是作业求助时才进入学习分支。
4. 看不清时承认看不清。
5. 已有图片上下文时不能说“我看不到图片”。

验收：
1. 用画、玩具、积木、书本、模糊图、题目图等测试。
2. 普通图片不默认进入作业模式。
3. 有测试覆盖“看不清”和“已有图片上下文”。
```

## 8. 第五轮会话：轻量记忆和连续陪伴

建议在第一轮和第二轮稳定后启动。

```text
任务：让小白狐能轻轻记得孩子近期兴趣或展示过的小东西，但不追踪、不施压、不制造依赖。

范围：
后端记忆召回、兴趣种子、图片作品记忆、对话上下文注入、测试。

目标：
1. 小白狐偶尔提起孩子近期兴趣或作品。
2. 召回频率低，不连续追旧话题。
3. 孩子换题、短答、不想聊时立即放下。
4. 睡前不召回兴奋话题。
5. 不说“我一直等你”“只有我懂你”“我最懂你”。
6. 儿童端不暴露敏感内容。

不要做：
1. 不做长期成长档案界面；
2. 不做成就、徽章、连续打卡；
3. 不让小白狐表现得像一直盯着孩子。

验收：
1. 同一兴趣不会高频重复出现。
2. 孩子拒绝或换题后不继续拉回旧记忆。
3. 覆盖普通兴趣、图片作品、未完成小话题、睡前场景。
4. 有依赖表达禁用测试。
```

## 9. 主控合并检查

每轮结束后，主控会话按以下顺序检查：

```text
1. 是否真正提升孩子可感知体验；
2. 是否没有引入上瘾、依赖、监控或过重治理；
3. 是否没有跨范围大改；
4. 是否有测试或明确手工验证；
5. 是否需要更新 docs/孩子体验五轮收尾计划_V0_1.md；
6. 是否可以进入下一轮。
```
