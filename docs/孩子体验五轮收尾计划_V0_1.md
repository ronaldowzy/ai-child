# 孩子体验五轮收尾计划 v1.0

项目：`ai-child`

用途：记录当前阶段“孩子可感知体验五轮收尾”的目标、执行结果和后续边界。

当前状态：五轮孩子体验收尾已经全部完成并推送到远程主分支。下一阶段不再开第六轮大改，而是进入小范围体验微调、真机感受确认和必要问题修复。

---

## 1. 当前阶段结论

当前项目已经完成一轮全局提示语和儿童端文案优化，也完成了五轮孩子可感知体验收尾：

```text
1. 小白狐视觉中心化和状态丰富；
2. 语音朗读延迟收敛；
3. 轻量自然说话机制；
4. 拍图分享体验；
5. 轻量记忆和连续陪伴。
```

这五轮的目标不是把所有功能做完，而是把产品从“内部能力基本齐全”推进到“孩子打开后能明显感到小白狐在这里、愿意说、愿意看、愿意拍给小白狐看”。

后续仍然坚持：只做孩子能直接感受到的改进，不再为了内部规划反复打转。

---

## 2. 暂时后置的方向

以下方向不是取消，而是不作为当前近期主线：

```text
1. 学习脚手架深度优化：先后置。当前只保留“不直接给答案、引导审题和第一步”。
2. 合规和家长治理重设计：家庭内测阶段先不做重型治理，以后正式上线前再补。
3. 多孩子、多家长、正式账号体系：后置。
4. 长期成长档案和成长巢界面：后置，等主体验成立后再做。
5. 继续大范围提示语优化：暂停。除非真实使用中发现明显不好，再局部改。
6. 反复做完整功能测试：已有多轮测试，不再为了测试而测试；后续测试只围绕实际改动验证。
```

状态：已完成，主控检查通过。

远程提交：

```text
ad9590ba25b2ca5d4fdf169c854950edc1d2110b
```

完成内容：

```text
1. 同一记忆话题每个会话最多召回一次。
2. 孩子短答、换题、表达边界后，记忆召回抑制 120 秒。
3. 睡前不召回比赛、游戏、冒险、挑战、任务、奖励等高刺激话题。
4. 不同会话的召回状态独立，不互相影响。
5. 对话服务在调用小白狐前加入轻量参与度预检查。
6. 开放对话提示语加入轻记忆召回规则。
7. 记忆召回规则强调：孩子不接话就立刻放下，图片作品只用宽泛线索，不说具体图片内容。
8. 新增轻量记忆召回测试，覆盖同话题限频、短答抑制、换题抑制、睡前过滤、作品召回、禁用依赖话术、无原文泄露、跨会话独立等场景。
```

主控检查结论：

```text
1. 第五轮已推送到远程主分支。
2. 改动范围集中在轻量记忆、对话上下文注入和测试，没有启动成长档案、积分、徽章、连续打卡或正式账号体系。
3. 记忆上下文不包含孩子原文，日志仍以标识、类型和数量为主。
4. 轻记忆召回被限制为低频、可放下、睡前不过度刺激。
5. 五轮孩子体验收尾已完成。
```

## 8. 五轮结束后的状态

五轮结束后，当前产品应达到：

```text
1. 孩子打开后先看到小白狐；
2. 小白狐有更清楚的听、想、说、看图、困了、开心等状态；
3. 小白狐回复文字先出现，语音分段生成和播放，不再明显等完整语音；
4. 小白狐说完后可以短时间等待孩子继续说；
5. 孩子拍图后能看到更自然的确认卡，小白狐能围绕普通图片做轻创作回应；
6. 小白狐偶尔轻轻记得孩子近期兴趣或作品，但不会反复拉旧话题；
7. 整体没有积分、打卡、奖励、秘密关系、唯一朋友、监控感或重治理感。
```

## 9. 后续工作边界

五轮完成后，不再继续开“第六轮体验大改”。后续只做三类事：

```text
1. 真机体验微调：基于实际设备感受，修界面挤压、语音等待、自然说话误触、图片卡片不舒服等具体问题。
2. 小范围问题修复：只修明确问题，不借机扩展成长档案、复杂学习、合规重设计。
3. 打包交付准备：整理当前能力、开发方接入说明、测试注意事项和已知风险。
```

如果真实使用中发现某一项体验仍明显不好，应写成小修任务，例如：

```text
小白狐横屏区域仍太小；
自然说话 5 秒等待太短或太长；
拍图确认卡图片预览太矮；
语音首段仍慢；
轻记忆出现得太频繁；
小白狐偶尔拉回孩子不想聊的话题。
```

每个小修任务都必须说明：孩子当前感受是什么、改完后孩子能感受到什么变化、如何确认不是原地打转。
