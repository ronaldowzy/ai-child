# 孩子体验五轮收尾计划 v0.1

项目：`ai-child`

用途：把下一阶段从“继续扩展内部能力”收敛到“孩子能明显感受到更好玩、更顺、更像一个有形象的小白狐陪伴者”。

本文件优先级高于旧路线图里过重、过长、过早的后续规划。当前不再把合规、复杂学习脚手架、长期成长档案、多孩子多家长等作为近期主线。它们保留为以后正式上线前再处理的事项。

---

## 1. 当前阶段判断

当前项目已经完成一轮全局提示语和儿童端文案优化。继续大范围改提示语的边际收益不高。

当前真正影响孩子体验的不是“文档里设计得是否完整”，而是：

```text
1. 小白狐在屏幕上是否足够有存在感；
2. 儿童主界面是否像孩子愿意玩的陪伴界面，而不是测试聊天界面；
3. 小白狐说话是否太慢；
4. 孩子说话、听回复、打断、重说是否顺；
5. 拍图给小白狐看是否像表达和创作，而不是上传文件；
6. 小白狐是否能轻轻记得孩子关心过的东西，但不制造依赖。
```

因此后续五轮以内只做能明显提升孩子体验的事项。

---

## 2. 暂时后置的方向

以下方向不是取消，而是不作为当前五轮主线：

```text
1. 学习脚手架深度优化：先后置。当前只保留“不直接给答案、引导审题和第一步”。
2. 合规和家长治理重设计：家庭内测阶段先不做重型治理，以后正式上线前再补。
3. 多孩子、多家长、正式账号体系：后置。
4. 长期成长档案和成长巢界面：后置，等主体验成立后再做。
5. 继续大范围提示语优化：暂停。除非真实使用中发现明显不好，再局部改。
6. 反复做完整功能测试：已有多轮测试，不再为了测试而测试；后续测试只围绕每轮实际改动验证。
```

---

## 3. 五轮收尾总目标

五轮内把产品从“功能已经可用”推进到“孩子明显更愿意看、愿意说、愿意拍给小白狐看”。

五轮不是五个长期阶段，而是五次有边界的修改窗口。每轮必须能回答：

```text
当前是什么样？
这一轮后变成什么样？
孩子能直接感受到什么变化？
怎么验收不是原地打转？
```

---

## 4. 第一轮：界面和小白狐形象丰富

### 当前状态

小白狐资源和状态机已有基础，安卓端也有儿童主界面和横屏方向，但整体仍偏“聊天产品加形象组件”。孩子首先是视觉思维优先，当前产品没有把小白狐作为第一感知中心。

### 优化后目标

进入儿童页后，孩子首先看到的是一个明确、温和、有表情和状态变化的小白狐，而不是一个普通聊天框。

目标状态：

```text
1. 小白狐在主界面中占据更核心的位置。
2. 听、想、说、拍图、出错、困了、开心等状态有更明显但不刺激的表现。
3. 聊天气泡、输入区、按钮围绕小白狐组织，不抢占画面。
4. 横屏下更像“小白狐陪我说话”的空间，而不是表单界面。
5. 不增加积分、奖励、任务、宠物饥饿或打卡机制。
```

### 验收标准

```text
1. 打开儿童页 3 秒内能明确感到“小白狐在这里”。
2. 录音、思考、说话、拍图处理中，小白狐状态变化清楚。
3. 低配平板横屏不挤压主要内容。
4. 不出现上瘾式动画和奖励机制。
5. 安卓单测或界面状态测试覆盖主要状态映射。
```

---

## 5. 第二轮：语音朗读延迟收敛

### 当前状态

后端已经有小白狐语音、音频链接、分段朗读、缓存和安卓播放队列，但实际感受仍是小白狐说话慢。只要说话慢，孩子就会觉得它“不聪明”或“不在场”。

### 优化后目标

让孩子感觉小白狐很快开始回应。即使完整音频还没好，也要先看到小白狐在想、文字先出现、第一段声音尽快开始。

目标状态：

```text
1. 文字先稳定出现，不等完整语音。
2. 小白狐思考和说话状态及时切换。
3. 第一段语音尽快播放。
4. 后续语音按段顺序播放，不混播、不乱序。
5. 语音失败时不切回系统音色冒充小白狐。
6. 常见短句可以预热或缓存，减少等待。
```

### 验收标准

```text
1. 有日志能看到：文字出现时间、语音开始请求时间、第一段语音可播时间、安卓开始播放时间。
2. 至少 10 次真实对话记录中，大部分不会出现长时间空白等待。
3. 停止朗读后不继续播旧音频。
4. 没有系统音色和小白狐音色混播。
5. 修改集中在语音生成、缓存、播放队列和状态提示，不扩大到无关模块。
```

---

## 6. 第三轮：自然说话机制

### 当前状态

当前儿童默认是语音优先，但仍以点击说话为主。免按键或自动识别说话可以做，但必须排在语音延迟之后。否则孩子刚说完，小白狐慢半拍，体验会更差。

### 前置条件

第二轮语音延迟必须明显改善，否则不进入本轮。

### 优化后目标

孩子和小白狐的说话节奏更自然。可以先做轻量自动识别，不一定一步到位做完整持续监听。

目标状态：

```text
1. 小白狐说完后，可以进入短时间等待孩子说话的状态。
2. 孩子开始说话时，旧语音立即停止。
3. 没听清时温和提示重说。
4. 孩子不说话时自动收回等待，不一直监听。
5. 家长可以关闭自动识别说话。
6. 不保存不必要的原始音频。
```

### 验收标准

```text
1. 孩子无需理解按钮逻辑，也能自然说下一句。
2. 自动等待不会长时间占用麦克风。
3. 错误提示没有技术词。
4. 开关关闭后退回点击说话。
5. 真机至少覆盖安静环境、普通噪声、孩子停顿、孩子打断小白狐说话四类场景。
```

---

## 7. 第四轮：拍图分享体验

### 当前状态

拍图上传和图片理解已接入，但产品感还不够。当前重点不应是继续扩展识别类型，而是让“拍给小白狐看”变成孩子表达和创作入口。

### 优化后目标

孩子拍一张画、玩具、积木、书页、自然物，小白狐能看见一个安全具体的细节，并给孩子一个轻松的创作或表达入口。

目标状态：

```text
1. 拍图后本地图片确认卡更清楚、更像分享给小白狐。
2. 小白狐优先回应一个安全具体细节。
3. 提供轻量后续入口：起名字、讲故事、说发生了什么。
4. 只有明确是作业求助时才进入学习分支。
5. 看不清时承认看不清，不假装看见。
6. 不默认把图片当作作业检查或隐私风险。
```

### 验收标准

```text
1. 用画、玩具、积木、书本、模糊图、题目图等测试图片验证。
2. 普通图片不默认进入作业模式。
3. 有图片上下文时不说“我看不到图片”。
4. 至少多数图片能给出一个安全具体细节或自然表达入口。
5. 不新增复杂相机系统，不做长期图库。
```

---

## 8. 第五轮：轻量记忆和连续陪伴

### 当前状态

项目已有记忆和兴趣种子，但还没有把它打磨成孩子能感受到的“轻轻记得”。这件事可以做，但要克制，不追踪、不施压、不制造依赖。

### 优化后目标

小白狐偶尔记得孩子之前喜欢或展示过的小东西，并以很轻的方式接一下。孩子不接话时，小白狐马上放下。

目标状态：

```text
1. 小白狐偶尔提起孩子近期兴趣或作品。
2. 召回频率低，不连续追旧话题。
3. 孩子换题、短答、不想聊时立即尊重。
4. 睡前不召回兴奋话题。
5. 不说“我一直等你”“我最懂你”“只有我记得你”。
6. 家长日报可以使用宽泛线索，但儿童端不暴露敏感内容。
```

### 验收标准

```text
1. 同一兴趣不会高频重复出现。
2. 孩子拒绝或换题后不继续拉回旧记忆。
3. 不出现依赖和秘密关系表达。
4. 至少覆盖普通兴趣、图片作品、未完成小话题、睡前场景。
5. 修改不扩大到长期成长档案。
```

---

## 9. 多会话并行原则

可以并行，但必须按依赖关系来。

可并行：

```text
第一轮界面和小白狐形象：安卓界面会话 + 资源接入会话可并行。
第二轮语音延迟：后端语音会话 + 安卓播放队列会话可并行。
第四轮拍图分享：安卓图片体验会话 + 后端图片回应会话可并行。
```

不可并行或需等待：

```text
第三轮自然说话机制必须等第二轮语音延迟改善后再做。
第五轮轻量记忆最好等第一轮和第二轮稳定后再做。
学习深度优化和合规重设计不进入当前五轮。
```

---

## 10. 当前推荐执行顺序

```text
第 1 轮：界面和小白狐形象丰富。
第 2 轮：语音朗读延迟收敛。
第 3 轮：自然说话机制。
第 4 轮：拍图分享体验。
第 5 轮：轻量记忆和连续陪伴。
```

如果第二轮没有明显改善语音等待感，第三轮暂停，优先继续处理语音延迟。

---

## 11. 成功标准

五轮结束后，产品应该从“内部能力基本齐全”变成：

```text
孩子打开后先看到小白狐；
孩子愿意说话；
小白狐回应不慢到让孩子出戏；
孩子能自然拍图给小白狐看；
小白狐偶尔轻轻记得孩子的小兴趣；
整个体验没有积分、打卡、依赖、监控或过重治理感。
```
