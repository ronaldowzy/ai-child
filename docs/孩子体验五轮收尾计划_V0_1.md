# 孩子体验五轮收尾计划 v0.3

项目：`ai-child`

用途：把下一阶段从“继续扩展内部能力”收敛到“孩子能明显感受到更好玩、更顺、更像一个有形象的小白狐陪伴者”。

当前状态：第一轮界面和小白狐形象丰富、第二轮语音朗读延迟收敛都已经完成并推送到远程主分支。下一步进入第三轮：自然说话机制。

---

## 1. 当前阶段判断

当前项目已经完成一轮全局提示语和儿童端文案优化，也完成了小白狐视觉中心化、状态丰富、后端语音并行生成、常见短句预热、安卓播放队列行为测试和播放日志增强。

当前真正影响孩子体验的核心问题变为：

```text
1. 小白狐已经更像主角，后续要通过真机观察继续微调；
2. 语音等待感已经做了一轮明显优化，但仍需真实设备继续观察；
3. 下一步可以开始轻量自然说话机制，让孩子更像在和小白狐轮流说话；
4. 之后再优化拍图分享体验和轻量记忆。
```

后续仍然坚持：只做孩子能直接感受到的改进，不再为了内部规划反复打转。

---

## 2. 暂时后置的方向

以下方向不是取消，而是不作为当前五轮主线：

```text
1. 学习脚手架深度优化：先后置。当前只保留“不直接给答案、引导审题和第一步”。
2. 合规和家长治理重设计：家庭内测阶段先不做重型治理，以后正式上线前再补。
3. 多孩子、多家长、正式账号体系：后置。
4. 长期成长档案和成长巢界面：后置，等主体验成立后再做。
5. 继续大范围提示语优化：暂停。除非真实使用中发现明显不好，再局部改。
6. 反复做完整功能测试：已有多轮测试，不再为了测试而测试；后续测试只围绕每轮实际改动验证。
```

---

## 3. 五轮收尾总目标

五轮内把产品从“功能已经可用”推进到“孩子明显更愿意看、愿意说、愿意拍给小白狐看”。

每轮必须能回答：

```text
当前是什么样？
这一轮后变成什么样？
孩子能直接感受到什么变化？
怎么验收不是原地打转？
是否已经提交并推送到远程主分支？
```

## 6. 第三轮：自然说话机制

状态：当前启动。

### 当前状态

当前儿童默认是语音优先，但仍以点击说话为主。第二轮已经降低了语音等待感，并补强了停止、静音、播放顺序等基础。现在可以开始做轻量自然说话机制。

### 优化后目标

孩子和小白狐的说话节奏更自然，但不能做永久监听，也不能长期开麦。

目标状态：

```text
1. 小白狐说完后，可以短时间等待孩子说话。
2. 孩子开始说话时，旧语音立即停止。
3. 没听清时温和提示重说。
4. 孩子不说话时自动收回等待，不一直监听。
5. 家长或调试设置可以关闭自动等待，退回点击说话。
6. 不保存不必要的原始音频。
7. 日志不记录孩子原文，只记录状态、时长、段序号和请求标识。
```

### 验收标准

```text
1. 孩子无需理解复杂按钮，也能自然说下一句。
2. 自动等待不会长时间占用麦克风。
3. 错误提示没有技术词。
4. 开关关闭后退回点击说话。
5. 覆盖安静环境、普通噪声、孩子停顿、孩子打断小白狐说话四类场景。
6. 最终报告必须包含远程主分支提交编号；没有远程提交编号视为未完成。
```

## 7. 第四轮：拍图分享体验

状态：等待第三轮完成后启动。

### 当前状态

拍图上传和图片理解已接入，但产品感还不够。当前重点不应是继续扩展识别类型，而是让“拍给小白狐看”变成孩子表达和创作入口。

### 优化后目标

孩子拍一张画、玩具、积木、书页、自然物，小白狐能看见一个安全具体的细节，并给孩子一个轻松的创作或表达入口。

目标状态：

```text
1. 拍图后本地图片确认卡更清楚、更像分享给小白狐。
2. 小白狐优先回应一个安全具体细节。
3. 提供轻量后续入口：起名字、讲故事、说发生了什么。
4. 只有明确是作业求助时才进入学习分支。
5. 看不清时承认看不清，不假装看见。
6. 不默认把图片当作作业检查或隐私风险。
```

---

## 8. 第五轮：轻量记忆和连续陪伴

状态：等待第三轮和第四轮稳定后启动。

### 当前状态

项目已有记忆和兴趣种子，但还没有把它打磨成孩子能感受到的“轻轻记得”。这件事可以做，但要克制，不追踪、不施压、不制造依赖。

### 优化后目标

小白狐偶尔记得孩子之前喜欢或展示过的小东西，并以很轻的方式接一下。孩子不接话时，小白狐马上放下。

目标状态：

```text
1. 小白狐偶尔提起孩子近期兴趣或作品。
2. 召回频率低，不连续追旧话题。
3. 孩子换题、短答、不想聊时立即尊重。
4. 睡前不召回兴奋话题。
5. 不说“我一直等你”“我最懂你”“只有我记得你”。
6. 家长日报可以使用宽泛线索，但儿童端不暴露敏感内容。
```

## 9. 多会话并行原则

可以并行，但必须按依赖关系来。

可并行：

```text
第一轮界面和小白狐形象：已完成。
第二轮语音延迟：已完成。
第四轮拍图分享：安卓图片体验会话 + 后端图片回应会话可并行。
```

当前不建议把第三轮拆太多并行。自然说话机制牵涉录音、播放、状态和开关，建议先用一个会话完成最小闭环。

## 11. 成功标准

五轮结束后，产品应该从“内部能力基本齐全”变成：

```text
孩子打开后先看到小白狐；
孩子愿意说话；
小白狐回应不慢到让孩子出戏；
孩子能自然拍图给小白狐看；
小白狐偶尔轻轻记得孩子的小兴趣；
整个体验没有积分、打卡、依赖、监控或过重治理感。
```
