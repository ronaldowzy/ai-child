# 孩子体验五轮收尾计划 v0.5

项目：`ai-child`

用途：把下一阶段从“继续扩展内部能力”收敛到“孩子能明显感受到更好玩、更顺、更像一个有形象的小白狐陪伴者”。

当前状态：第一轮界面和小白狐形象丰富、第二轮语音朗读延迟收敛、第三轮自然说话机制、第四轮拍图分享体验都已经完成并推送到远程主分支。下一步进入第五轮：轻量记忆和连续陪伴。

---

## 1. 当前阶段判断

当前项目已经完成一轮全局提示语和儿童端文案优化，也完成了小白狐视觉中心化、状态丰富、后端语音并行生成、常见短句预热、安卓播放队列行为测试、播放日志增强、轻量自然说话机制和拍图分享体验优化。

当前真正影响孩子体验的核心问题变为：

```text
1. 小白狐已经更像主角；
2. 语音等待感已做一轮收敛；
3. 小白狐说完后已经可以短时间等待孩子继续说；
4. 拍图分享已经更像表达和创作入口；
5. 下一步应让小白狐轻轻记得孩子近期兴趣或作品，但不追踪、不施压、不制造依赖。
```

后续仍然坚持：只做孩子能直接感受到的改进，不再为了内部规划反复打转。

---

## 2. 暂时后置的方向

以下方向不是取消，而是不作为当前五轮主线：

```text
1. 学习脚手架深度优化：先后置。当前只保留“不直接给答案、引导审题和第一步”。
2. 合规和家长治理重设计：家庭内测阶段先不做重型治理，以后正式上线前再补。
3. 多孩子、多家长、正式账号体系：后置。
4. 长期成长档案和成长巢界面：后置，等主体验成立后再做。
5. 继续大范围提示语优化：暂停。除非真实使用中发现明显不好，再局部改。
6. 反复做完整功能测试：已有多轮测试，不再为了测试而测试；后续测试只围绕每轮实际改动验证。
```

## 8. 第五轮：轻量记忆和连续陪伴

状态：当前启动。

### 当前状态

项目已有记忆和兴趣种子，但还没有把它打磨成孩子能感受到的“轻轻记得”。这件事可以做，但要克制，不追踪、不施压、不制造依赖。

当前已经具备的基础：

```text
1. 小白狐主视觉已经更突出；
2. 语音等待感已经做过收敛；
3. 小白狐说完可以短时间等孩子接话；
4. 拍图分享已经更像表达和创作入口；
5. 后端已经有记忆、兴趣和图片作品相关基础能力。
```

### 优化后目标

小白狐偶尔记得孩子之前喜欢或展示过的小东西，并以很轻的方式接一下。孩子不接话时，小白狐马上放下。

目标状态：

```text
1. 小白狐偶尔提起孩子近期兴趣或作品。
2. 召回频率低，不连续追旧话题。
3. 孩子换题、短答、不想聊时立即尊重。
4. 睡前不召回兴奋话题。
5. 不说“我一直等你”“我最懂你”“只有我记得你”。
6. 家长日报可以使用宽泛线索，但儿童端不暴露敏感内容。
7. 不做成长档案、成就、徽章、连续打卡。
```

### 本轮执行方式

第五轮不建议并行拆分。记忆召回、频率限制、上下文注入和测试彼此依赖，建议用一个干净会话完成最小闭环。
