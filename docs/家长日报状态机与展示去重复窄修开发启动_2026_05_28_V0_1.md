# 家长日报状态机与展示去重复窄修开发启动 v0.2

项目：`ai-child`

用途：作为“家长日报状态机与展示去重复窄修”的开发启动文档。本文用于指导开发端 B 修复家长日报在真机测试中暴露的生成中、失败、素材不足和展示重复问题。本文不涉及儿童端 Prompt Fast Path，不改儿童聊天链路。

更新时间：2026-05-28

---

## 0. 主控判断

家长日报是家长信任入口，不是儿童体验主线。真机测试中它暴露的问题是：

```text
1. 家长不知道日报是在生成、失败、没内容还是已完成；
2. 页面容易像空了或坏了；
3. 已生成内容存在重复摘要；
4. “暂无结构化观察”等工程化空状态伤害信任；
5. 家长日报容易滑向话术建议、教育建议或监控感。
```

本轮只做窄修：

```text
家长日报状态机与展示去重复。
```

不扩展复杂家长系统，不做成长档案，不做多孩子/多家长治理，不做家长话术建议系统。

---

## 1. 本轮目标

必须区分四种状态：

```text
1. 正在整理；
2. 已生成；
3. 今天内容还不够；
4. 整理失败，可重试。
```

家长端展示目标：

```text
1. 点击后马上知道正在整理；
2. 不盯着空页面等 90 秒；
3. 已生成日报不重复三遍摘要；
4. 素材不足和失败能区分；
5. 不展示 raw transcript；
6. 不提供“今晚可以这样接一句”或类似家长话术卡片；
7. 只给家长一个轻量、可信、不监控的当天小结。
```

---

## 2. 状态机口径

### 2.1 正在整理

触发：

```text
家长点击生成/查看今天小结，后端尚未返回最终日报。
```

可见文案：

```text
正在整理今天的小结。
```

规则：

```text
1. 0-1 秒内显示；
2. 3 秒内没有结果，继续保持生成中状态；
3. 允许家长离开；
4. 30 秒仍无结果，提示“今天的小结还没整理好，可以稍后再看”；
5. 不显示技术错误或空白页。
```

### 2.2 已生成

触发：

```text
日报生成成功，且至少有可展示内容。
```

已生成日报只展示轻量小结，不展示话术建议。

展示结构固定为：

```text
1. 今天一句话；
2. 孩子今天提到的内容；
3. 需要留意的地方。
```

### 2.3 今天内容还不够

触发：

```text
今天没有足够对话或可分析内容，不能生成有意义日报。
```

可见文案：

```text
今天聊得还不多，小结会短一点。
```

展示规则：

```text
1. 不伪造日报；
2. 不展示大片空卡片；
3. 可以提示稍后再看；
4. 不暗示孩子必须多聊；
5. 不展示 raw transcript。
```

### 2.4 整理失败，可重试

触发：

```text
接口失败、模型失败、超时、解析失败或后端无法返回可用日报。
```

可见文案：

```text
这次没有整理成功，可以再试一次。
```

规则：

```text
1. 显示重试入口；
2. 不显示技术词；
3. 不显示空报告；
4. 不把失败伪装成素材不足；
5. 不丢失家长当前页面上下文。
```

## 3. 展示区块设计定稿

开发端不得自行判断删哪些区块、保留哪些区块。本轮按以下设计执行。

### 3.1 删除 / 不再展示的区块

以下区块不再作为独立卡片展示：

```text
1. 今天聊了什么；
2. 今日整体摘要；
3. 日常聊天；
4. 今晚可以这样接一句；
5. 今晚可以这样聊；
6. 亲子沟通建议；
7. 建议话术；
8. 家长指导建议；
9. 暂无结构化观察；
10. 结构化观察；
11. 原始对话；
12. 对话全文；
13. raw transcript；
14. 无数据；
15. 请求错误 / 生成失败 / 接口超时 等工程失败卡片。
```

说明：

```text
“今天聊了什么 / 今日整体摘要 / 日常聊天”本质都在重复总结今天内容，家长看到会觉得页面冗余、像系统拼装出来的报告。

“今晚可以这样接一句 / 今晚可以这样聊 / 建议话术”等卡片不保留，容易显得不伦不类，也会让家长日报从轻量了解滑向教育指导或话术系统。

“暂无结构化观察”是工程化空状态，不应给家长看。

raw transcript、原始对话、对话全文都不展示，避免把家长日报变成监控工具。
```

### 3.2 保留 / 新增的区块

已生成日报只保留以下 3 个区块，按顺序展示：

```text
1. 今天一句话；
2. 孩子今天提到的内容；
3. 需要留意的地方。
```

顺序不能改。不要在三者之间插入“家长接话”“沟通建议”“亲子建议”等话术卡片。

### 3.3 区块 1：今天一句话

用途：给家长一个很快能理解的当天概览。

标题：

```text
今天一句话
```

内容：

```text
1 句自然语言摘要，建议 20-45 字。
```

示例风格：

```text
孩子今天主要聊了运动和一点点输赢感受，整体表达比较轻松。
```

禁止：

```text
1. 不写成诊断；
2. 不写成心理评估；
3. 不写“孩子情绪异常”等重判断；
4. 不复述孩子原话全文；
5. 不给家长话术建议。
```

### 3.4 区块 2：孩子今天提到的内容

主控决策：使用简单列表，不做 topic 卡片。

原因：

```text
1. topic 卡片会让日报显得像成长档案或兴趣画像；
2. topic 卡片容易诱导开发端加标签、分类、长期沉淀；
3. 当前只是日报小结，不是画像系统；
4. 简单列表更轻、更可信，也更符合当前窄修边界。
```

标题：

```text
孩子今天提到的内容
```

展示形式：

```text
简单项目列表，2-4 条。
```

每条格式：

```text
• 一个短点，12-28 字左右。
```

示例风格：

```text
• 和同学一起跳绳，玩了很多轮。
• 提到比赛输赢，但没有继续深聊。
• 分享了一个自己画的小东西。
```

禁止：

```text
1. 不做彩色 topic 卡片；
2. 不做“运动兴趣”“社交能力”“情绪标签”等标签；
3. 不做长期兴趣画像；
4. 不显示儿童原话全文；
5. 不把孩子内容做成可追踪档案。
```

### 3.5 区块 3：需要留意的地方

用途：只在有真实需要时提醒家长留意，不制造焦虑。

标题：

```text
需要留意的地方
```

展示条件：

```text
只有有内容时展示。
```

内容规则：

```text
1. 0-2 条；
2. 每条短句；
3. 只描述轻量观察；
4. 不做诊断；
5. 不做风险扩大；
6. 没有内容时整个区块不显示。
```

示例风格：

```text
• 今天有提到一点输赢感受，可以先观察，不需要追问太多。
```

禁止：

```text
1. 不展示“暂无结构化观察”；
2. 不展示空卡片；
3. 不写“无风险”“风险等级低”等工程化判断；
4. 不写成心理评估报告。
```

## 4. 家长端 master-copy

只能使用或贴近以下方向。

```text

今天聊得还不多，小结会短一点。
今天的小结还没整理好，可以稍后再看。
这次没有整理成功，可以再试一次。

需要留意的地方。
```

禁止：

```text
生成失败。
接口超时。
暂无结构化观察。
无数据。
请求错误。
模型返回为空。
解析失败。
raw transcript。
今晚可以这样接一句。
今晚可以这样聊。
亲子沟通建议。
建议话术。
家长指导建议。
```

如果必须新增家长端可见文案，先在计划中说明，不要自行发挥一整套新文案。

## 5. 设计边界

本轮可以做：

```text
1. 日报状态机；
2. 家长端日报展示去重复；
3. 失败/素材不足/生成中状态区分；
4. 重试入口；
5. 必要的后端字段适配；
6. 必要的 Android 家长端 UI 小改。
```

本轮不得做：

```text
1. 不做成长档案；
2. 不做多孩子、多家长治理；
3. 不做家长监控面板；
4. 不展示儿童原文全文；
5. 不改儿童端聊天 prompt；
6. 不改 Prompt Fast Path；
7. 不改儿童端小白狐界面；
8. 不新增日报订阅、提醒、打卡；
9. 不做周报/月报；
10. 不引入复杂数据图表；
11. 不做家长话术建议系统。
```

## 7. 测试要求

至少覆盖：

```text
1. 日报生成中状态显示“正在整理今天的小结”；
2. 生成超过阈值时显示“今天的小结还没整理好，可以稍后再看”；
3. 素材不足时显示“今天聊得还不多，小结会短一点”；
4. 后端失败时显示“这次没有整理成功，可以再试一次”；
5. 成功日报只展示“今天一句话 / 孩子今天提到的内容 / 需要留意的地方”；
6. “孩子今天提到的内容”使用简单列表，不使用 topic 卡片；
7. 没有留意内容时，不展示“需要留意的地方”；
8. 无内容时不展示“暂无结构化观察”大卡片；
9. 不展示 raw transcript；
10. 不出现禁止技术词和禁止话术建议词；
11. 不影响儿童端聊天接口。
```

## 8. 与开发进程 A 的边界

本任务是开发进程 B。

不得修改开发进程 A 的范围：

```text
1. 不改 PromptManager 的普通聊天 fast path；
2. 不改 child_chat prompt；
3. 不改 ChildAgentRuntime；
4. 不改 conversation.open 的输出契约；
5. 不改儿童聊天 latency trace；
6. 不改儿童端低压等待态。
```

如果你发现必须修改这些文件，先停止并向主控说明，不要直接改。

## 9. 执行流程要求

请先输出计划，不要直接改代码。

计划必须包含：

```text
1. 当前家长日报生成和展示实现在哪里；
2. 当前有哪些状态混淆；
3. 准备修改哪些文件；
4. 明确不会修改哪些文件；
5. 状态机如何设计；
6. 展示结构如何去重复；
7. 如何删除“今晚可以这样接一句 / 今晚可以这样聊”等接话区块；
8. “孩子今天提到的内容”如何改为简单列表；
9. 如何避免 raw transcript 和工程化空状态；
10. 测试策略；
11. 是否需要主控确认。
```

完成后输出交接摘要，必须包含：

```text
Summary:
- 做了什么。

Files changed:
- 修改了哪些文件。

Parent-visible change:
- 家长能感受到什么变化。

State machine:
- 四种状态如何区分。

Display:
- 删除了哪些区块。
- 保留了哪些区块。
- 孩子今天提到的内容是否为简单列表。
- 是否移除了今晚接话/家长话术区块。

Tests:
- 运行了哪些测试，结果如何。

Safety/privacy:
- 是否没有展示 raw transcript、儿童原文全文或隐私内容。

Known issues:
- 未完成事项或风险。

Commit:
- 远程主分支提交编号。
```
