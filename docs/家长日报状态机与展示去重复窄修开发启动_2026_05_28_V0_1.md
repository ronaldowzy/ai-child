# 家长日报状态机与展示去重复窄修开发启动 v0.1

项目：`ai-child`

用途：作为“家长日报状态机与展示去重复窄修”的开发启动文档。本文用于指导开发端 B 修复家长日报在真机测试中暴露的生成中、失败、素材不足和展示重复问题。本文不涉及儿童端 prompt fast path，不改儿童聊天链路。

更新时间：2026-05-28

---

## 0. 主控判断

家长日报是家长信任入口，不是儿童体验主线。真机测试中它暴露的问题是：

```text
1. 家长不知道日报是在生成、失败、没内容还是已完成；
2. 页面容易像空了或坏了；
3. 已生成内容存在重复摘要；
4. “暂无结构化观察”等工程化空状态伤害信任；
5. “今晚可以怎么接一句”价值点不够突出。
```

本轮只做窄修：

```text
家长日报状态机与展示去重复。
```

不扩展复杂家长系统，不做成长档案，不做多孩子/多家长治理。

---

## 1. 本轮目标

必须区分四种状态：

```text
1. 正在整理；
2. 已生成；
3. 今天内容还不够；
4. 整理失败，可重试。
```

家长端展示目标：

```text
1. 点击后马上知道正在整理；
2. 不盯着空页面等 90 秒；
3. 已生成日报不重复三遍摘要；
4. 素材不足和失败能区分；
5. 不展示 raw transcript；
6. “今晚可以这样接一句”成为最容易理解的价值点。
```

---

## 2. 状态机口径

### 2.1 正在整理

触发：

```text
家长点击生成/查看今天小结，后端尚未返回最终日报。
```

可见文案：

```text
正在整理今天的小结。
```

规则：

```text
1. 0-1 秒内显示；
2. 3 秒内没有结果，继续保持生成中状态；
3. 允许家长离开；
4. 30 秒仍无结果，提示“今天的小结还没整理好，可以稍后再看”；
5. 不显示技术错误或空白页。
```

### 2.2 已生成

触发：

```text
日报生成成功，且至少有可展示内容。
```

展示结构必须收敛为：

```text
1. 今天一句话；
2. 孩子今天提到的内容；
3. 今晚可以这样接一句；
4. 需要留意的地方：只有有内容时展示。
```

说明：

```text
“今晚可以这样接一句”应前置到家长容易看到的位置，不要埋在长报告后面。
```

### 2.3 今天内容还不够

触发：

```text
今天没有足够对话或可分析内容，不能生成有意义日报。
```

可见文案：

```text
今天聊得还不多，小结会短一点。
```

展示规则：

```text
1. 不伪造日报；
2. 不展示大片空卡片；
3. 可以给一个低压提示：晚点再看看；
4. 不暗示孩子必须多聊；
5. 不展示 raw transcript。
```

### 2.4 整理失败，可重试

触发：

```text
接口失败、模型失败、超时、解析失败或后端无法返回可用日报。
```

可见文案：

```text
这次没有整理成功，可以再试一次。
```

规则：

```text
1. 显示重试入口；
2. 不显示技术词；
3. 不显示空报告；
4. 不把失败伪装成素材不足；
5. 不丢失家长当前页面上下文。
```

---

## 3. 展示去重复要求

必须删除或合并重复结构。

当前不应再出现：

```text
1. “今天聊了什么”；
2. “今日整体摘要”；
3. “日常聊天”；
```

三层都在说同一件事的重复展示。

建议合并为：

```text
今天一句话：一句自然总结。
孩子今天提到的内容：2-4 个短点。
今晚可以这样接一句：1-2 条可直接对孩子说的话。
需要留意的地方：只有有真实内容时展示。
```

不得展示：

```text
1. 暂无结构化观察；
2. 无数据；
3. 请求错误；
4. 生成失败；
5. raw transcript；
6. 对话全文；
7. 工程字段名。
```

---

## 4. 家长端 master-copy

只能使用或贴近以下方向。

```text
正在整理今天的小结。
今天聊得还不多，小结会短一点。
今天的小结还没整理好，可以稍后再看。
这次没有整理成功，可以再试一次。
今晚可以这样接一句。
今天一句话。
孩子今天提到的内容。
需要留意的地方。
```

禁止：

```text
生成失败。
接口超时。
暂无结构化观察。
无数据。
请求错误。
模型返回为空。
解析失败。
raw transcript。
```

如果必须新增家长端可见文案，先在计划中说明，不要自行发挥一整套新文案。

## 5. 设计边界

本轮可以做：

```text
1. 日报状态机；
2. 家长端日报展示去重复；
3. 失败/素材不足/生成中状态区分；
4. 重试入口；
5. 必要的后端字段适配；
6. 必要的 Android 家长端 UI 小改。
```

本轮不得做：

```text
1. 不做成长档案；
2. 不做多孩子、多家长治理；
3. 不做家长监控面板；
4. 不展示儿童原文全文；
5. 不改儿童端聊天 prompt；
6. 不改 Prompt Fast Path；
7. 不改儿童端小白狐界面；
8. 不新增日报订阅、提醒、打卡；
9. 不做周报/月报；
10. 不引入复杂数据图表。
```

## 7. 测试要求

至少覆盖：

```text
1. 日报生成中状态显示“正在整理今天的小结”；
2. 生成超过阈值时显示“今天的小结还没整理好，可以稍后再看”；
3. 素材不足时显示“今天聊得还不多，小结会短一点”；
4. 后端失败时显示“这次没有整理成功，可以再试一次”；
5. 成功日报展示不重复三层摘要；
6. “今晚可以这样接一句”可见且靠前；
7. 无内容时不展示“暂无结构化观察”大卡片；
8. 不展示 raw transcript；
9. 不出现禁止技术词；
10. 不影响儿童端聊天接口。
```

---

## 8. 与开发进程 A 的边界

本任务是开发进程 B。

不得修改开发进程 A 的范围：

```text
1. 不改 PromptManager 的普通聊天 fast path；
2. 不改 child_chat prompt；
3. 不改 ChildAgentRuntime；
4. 不改 conversation.open 的输出契约；
5. 不改儿童聊天 latency trace；
6. 不改儿童端低压等待态。
```

如果你发现必须修改这些文件，先停止并向主控说明，不要直接改。

## 9. 执行流程要求

请先输出计划，不要直接改代码。

计划必须包含：

```text
1. 当前家长日报生成和展示实现在哪里；
2. 当前有哪些状态混淆；
3. 准备修改哪些文件；
4. 明确不会修改哪些文件；
5. 状态机如何设计；
6. 展示结构如何去重复；
7. 如何避免 raw transcript 和工程化空状态；
8. 测试策略；
9. 是否需要主控确认。
```

完成后输出交接摘要，必须包含：

```text
Summary:
- 做了什么。

Files changed:
- 修改了哪些文件。

Parent-visible change:
- 家长能感受到什么变化。

State machine:
- 四种状态如何区分。

Display:
- 日报展示结构如何收敛。

Tests:
- 运行了哪些测试，结果如何。

Safety/privacy:
- 是否没有展示 raw transcript、儿童原文全文或隐私内容。

Known issues:
- 未完成事项或风险。

Commit:
- 远程主分支提交编号。
```
