# 小展台小发现轻动静设计 v0.1

项目：`ai-child`

日期：2026-06-09

## 0. 主控判断

当前奇怪小门主线已经完成：

```text
奇怪小门事件 -> 三种机关 -> 拍照变身 -> 小展台保存 -> 小展台回看 -> 小展台旧物帮忙 -> 重玩/退出
```

下一步不应继续往奇怪小门里增加更多按钮、机关或规则，而应让小展台里的“小发现”在孩子再次进入 App 时产生一点轻微存在感。

核心目标：

```text
孩子以前放进小展台的小发现，不只是静静躺在列表里；它偶尔会在小屋里有一点小动静。
```

这能制造新的打开理由：

```text
我想看看小展台里的东西今天有没有动一下。
```

## 1. S6 名称

```text
S6：小发现轻动静
```

一句话体验：

```text
孩子进入小屋时，如果小展台里已有小发现，小白狐偶尔会说：有一个小发现好像轻轻动了一下。
```

## 2. 为什么做 S6

S5 让小展台物品能回到奇怪小门帮忙，但仍需要孩子主动选择。

S6 要解决的是“回来时的轻牵引”：

```text
孩子打开 App 后，除了看小门，也能感觉自己以前留下的东西在小屋里被小白狐认真看着。
```

心理感受：

```text
这个 App 记得我留下的小东西；
小白狐不是只在我拍照时才回应；
小屋里有一点点和我有关的小动静。
```

## 3. 产品边界

S6 不是：

```text
宠物饥饿
每日任务
签到提醒
道具刷新
奖励通知
背包事件
养成系统
```

S6 只是：

```text
小展台里一个已有小发现，被小白狐轻轻提到一次。
```

## 4. 触发时机

首版建议只在 App 内触发，不做系统通知。

触发条件：

```text
1. 当前 child 小展台中至少有 1 个 active item；
2. 当前不是奇怪小门强事件进行中；
3. 本次 App / ViewModel 生命周期内未展示过小发现轻动静；
4. 未处于睡前场景、学习场景、安全/隐私场景；
5. 普通 opening 可以被轻动静替代或插入。
```

首版可用本地内存标记控制“一次生命周期只展示一次”。

不做：

```text
按天持久化
每日必出
推送通知
红点未读
连续天数
```

## 5. 入口形态

### 5.1 首页轻提示

如果触发，小白狐气泡显示：

```text
小展台那边
好像有个小发现轻轻动了一下
```

按钮：

```text
去看看
先聊别的
```

### 5.2 点击“去看看”

进入一个轻查看状态，而不是直接打开完整列表。

显示选中的小发现：

```text
{name} 在小展台上待着呢
小白狐说：它刚才好像轻轻晃了一下
```

按钮：

```text
打开小展台
拿去帮小门
先聊别的
```

如果当前奇怪小门不可用或未激活，可以隐藏“拿去帮小门”。

### 5.3 点击“打开小展台”

进入 S2 小展台列表。

### 5.4 点击“拿去帮小门”

进入奇怪小门 PhotoPrompt 或直接复用 S5 旧物帮忙入口，具体实现计划阶段确认。

首版建议：

```text
如果当前已有 StrangeDoorDemo，复用 S5 useXiaozhantaiItemForStrangeDoor。
如果当前没有 StrangeDoorDemo，先激活奇怪小门，再使用该 item 帮忙。
```

如实现复杂，首版只保留“打开小展台 / 先聊别的”。

## 6. 选择哪一个小发现

首版不做复杂推荐。

建议 deterministic 选择：

```text
优先选择最近保存的 active item。
```

如果最近 item 图片缺失，仍可只显示名字和小白狐短句，不显示新增错误文案。

## 7. 儿童端文案

允许新增文案仅限：

```text
小展台那边
好像有个小发现轻轻动了一下
去看看
{name} 在小展台上待着呢
小白狐说：它刚才好像轻轻晃了一下
打开小展台
拿去帮小门
先聊别的
```

其中“先聊别的”已存在，可复用。

禁止新增：

```text
提醒
通知
任务
今日事件
未读
奖励
惊喜
领取
刷新
```

## 8. 与奇怪小门关系

S6 不改奇怪小门主链路。

可选承接：

```text
拿去帮小门
```

但它必须复用 S5 旧物帮忙能力，不能新增第二套旧物使用逻辑。

如果实现成本高，S6 首版可不做“拿去帮小门”，只做“打开小展台”。

## 9. 与小展台关系

S6 只读取小展台 item，不修改 item。

不新增：

```text
小发现状态
动了几次
小发现心情
小发现等级
小发现未读数
```

不写入：

```text
后端
GrowthEvent
本地历史
小展台 item
```

## 10. 家长端边界

S6 不改家长端。

家长端不展示“哪个小发现动了”。

原因：

```text
这是儿童端轻体验，不应变成家长监控项。
```

## 11. 实施建议

S6 先做计划，不直接编码。

计划必须回答：

```text
1. S6 是否放在 ChatViewModel opening 之前、之后，还是作为独立本地 opening；
2. 如何读取小展台最近 item；
3. 如何保证本生命周期只展示一次；
4. 如果奇怪小门当前自动激活，S6 和奇怪小门谁优先；
5. 是否做“拿去帮小门”；
6. 如果做，如何复用 S5；
7. 如果不做，如何只打开小展台；
8. 是否需要新增 UI state；
9. 是否改后端；
10. 是否新增 GrowthEvent；
11. 会修改哪些文件；
12. 不会修改哪些文件；
13. 测试策略；
14. 风险点；
15. 需要主控确认的问题。
```

## 12. 主控建议

首版建议保守：

```text
1. S6 只在当前没有 active StrangeDoorDemo 时触发；
2. 如果奇怪小门默认自动激活优先，则 S6 暂不抢首屏；
3. S6 可作为孩子退出小门或从小展台返回后的轻提示；
4. 首版可以只提供“打开小展台 / 先聊别的”；
5. “拿去帮小门”如果实现复杂，留到 S6-B。
```

原因：

```text
奇怪小门仍是当前最强事件；
S6 是关系感补充，不要抢首屏。
```

## 13. 验收标准

必须验证：

```text
1. 小展台为空时不会出现 S6；
2. 有小展台 item 时，轻动静不会抢奇怪小门首屏；
3. 文案像“轻轻被想起”，不是通知或任务；
4. 点击去看看能看到对应小发现；
5. 打开小展台能进入 S2 列表；
6. 不新增未读、红点、奖励、任务感；
7. 不写后端、不写 GrowthEvent、不改 item。
```
