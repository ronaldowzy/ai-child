# 小展台物品回到奇怪小门设计 v0.1

项目：`ai-child`

日期：2026-06-08

## 0. 主控判断

奇怪小门 Demo 已经完成：

```text
开场事件 -> 拍照变身 -> 三种机关 -> 门状态推进 -> 怪问题 -> 小展台保存 -> 小展台回看 -> 重玩/退出
```

下一步不应继续叠加门状态、关卡、奖励或地图，而应让孩子之前保存的小展台物品重新参与当前事件。

核心目标：

```text
孩子以前放进小展台的小发现，可以被拿出来帮小白狐开门。
```

这能让小展台从“回看空间”变成“孩子和小白狐共同留下的东西真的有用”。

## 1. S5 名称

```text
S5：用小展台的小发现帮忙
```

一句话体验：

```text
孩子不用每次都重新拍照，也可以从小展台里选一个以前留下的小发现，拿来试着帮小白狐打开奇怪小门。
```

## 2. 为什么要做 S5

当前小展台已经能保存和回看，但还偏静态。S5 要验证：

```text
孩子是否会因为“我以前留下的东西还能再帮小白狐”而更愿意保存、回看和重玩。
```

心理牵引：

```text
这不是普通照片列表；
这是我和小白狐一起留下的小东西；
它下次还能派上用场。
```

## 3. 入口设计

### 3.1 首屏不新增入口

首屏仍保持两个主入口：

```text
找东西帮忙
动脑试试
```

不在首屏增加第三个按钮，避免首屏复杂。

### 3.2 PhotoPrompt 增加弱入口

孩子点击“找东西帮忙”后，在现有按钮中增加：

```text
用小展台里的
```

PhotoPrompt 按钮顺序建议：

```text
拍给小白狐看
用小展台里的
先换个办法
```

含义：

```text
拍照仍是第一主路径；
小展台是复用路径；
怪问题是备用路径。
```

## 4. 小展台选择页

点击“用小展台里的”后进入一个选择页或复用 S2 列表选择模式。

标题：

```text
选一个小发现
```

空状态：

```text
小展台还空空的
我们先找一个东西拍给小白狐看
```

有展品时，每个卡片显示：

```text
图片缩略图
名字
保存时间
```

点击一个展品后，回到奇怪小门，并用它生成一次“旧物帮忙”的反馈。

## 5. 旧物帮忙反馈

反馈结构：

```text
{name} 又来帮忙啦

小白狐把它轻轻放到门前
{小门反应}
```

可用小门反应沿用 R1-A 已批准词池。

示例：

```text
小棉花又来帮忙啦

小白狐把它轻轻放到门前
门缝里冒出一点暖风
```

## 6. 门状态推进规则

使用小展台物品帮忙时：

```text
安全已保存展品 -> AdvanceOneStep
如果当前 doorState=Open -> 不应出现该入口，或先重置为 Closed 后再进入 PhotoPrompt，具体由 S3 现有逻辑控制
```

建议：

```text
1. doorState 非 Open 时，旧物帮忙推进一小步。
2. doorState Open 时，完成态不显示“用小展台里的”，只显示 S3 已确认按钮。
3. “再找一个”从 Open 重置到 Closed + PhotoPrompt 后，才可使用小展台入口。
```

## 7. 与三种机关的关系

旧物帮忙不做强匹配，不要求旧物一定符合圆圆 / 软软 / 亮亮。

原因：

```text
旧物是孩子已经保存过的小发现；
重点是“它又回来帮忙”，不是重新识别它是否符合机关。
```

但反馈要轻：

```text
它帮门动了一点点，而不是直接开门。
```

## 8. 与小展台详情的关系

S5 不改 S2 小展台详情页。

S5 可以复用列表 UI，但进入时是“选择模式”。

S2 普通入口：

```text
我的小展台
```

S5 选择入口：

```text
选一个小发现
```

两者行为不同：

```text
普通入口：看列表和详情
选择入口：点展品后回到奇怪小门并触发帮忙反馈
```

## 9. 数据边界

S5 不新增后端接口，不新增数据表，不新增 GrowthEvent。

只读取现有本地小展台 items：

```text
id
photoUri
name
foxQuote
createdAt
```

旧物帮忙时，不修改 item，不新增副本，不写入历史。

## 12. 实施建议

S5 应先做计划，不直接编码。

计划需要回答：

```text
1. 是否复用 S2 XiaozhantaiListScreen，还是新增选择模式页面；
2. 如何从 PhotoPrompt 打开选择模式；
3. 选择 item 后如何回到 StrangeDoorDemo；
4. 是否需要新增 StrangeDoorDemoState，例如 ShowcasePickPrompt / ShowcaseItemResult；
5. 旧物帮忙反馈如何进入 UI model；
6. 旧物帮忙如何推进门状态；
7. 空小展台时如何回到拍照路径；
8. 会修改哪些文件；
9. 不会修改哪些文件；
10. 测试策略；
11. 需要主控确认的问题。
```

## 13. 验收标准

S5 不是看“入口是否能点”，而是看孩子是否理解：

```text
我以前放进小展台的小东西，又回来帮小白狐了。
```

必须验证：

```text
1. 小展台为空时不会卡住；
2. 有展品时能选择；
3. 选择后能回到小门；
4. 小门状态推进一小步；
5. 文案不让孩子觉得这是装备、道具卡、奖励或背包；
6. 小展台仍保持温柔回看空间，不变成游戏物品库。
```
