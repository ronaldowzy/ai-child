## 2. 包体判断

开发端报告：

```text
v1 死资源移除：约 4.9MB；
10 状态 v2 资源：约 13MB；
APK：约 23.13MB；
全量 v2 10 状态在 10-15MB 资源目标区间附近。
```

主控判断：

```text
包体可以接受；不需要在阶段 2 继续压缩。
```

理由：

```text
1. 没有重演 100MB+ 包体事故；
2. v1 已移除；
3. 10 状态资源已落在可接受范围；
4. 继续为了几 MB 压缩可能牺牲角色质感，当前阶段不值得。
```

如后续正式发版前仍需压缩，再单独评估：

```text
1. 降低 WebP quality；
2. 个别低频状态降帧；
3. 低频状态静态化；
4. 裁剪透明边界。
```

## 3. 已知注意点

### 3.1 真机仍未验证

当前仍未完成 Android 真机播放/切换/降级验证。

处理方式：

```text
不阻塞阶段 3，但阶段 3 完成后必须真机复测。
```

真机复测重点：

```text
1. 10 状态播放是否流畅；
2. 48 帧资源是否导致低端设备卡顿；
3. 状态切换是否闪烁；
4. 静态首帧 fallback 是否正常；
5. 小白狐尺寸是否合适；
6. 状态气泡是否遮挡小白狐；
7. APK 运行时内存是否异常。
```

### 3.2 `paused` 优先级需观察

当前 `MascotStatePriority` 与 v2 manifest 中，`paused` 排在 `listening/speaking` 前面。

主控判断：

```text
不作为阶段 2 阻塞项；阶段 3 不要顺手改状态优先级。
```

但后续真机需要观察：

```text
安全/隐私/睡前场景是否会把正在朗读或真实录音的小白狐过早压成 paused。
```

如出现真实冲突，再单独修状态优先级。不要在阶段 3 视觉舞台任务里顺手改业务状态机。

### 3.3 预存测试失败仍独立处理

主控判断：
