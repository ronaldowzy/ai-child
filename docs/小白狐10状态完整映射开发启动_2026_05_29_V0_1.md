## 3. 转换方案

继续复用阶段 1 脚本：

```text
python3 scripts/convert_states_v2.py --states <all 10>
```

或等价命令。

要求：

```text
1. 继续输出 512px WebP 帧序列；
2. 保持 fps=12；
3. 保持 frameCount=48，除非包体超预算；
4. 保持每状态 manifest；
5. 更新 v2 mascot_manifest.json；
6. 不提交 1024px PNG 原始帧；
7. 不把源素材目录整包打入 APK。
```

如包体超过预算，优先考虑：

```text
1. 降低 WebP quality；
2. 个别低频状态降帧；
3. 裁剪透明边界；
4. 低频状态使用静态或 24 帧版本。
```

但调整前必须先报告，不允许自行牺牲画质或删除状态。

## 4. 10 状态映射规则

### 4.1 统一映射

按以下产品含义映射：

```text
idle：默认陪伴；
waiting_soft：说完后的低压等待，未录音；
listening：真实录音中；
thinking：模型回复生成中；
preparing_speech：文字已出，等待首段语音或准备朗读；
speaking：TTS 正在播放；
image_viewing：图片已被后端收到，小白狐正在看；
co_create：轻共创状态；
paused：停一下、放下、暂停；
retry：没听清、上传失败、图片失败、可重试。
```

### 4.2 禁止混淆

```text
waiting_soft 不能表示真实录音；
listening 才能表示真实录音；
retry 不能变成红色错误警告；
image_viewing 只有图片已进入后端处理或确认收到后才显示；
图片本地选中但未上传成功，不得显示“已经给小白狐看啦”；
co_create 不能在图片失败时出现；
paused 不能做成委屈、求陪或宠物饥饿状态。
```

## 5. 状态优先级建议

如果多个状态同时存在，建议优先级：

```text
retry / 图片失败 / 没听清
> listening / 真实录音
> speaking / 正在朗读
> preparing_speech / 准备说
> thinking / 正在想
> image_viewing / 正在看图
> co_create / 轻共创
> paused / 停一下
> waiting_soft / 低压等待
> idle / 默认陪伴
```

开发端可根据当前状态机实际实现调整，但必须说明调整原因，不能自行改变产品含义。
