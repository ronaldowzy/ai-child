# 小白狐 v1 到 v2 状态映射确认 v0.1

项目：`ai-child`

用途：确认阶段 2“小白狐 10 状态完整映射”中，旧 v1 状态到新 v2 陪伴状态的映射口径。本文用于回答开发端关于 `HomeworkFocus`、`Sleepy`、`PrivacyBoundary`、`SafetyConcern`、`JumpingHappy` 等旧状态去向的问题。

更新时间：2026-05-29

---

## 0. 主控结论

v2 的 10 个状态不是 v1 的 10 个状态的文件级一一替换。

阶段 2 必须按产品语义映射：

```text
v1 可以删除，是因为当前资源根路径已指向 v2 且 v1 无代码引用；
不是因为 v1 每个状态都能机械替换成 v2 同名状态。
```

本次确认的核心原则：

```text
1. v2 不保留高刺激、警告感、宠物感、老师感的旧状态；
2. 安全/隐私/睡前/学习等旧状态不再做强表情，而是折叠到更温和的陪伴状态；
3. 真实录音、低压等待、准备说、看图、轻共创等新状态按当前产品状态机驱动；
4. 开发端不得自行新增 v2 状态或重新解释小白狐状态含义。
```

## 1. 旧状态到 v2 的确认映射

| 旧 v1 状态 | v2 映射 | 说明 |
|---|---|---|
| `Idle` / `idle` | `idle` | 默认陪伴。 |
| `Listening` / `listening` | `listening` | 只表示真实录音中。不得用于低压等待。 |
| `Speaking` / `speaking` | `speaking` | TTS 正在播放。 |
| `Thinking` / `thinking` | `thinking` | 模型生成中、小白狐在想。 |
| `NetworkError` / `network_error` | `retry` | 没听清、没传好、网络/图片失败等可重试状态。不得做红色警告。 |
| `HomeworkFocus` / `homework_focus` | `thinking` | 学习/拍题处理中默认映射为“正在想”，避免老师感。若当前明确是图片已被后端收到并在看图，可由图片状态优先映射到 `image_viewing`。 |
| `Calm` / `calm` | `idle` | 安静陪伴即可，不单独保留。若当前是“停一下/收住”，由业务状态映射到 `paused`。 |
| `Sleepy` / `sleepy` | `paused` | 睡前/放下/收住，不做困倦宠物状态，不暗示小白狐疲惫或需要陪伴。 |
| `PrivacyBoundary` / `privacy_boundary` | `paused` | 隐私边界应温和收住，不做警告、不做 retry，不鼓励孩子重复输入隐私内容。 |
| `SafetyConcern` / `safety_concern` | `paused` | 安全关切应低刺激收住，不做红色警告、不做恐慌表情。 |
| `JumpingHappy` / `jumping_happy` | `co_create`，但仅作为温和正向/灵感态 | 不保留跳跃兴奋感。不作为奖励、任务完成或连续使用反馈。若当前只是普通正向回应且没有共创意味，也可以由更高层状态回到 `idle` 或 `speaking`。 |

## 2. v2 新状态的来源

以下 v2 状态不是从 v1 机械替换而来，应由当前产品状态机驱动：

| v2 状态 | 来源 |
|---|---|
| `waiting_soft` | 小白狐说完后的低压等待态。未录音。 |
| `preparing_speech` | 文字已出现，首段语音尚未开始或小白狐准备朗读。 |
| `image_viewing` | 图片已被后端收到，正在给小白狐看/小白狐正在看。 |
| `co_create` | 明确轻共创状态，例如作品起名、讲一句小故事、故事接龙的轻灵感态。 |
| `paused` | 停一下、放下、睡前收住、安全/隐私边界收住。 |
| `retry` | 没听清、上传失败、图片失败、网络失败、可重试。 |

## 3. 五个开发端特别询问状态的确认

### 3.1 `HomeworkFocus`

确认映射：

```text
HomeworkFocus -> thinking
```

理由：

```text
1. 当前不希望小白狐变成老师或学习平台；
2. 拍题/学习处理中，本质是“小白狐在想”；
3. 真正看图时应由图片状态映射到 image_viewing，而不是 HomeworkFocus 自己决定。
```

### 3.2 `Sleepy`

确认映射：

```text
Sleepy -> paused
```

理由：

```text
1. 不做困倦宠物状态；
2. 不让孩子觉得小白狐累了、需要孩子照顾；
3. 睡前或放下时，最适合用 paused 表达“收住、安静、可以停”。
```

### 3.3 `PrivacyBoundary`

确认映射：

```text
PrivacyBoundary -> paused
```

理由：

```text
1. 隐私边界不是错误，也不是失败；
2. 不应该用 retry 鼓励孩子再说一次隐私内容；
3. paused 能表达温和收住和边界。
```

### 3.4 `SafetyConcern`

确认映射：

```text
SafetyConcern -> paused
```

理由：

```text
1. 不做红色警告、不制造紧张；
2. 安全关切需要低刺激、稳定、收住；
3. 具体安全文本由后端安全策略负责，视觉只表达“我们先停一下/稳住”。
```

### 3.5 `JumpingHappy`

确认映射：

```text
JumpingHappy -> co_create
```

但必须加限制：

```text
1. 只能作为温和正向/灵感态；
2. 不表示奖励；
3. 不表示任务完成；
4. 不表示连续使用反馈；
5. 不做高兴跳跃感。
```

如果现有旧逻辑把普通鼓励、普通开心、普通回复都映射到 `JumpingHappy`，开发端应谨慎处理：

```text
普通回复播放中优先由 speaking 覆盖；
普通默认状态回 idle；
只有明确轻共创、作品分享成功后的温和灵感态才使用 co_create。
```

## 4. 状态优先级补充

阶段 2 建议优先级仍为：

```text
retry / 图片失败 / 没听清
> listening / 真实录音
> speaking / 正在朗读
> preparing_speech / 准备说
> thinking / 正在想
> image_viewing / 正在看图
> co_create / 轻共创
> paused / 停一下/边界收住
> waiting_soft / 低压等待
> idle / 默认陪伴
```

补充说明：

```text
1. 安全/隐私进入 paused，但如果正在真实录音，listening 仍可短暂显示；
2. 低压等待永远不能显示 listening；
3. 图片失败优先 retry；
4. 图片后端已收到优先 image_viewing；
5. TTS 播放优先 speaking；
6. co_create 不覆盖 retry，也不覆盖真实录音和正在朗读。
```

## 5. 阶段 2 实施要求

开发端应按本文修正阶段 2 计划或实现。

必须做：

```text
1. 在交接摘要中列出 v1 -> v2 最终映射表；
2. 明确 `HomeworkFocus/Sleepy/PrivacyBoundary/SafetyConcern/JumpingHappy` 的映射已按本文执行；
3. 删除 v1 死资源；
4. 接入 v2 10 状态；
5. 保持业务逻辑不变。
```

不得做：

```text
1. 不新增 v2 状态；
2. 不恢复 v1 高刺激状态；
3. 不把 `Sleepy` 做成宠物困倦；
4. 不把 `SafetyConcern` 做成红色警告；
5. 不把 `PrivacyBoundary` 做成 retry；
6. 不把 `JumpingHappy` 做成奖励动画；
7. 不把 `HomeworkFocus` 做成老师/课堂感。
```
