## 4. 已知问题与处理方式

### 4.1 v1 死资源

当前问题：

```text
v1 目录约 4.9MB，仍保留在 APK 中，但已无代码引用。
```

主控决策：

```text
阶段 2 的第一步必须移除 v1 死资源。
```

原因：

```text
1. 回收包体；
2. 避免 v1+v2 共存导致包体评估失真；
3. 避免后续开发误用旧资源；
4. 为 10 状态完整映射清理资源根路径。
```

删除前必须确认：

```text
1. AssetManifestLoader.DEFAULT_ROOT_PATH 已指向 v2；
2. 没有代码直接引用 mascot/xiaobaohu/v1；
3. 没有测试依赖 v1；
4. v2 至少能 fallback 到 idle；
5. 删除后 APK 能正常构建。
```

### 4.2 帧数差异

当前事实：

```text
v2 每状态 48 帧；
v1 每状态 24 帧；
v2 每状态约 1.5MB；
v1 每状态约 0.4MB。
```

主控判断：

```text
48 帧可以暂时保留，因为三状态试点增量在预算内，全量预估也在目标区间内。
```

但阶段 2 仍需继续输出包体报告。若全量后超过预算，再考虑：

```text
1. 降帧到 24 或 36；
2. 降 WebP quality；
3. 对部分低频状态用静态首帧或较少帧；
4. 统一裁剪透明边界。
```

### 4.3 无真机验证

当前事实：

```text
阶段 1 无 Android 真机连接，播放/切换/降级尚未真机复测。
```

主控判断：

```text
不阻塞阶段 2，但阶段 2 仍不能声称真机通过。
```

阶段 2 完成后必须保留：

```text
真机复测待办。
```
