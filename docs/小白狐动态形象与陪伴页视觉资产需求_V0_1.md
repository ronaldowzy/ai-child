# 小白狐动态形象与陪伴页视觉资产需求 v0.1

项目：`ai-child`

用途：作为“小白狐动态形象与陪伴页视觉资产升级”的主控设计文档。本文用于指导产品负责人新开 ChatGPT / image-2 会话生成小白狐状态素材，并用于后续开发端接入。本文不是开发实现方案，不要求当前立即改代码。

更新时间：2026-05-28

---

## 0. 主控结论

第 4 轮 App 界面重设计已经完成结构层落地。下一步若要更接近 image-2 生成的陪伴页质感，关键不是继续让开发端凭想象调布局，而是补齐视觉资产：

```text
1. 小白狐状态图；
2. 轻量背景素材；
3. 状态光效与卡片样式参考；
4. Android 状态到素材的映射规则。
```

当前先做素材生成，不直接接入开发。晚上真机测试后，再决定是否启动素材接入开发任务。

---

## 1. 视觉资产原则

### 1.1 必须坚持

```text
1. 小白狐是儿童页第一视觉；
2. 小白狐温和、好奇、柔软、低压；
3. 不像老师、客服、机器人、任务发布者；
4. 不像宠物饥饿或宠物养成系统；
5. 不制造依赖感、委屈感或索取陪伴感；
6. 不做高刺激表情或夸张动作；
7. 状态变化要让孩子看懂：听、想、说、看图、等待、停一下、没听清。
```

### 1.2 不能出现

```text
1. 积分、徽章、打卡、排行、任务；
2. 宠物饥饿、哭闹、委屈求陪；
3. 老师指挥感、客服坐席感、机器人机械感；
4. 复杂道具和文字；
5. 真实人物、学校、家庭环境隐私；
6. 高刺激霓虹色；
7. 背景复杂到影响 App 接入。
```

---

## 2. 资产生成方式

当前阶段推荐：

```text
透明背景小白狐状态图 + Android 轻量 Compose 动效
```

不推荐当前做：

```text
1. 实时 3D 模型；
2. 骨骼动画系统；
3. 大规模逐帧动画；
4. 完整 3D 房间背景；
5. 复杂游戏化动态资源。
```

Android 后续可通过以下方式产生动态感：

```text
1. 轻微呼吸缩放；
2. 轻微上下浮动；
3. 状态光晕；
4. 眨眼帧或局部替换；
5. 状态图切换。
```

## 5. 输出规格

### 5.1 图片规格

```text
格式：PNG 或 WebP；
背景：透明背景优先；
尺寸：1024×1024 或 1536×1536；
构图：小白狐居中，完整身体或稳定半身，四周留安全边距；
方向：正面或轻微 3/4 角度；
光照：柔和；
风格：与 App 界面方向图中的小白狐一致。
```

如果 image-2 不能稳定输出透明背景：

```text
使用纯色浅暖背景；
背景不要有复杂物件；
后续再由工具抠图处理。
```

### 5.2 一致性要求

必须尽量保持：

```text
1. 耳朵比例一致；
2. 眼睛大小和位置一致；
3. 脸型一致；
4. 毛色一致；
5. 身体比例一致；
6. 角色年龄感一致；
7. 不同状态像同一只小白狐。
```

筛掉：

```text
1. 耳朵突然变长或变短；
2. 眼睛风格变化太大；
3. 毛色偏黄、偏灰或过白发光；
4. 姿势太复杂，小屏看不清；
5. 表情太兴奋、太委屈、太伤心；
6. 太像宠物养成、老师、客服或机器人。
```

## 一、项目背景

这是一个面向 5-10 岁儿童的 AI 成长陪伴 App，角色是“小白狐”。产品不是游戏，不是学习平台，不是宠物养成，而是温和、低压、安全的小白狐陪伴入口。

小白狐需要看起来：

奶白色、柔软、圆润、干净；
眼睛大而温和；
表情好奇、友好；
像会轻轻听孩子说话的小伙伴；
不兴奋过头，不委屈，不像老师，不像客服，不像机器人。

---

## 二、整体风格

请保持和我上传的 App 界面图一致：

3D 可爱小白狐；
柔软毛发；
低刺激、温暖、陪伴感；
适合儿童 App；
角色比例统一；
耳朵、眼睛、脸型、毛色、身体比例尽量一致。

请生成透明背景 PNG 风格素材。小白狐主体居中，四周留出安全边距，适合后续放进 App 界面。

如果无法输出透明背景，请使用纯浅暖色背景，不要复杂场景，方便后续抠图。

## 9. 后续开发接入边界

素材通过主控评审后，开发端只负责：

```text
1. 放入 Android 资源目录；
2. 按现有状态映射替换小白狐状态图；
3. 加轻量呼吸、浮动、光晕或眨眼动效；
4. 保持第 4 轮已落地的界面结构；
5. 不新增功能、不新增页面、不改后端主逻辑。
```

开发端不得自行：

```text
1. 重新设计小白狐形象；
2. 增加新状态；
3. 新增宠物养成表现；
4. 加入任务、奖励、连续回来等视觉；
5. 用未经主控评审的图片替换产品资产。
```
