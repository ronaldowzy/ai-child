# 小白狐拍图状态与横屏优化开发启动 v0.1

项目：`ai-child`

用途：作为“小白狐动态陪伴页 UI 升级”的阶段 5 开发启动文档。本文用于指导开发端在阶段 4 完成后，只做拍图状态表达和横屏/平板轻量优化。本文不是图片业务链路重做，不允许新增作品库、故事库或复杂图片功能。

更新时间：2026-05-29

---

## 0. 主控结论

阶段 4 已完成，可以进入：

```text
阶段 5：拍图状态和横屏优化
```

阶段 5 的目标是：

```text
1. 让拍图后状态更像“给小白狐看看”，不是文件上传；
2. 明确区分图片本地已选中、上传中、后端已收到、失败；
3. 成功后只保留一个低压共创入口；
4. 失败时不出现共创入口；
5. 横屏/平板下保持小白狐舞台稳定，不让消息区变回聊天窗口。
```

本阶段仍然不是完整 UI 大改，也不是图片功能重做。

---

## 1. 本阶段只做什么

阶段 5 只做：

```text
1. 拍图状态表达优化；
2. 图片预览卡视觉轻量化；
3. 图片失败态可见但不吓人；
4. 成功后轻共创入口收敛为一个；
5. 横屏/平板布局轻量优化；
6. 真机复测清单整理。
```

---

## 2. 本阶段不做什么

不得做：

```text
1. 不改图片上传业务链路；
2. 不改图片识别/拍题后端；
3. 不改 Prompt Fast Path；
4. 不改家长日报；
5. 不改小白狐 10 状态资源；
6. 不改低压等待态；
7. 不新增作品库；
8. 不新增故事库；
9. 不新增复杂图片编辑；
10. 不新增相册管理；
11. 不新增奖励、任务、打卡；
12. 不修 unrelated audioReadyUsesQueueWhenNotMuted；
13. 不清理 dead code，除非影响编译。
```

## 3. 拍图状态设计

### 3.1 必须区分的状态

拍图后至少区分：

```text
1. 本地已选中；
2. 正在给小白狐看；
3. 小白狐已经看到 / 正在看；
4. 这张图还没给小白狐看到；
5. 重试 / 先不看。
```

### 3.2 状态文案口径

可用或贴近以下短句：

```text
正在给小白狐看看。
小白狐正在看。
这张图还没给小白狐看到。
我们可以再试一次。
先不看也可以。
```

禁止：

```text
1. 上传文件；
2. 图片上传失败；
3. 接口错误；
4. 识别失败；
5. 系统异常；
6. 已经给小白狐看啦，除非后端已确认收到；
7. 快给小白狐看看；
8. 今天再拍一张；
9. 完成任务。
```

---

## 4. 图片成功状态

当图片后端确认收到或正在处理时，可以显示：

```text
小白狐正在看。
```

可以触发：

```text
MascotState.ImageViewing
```

成功后如需要露出轻共创入口，只允许一个：

```text
起个名字
```

或：

```text
讲一句小故事
```

二选一，不得两个同时出现。

建议优先级：

```text
普通作品/画作：起个名字；
明显故事性图片：讲一句小故事；
不确定时：起个名字。
```

禁止：

```text
1. 同时展示“起个名字”和“讲一句小故事”；
2. 成功后自动进入故事系统；
3. 保存到作品库；
4. 生成长故事；
5. 做任务奖励。
```

---

## 5. 图片失败状态

失败时必须明确：

```text
这张图还没给小白狐看到。
```

按钮：

```text
我们可以再试一次
先不看也可以
```

失败时不得显示：

```text
1. 起个名字；
2. 讲一句小故事；
3. 小白狐正在看；
4. 已经给小白狐看啦；
5. image_viewing 状态；
6. co_create 状态。
```

失败视觉：

```text
1. 不用红色警告；
2. 不用错误图标吓人；
3. 可使用 MascotState.Retry；
4. 保持低压。
```

---

## 6. 图片预览卡

图片预览卡目标：

```text
让家长和孩子知道“这张图正在给小白狐看”；
但不要变成作品库、相册或文件管理器。
```

建议：

```text
1. 图片缩略图小而清楚；
2. 圆角浅色卡片；
3. 不占据小白狐主视觉；
4. 不显示文件名、文件大小、技术路径；
5. 不显示上传进度百分比；
6. 失败时卡片弱化，不强警告。
```

---

## 7. 横屏/平板优化

阶段 5 横屏目标：

```text
小白狐舞台稳定；
右侧消息区轻量；
图片预览不压缩小白狐；
底部操作仍清楚；
不要变回聊天工作台。
```

建议：

```text
1. 左侧/主区域保持小白狐舞台；
2. 右侧只显示最近 2 条消息和轻量图片预览；
3. 图片预览卡不超过右侧主要高度的一小部分；
4. 语音主按钮仍明显；
5. 拍图入口仍在次级位置；
6. 不新增完整作品面板。
```

如果当前横屏已经基本稳定，只做轻量修正，不要重构。

---

## 8. 与小白狐状态的关系

状态映射必须遵守：

```text
图片本地已选中但未上传/未确认：不显示 image_viewing；
上传中或给小白狐看中：可显示 thinking 或 preparing_speech，视现有状态而定；
后端已收到并处理：image_viewing；
失败：retry；
成功后的轻共创：co_create，但仅在明确共创入口出现或触发时。
```

禁止：

```text
1. 图片失败显示 image_viewing；
2. 图片失败显示 co_create；
3. 本地刚选中就说“小白狐已经看到”；
4. 把图片状态做成任务流程；
5. 把图片状态做成作品沉淀。
```

## 9. 真机复测清单

阶段 5 完成后，应输出整体真机复测清单。

至少包括：

```text
1. 打开 App，小白狐是否是第一视觉；
2. 10 状态动图是否能正常切换；
3. 低压等待按钮是否仍是“按一下开始说”；
4. 录音态是否才显示“说完了”；
5. 首段语音是否不被后续段拖住；
6. 停一下是否能停止迟到语音；
7. 拍图成功是否显示小白狐正在看；
8. 拍图失败是否显示“这张图还没给小白狐看到”；
9. 拍图失败是否没有共创入口；
10. 横屏是否不挤压小白狐；
11. paused 是否过早压掉 listening/speaking；
12. 动图是否卡顿；
13. APK 包体和运行内存是否可接受。
```

## 11. 测试与验收

必须至少验证：

```text

2. 图片失败态不会显示成功文案；
```

