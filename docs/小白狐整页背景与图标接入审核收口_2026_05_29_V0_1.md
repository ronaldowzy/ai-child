## 2. 视觉判断

### 2.1 整页背景

当前使用 Compose 渐变和柔光实现：

```text
顶部浅蓝；
中部奶白；
底部浅暖黄；
局部柔光；
无大背景图。
```

主控判断：

```text
符合“参考效果图但不照搬效果图”的要求。
```

通过点：

```text
1. 没有把 image-2 整张图当背景；
2. 没有新增复杂 3D 房间；
3. 没有新增大体积背景图；
4. 没有引入窗户、书架、地毯、吊灯等复杂场景；
5. 页面整体更像陪伴页。
```

### 2.2 消息区和底部操作区

当前消息区与底部操作区的调整符合本轮目标：

```text
1. 气泡更轻；
2. 卡片更柔；
3. 底部不再像硬工具栏；
4. 语音主按钮仍保持第一操作；
5. 拍图入口仍保持“给小白狐看看”的自然分享感。
```

注意：本轮修改了 `InputBar` 的视觉样式，但没有改变低压等待、真实录音、拍图入口等业务逻辑。该调整可以接受。

### 2.3 图标

当前已接入 launcher icon 和 round icon。

主控判断：

```text
图标方向符合当前产品“温暖陪伴、小白狐主视觉”的定位。
```

真机需要重点看：

```text
1. 桌面小尺寸是否清晰；
2. 圆形 icon 是否裁耳朵；
3. 方形 icon 是否有异常白边或黑边；
4. 下方爪子和云朵是否过度贴边；
5. 是否与新版儿童页视觉一致。
```

## 5. 晚上真机测试结论口径

晚上测试后，请按以下三类归档问题：

### 5.1 必须返修

```text
1. App 无法启动；
2. 图标严重裁切或显示异常；
3. 小白狐动图卡死或明显闪烁；
4. 低压等待又变成自动录音；
5. 录音态/等待态按钮文案混乱；
6. 拍图失败仍显示成功或共创入口；
7. prompt 延迟问题明显回退；
8. 家长日报又出现接话/话术建议。
```

### 5.2 可小修

```text
1. 背景稍微太亮或太淡；
2. 某些气泡文字可读性一般；
3. 横屏局部拥挤；
4. 图片卡稍大或稍小；
5. 图标小尺寸略糊但可识别；
6. 状态气泡位置略挡。
```

### 5.3 暂不处理

```text
1. 想继续追求更像效果图的复杂房间；
2. 想新增云朵/星星/窗户等装饰；
3. 想新增故事库或作品库；
4. 想做新首页、新启动页、新引导页；
5. 想继续打磨不影响使用的微小视觉差异。
```
