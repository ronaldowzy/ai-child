# 小白狐文字气泡与顶部入口返修 v0.1

项目：`ai-child`

用途：根据当前竖屏/横屏截图，对儿童主界面的文字呈现、状态气泡、顶部家长入口和冗余按钮做一次返修。本文只指导 UI 表达层，不改业务状态机、不改 prompt、不改家长日报、不改小白狐资源。

更新时间：2026-05-29

---

## 0. 主控结论

当前背景方向已经变好，但文字呈现仍不统一。主要问题：

```text
1. 右上角显示“大人”，应改为“家长入口”；
2. 顶部“大人长按进入。”太像正文标题，应移除或弱化；
3. 小白狐下方同时出现“小白狐在这里。”和“不说也没关系。”，像散落标签；
4. “我想说话”小按钮与大语音按钮重复；
5. 效果图中的文字应更像轻量气泡，而不是平铺文字。
```

本轮目标：

```text
把页面文字从“散落标签”改成“顶部家长入口 + 单一动态状态气泡 + 轻量消息卡”。
```

---

## 1. 必须修改

```text
1. 右上角入口改为“家长入口”；
2. 不再显示“大人”单字入口；
3. 不再大字显示“大人长按进入。”；
4. 移除“我想说话”小按钮；
5. 小白狐附近同一时间只显示一个状态气泡；
6. 状态气泡按 MascotState 动态切换；
7. speaking 状态不显示状态气泡，避免和回复内容重复；
8. 状态气泡不进入聊天记录。
```

---

## 2. 状态气泡文案

| 状态 | 气泡文案 |
|---|---|
| idle | 我在这里。 |
| waiting_soft | 想说的时候再说。 |
| listening | 我在听。 |
| thinking | 我想一想。 |
| preparing_speech | 我准备说。 |
| speaking | 不显示气泡。 |
| image_viewing | 我正在看。 |
| co_create | 我们一起想想。 |
| paused | 好，我们先停一下。 |
| retry | 这次没弄好，可以再试一次。 |

说明：

```text
1. idle 不再写“小白狐在这里。”；
2. waiting_soft 不再额外显示“不说也没关系。”；
3. listening 用更短的“我在听。”；
4. retry 不出现错误、接口、失败等技术词。
```

---

## 3. 气泡视觉和动效

气泡要求：

```text
1. 浅白或奶白半透明；
2. 大圆角；
3. 轻阴影或柔和边框；
4. 靠近小白狐下方或右下方；
5. 不贴在舞台底部；
6. 不遮挡小白狐脸、耳朵、爪子；
7. 不像按钮、不像 Toast、不像页面脚注。
```

动态要求：

```text
1. 状态变化时轻微 fade/slide/scale；
2. 建议时长 180-260ms；
3. 不做弹跳、奖励、闪烁；
4. 不同时叠两个气泡。
```

---

## 4. 消息卡关系

```text
状态气泡：表达当前状态，不进入聊天记录；
消息卡：只展示真实对话内容。
```

禁止：

```text
1. 空消息时额外显示一张消息卡；
2. 横屏右侧重复显示状态气泡内容；
3. 把等待态文案写进消息记录；
4. 小白狐附近同时显示两个状态提示。
```

---

## 5. 横屏和竖屏要求

横屏：

```text
1. 小白狐状态气泡放在左侧舞台内；
2. 右侧只保留最近真实消息和操作按钮；
3. 右上角为“家长入口”；
4. 移除“我想说话”小按钮。
```

竖屏：

```text
1. 顶部不要大字显示“大人长按进入。”；
2. 右上角显示“家长入口”；
3. 小白狐下方只保留一个气泡；
4. 不再把“不说也没关系。”作为舞台底部脚注；
5. 消息卡不要从底部露出半截。
```
