## 2. 底部操作区判断

开发端未修改 `InputBar.kt`，理由是当前优先级已正确：

```text
语音 > 拍图 > 换话题 > 停一下 > 静音
```

主控判断：

```text
可以接受。
```

原因：

```text
1. 语音主按钮仍是第一操作；
2. 给小白狐看看仍是第二入口；
3. 低压等待态仍显示“按一下开始说”；
4. 真实录音态才显示“说完了”；
5. 当前不需要为了“重排”而强行改已稳定的 InputBar 逻辑。
```

## 5. 是否进入阶段 5

允许进入：

```text
阶段 5：拍图状态和横屏优化。
```

阶段 5 目标是：

```text
1. 让拍图后状态更像“给小白狐看看”，不是文件上传；
2. 区分本地已选中、上传中、已给小白狐看到、失败；
3. 成功后只保留一个低压共创入口；
4. 失败时不出现共创入口；
5. 横屏/平板下保持小白狐舞台稳定，不让消息区变回聊天窗口。
```

阶段 5 不得做：

```text
1. 不改图片上传业务链路；
2. 不改 prompt；
3. 不改家长日报；
4. 不新增作品库；
5. 不新增故事库；
6. 不新增复杂图片编辑；
7. 不新增奖励、任务或打卡；
8. 不修 unrelated audioReadyUsesQueueWhenNotMuted。
```
