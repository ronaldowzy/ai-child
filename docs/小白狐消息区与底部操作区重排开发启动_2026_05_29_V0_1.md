# 小白狐消息区与底部操作区重排开发启动 v0.1

项目：`ai-child`

用途：作为“小白狐动态陪伴页 UI 升级”的阶段 4 开发启动文档。本文用于指导开发端在陪伴舞台完成后，收敛消息区视觉层级，并按孩子真实操作优先级重排底部操作区。本文不是功能扩展，不允许改业务状态机、prompt、家长日报或拍图业务链路。

更新时间：2026-05-29

---

## 0. 主控结论

阶段 3 已完成，可以进入：

```text
阶段 4：消息区和底部操作区重排
```

阶段 4 的目标是：

```text
让小白狐继续保持第一视觉中心；
让消息区退到第二视觉层级；
让底部操作区按孩子最自然的动作排序；
进一步靠近 image-2 效果图中的陪伴页结构。
```

本阶段仍然不是完整 UI 大改。

---

## 1. 本阶段只做什么

阶段 4 只做：

```text
1. 消息区视觉降级；
2. 最近回复/最近输入展示收敛；
3. 底部操作区优先级重排；
4. 语音主按钮视觉强化；
5. 拍图入口作为第二操作入口；
6. 停一下/换个话题作为低压辅助入口；
7. 文字输入弱化但保留；
8. 竖屏和横屏的轻量布局适配。
```

---

## 2. 本阶段不做什么

不得做：

```text
1. 不改业务状态机；
2. 不改 Prompt Fast Path；
3. 不改家长日报；
4. 不改小白狐资源和 10 状态映射；
5. 不做拍图面板重设计；
6. 不做作品库/故事库；
7. 不新增任务、奖励、打卡、排行榜、宠物养成；
8. 不新增页面；
9. 不恢复自动录音式自然接话；
10. 不改低压等待口径；
11. 不修 unrelated audioReadyUsesQueueWhenNotMuted；
12. 不清理 dead code，除非它影响编译。
```

## 3. 消息区设计

### 3.1 目标

消息区不再像普通聊天软件一样抢占主视觉。

目标：

```text
小白狐是第一视觉；
消息只是小白狐和孩子刚刚说过的话；
孩子不需要看到密密麻麻的聊天记录；
界面更像陪伴页，不像客服窗口或 ChatGPT 聊天页。
```

### 3.2 竖屏展示规则

竖屏建议：

```text
1. 只突出最近 1 条小白狐回复；
2. 可保留最近 1 条孩子输入，但视觉弱于小白狐回复；
3. 更早消息不在主区域强展示；
4. 不做复杂历史抽屉；
5. 空消息时不显示空列表，只保留小白狐舞台和状态气泡。
```

### 3.3 横屏/平板展示规则

横屏/平板建议：

```text
1. 可显示最近 2-3 条消息；
2. 不把右侧做成完整聊天窗口；
3. 小白狐舞台仍保持稳定宽度和视觉存在感；
4. 消息区不压缩小白狐。
```

### 3.4 消息视觉规则

建议：

```text
1. 小白狐回复卡片：浅色、圆角、轻阴影或无阴影；
2. 孩子输入卡片：更轻、更小、更弱；
3. 更早消息：不突出，必要时弱化；
4. 不显示技术状态消息；
5. 不把状态气泡混进聊天记录。
```

禁止：

```text
1. 不做长聊天列表作为主视觉；
2. 不显示大量历史气泡；
3. 不新增“查看全部聊天记录”主入口；
4. 不把状态气泡写进消息记录；
5. 不用 Markdown 列表风格展示小白狐回复。
```

## 4. 底部操作区设计

### 4.1 操作优先级

底部操作区必须按以下优先级组织：

```text
第一：语音主按钮；
第二：给小白狐看看；
第三：停一下 / 换个话题；
第四：文字输入。
```

### 4.2 语音主按钮

语音主按钮是儿童页第一操作。

要求：

```text
1. 居中或最显眼；
2. 面积最大；
3. 文案清晰；
4. 低压等待态显示“按一下开始说”；
5. 真实录音态显示“说完了”；
6. 不因等待态误显示“说完了”；
7. 不被拍图或文字输入抢视觉。
```

### 4.3 拍图入口

拍图入口是第二操作。

文案继续使用或贴近：

```text
给小白狐看看
```

要求：

```text
1. 不写“上传文件”；
2. 不抢过语音主按钮；
3. 保持自然分享感；
4. 不在本阶段改拍图状态机。
```

### 4.4 停一下 / 换个话题

停一下和换个话题属于低压辅助入口。

要求：

```text
1. 不做成任务按钮；
2. 不放得比语音主按钮更显眼；
3. 保持轻量；
4. 停一下必须仍能停止迟到语音或当前播放；
5. 换个话题不诱导孩子继续聊。
```

### 4.5 文字输入

文字输入保留，但不作为第一视觉。

要求：

```text
1. 弱化为次级入口；
2. 不删除；
3. 不做复杂输入框扩展；
4. 不影响语音和拍图主路径。
```

---

## 5. 与低压等待态的关系

阶段 4 必须继续遵守：

```text
低压等待态不自动录音；
WaitingForChild 不等于 recording；
Listening 才是真实录音；
等待态按钮是“按一下开始说”；
录音态按钮才是“说完了”。
```

禁止因为底部操作区重排而重新引入：

```text
1. 自动开麦；
2. 常开麦克风；
3. 环境声触发录音；
4. 等待态显示“说完了”；
5. 孩子不说时催促孩子。
```

## 6. 布局建议

### 6.1 竖屏

建议结构：

```text
上/中部：小白狐陪伴舞台，占第一视觉；
中下部：最近回复/最近输入，轻量展示；
底部：操作区，语音主按钮最突出。
```

不要让消息区把小白狐压到很小。

### 6.2 横屏/平板

建议结构：

```text
左侧或主区域：小白狐陪伴舞台；
右侧：最近消息 + 操作区；
语音按钮仍突出；
消息区不变成完整聊天窗口。
```

如果当前横屏结构已经接近，不要大改。
