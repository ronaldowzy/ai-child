# 小白狐状态素材脚本动效方案 v0.1

项目：`ai-child`

用途：作为新版小白狐状态素材的脚本动效生成方案。本文用于指导开发端或 Codex 基于已生成的小白狐静态状态图，复用上一批“Pillow 局部绘制 + 呼吸帧 + 眨眼帧”的轻量方案，生成可接入 Android 的小白狐动态帧。本文不是新功能设计，不改变产品方向，不要求重新生图。

更新时间：2026-05-28

---

## 0. 主控结论

新版小白狐素材不应通过 image-2 继续生成闭眼、半闭眼、口型等完整状态图。

更稳的方式是：

```text
以同一张清理后的状态 PNG 为源图；
用脚本生成呼吸帧；
在指定局部区域绘制眼皮、闭眼弧线、口型或轻微局部变化；
导出帧序列或轻量资源包。
```

这样可以避免：

```text
1. 每一帧小白狐脸型变化；
2. 耳朵、眼睛、毛色不一致；
3. 生图造成角色跳变；
4. 动画看起来像换了一只狐狸；
5. 美术成本失控。
```

---

## 1. 旧方案要点

用户确认上一批小白狐眨眼动画不是由生图方重新生成闭眼图，而是由 Codex 用脚本加工透明 PNG 实现。

旧方案关键点：

```text
1. 输入源图，例如 fox_idle_clean.png；
2. 脚本读取原图后生成呼吸动画帧；
3. 到眨眼帧时，只在眼睛区域局部覆盖和绘制；
4. 第 10 帧：半闭；
5. 第 11 帧：闭眼；
6. 第 12 帧：半闭；
7. 身体、围巾、背包、尾巴、脸型保持不变；
8. 眼睛区域通过配置里的 eyeBoxes 固定坐标定位；
9. 用接近脸部毛色的 eyelidFill 覆盖眼睛区域；
10. 用深灰/黑色 lineColor 画闭眼弧线；
11. manifest 记录 features.blink: true 和生成参数。
```

可参考旧文件名：

```text
make_state_loop.py
apply_blink(...)
draw_blink_eye(...)
idle.json
manifest.json
```

说明：当前主分支未确认存在这些旧文件；如果开发端本地或资源包中有，应优先复用。如果没有，可以按本文复刻最小版。

## 2. 新版素材包候选

第一版新版小白狐静态状态素材建议收敛为以下 10 个：

```text
xiaobaohu_idle.png
xiaobaohu_waiting_soft.png
xiaobaohu_listening.png
xiaobaohu_thinking.png
xiaobaohu_preparing_speech.png
xiaobaohu_speaking.png
xiaobaohu_image_viewing.png
xiaobaohu_co_create.png
xiaobaohu_paused.png
xiaobaohu_retry.png
```

其中：

```text
idle：使用最后生成的默认陪伴图；
image_viewing：使用最后生成的低头看图图；
waiting_soft：使用上一批较像等待的 listening 图；
listening：使用新生成的 listening_v2；
thinking：使用 thinking_v2；
preparing_speech / speaking / co_create / paused / retry：使用第二批通过的 6 张图。
```

当前这些图多为纯浅暖色背景，不一定是真透明。正式生成动效前，应先统一抠图和裁切，得到 clean PNG。

## 3. 动效范围

### 3.1 第一版必须做

```text
1. 通用呼吸帧；
2. 通用眨眼帧；
3. 每个状态循环 24 帧左右；
4. manifest 记录素材来源、状态名、帧数、是否启用 blink/breathe。
```

### 3.2 第一版可以尝试

```text
1. speaking 状态做 2-3 帧轻口型；
2. listening 状态做耳朵轻动帧；
3. image_viewing 状态做轻微低头/呼吸；
4. co_create 状态做轻微抬爪或光晕，但不做游戏化闪光。
```

### 3.3 第一版不要做

```text
1. 复杂骨骼动画；
2. 小白狐跑动、跳跃、大幅摆尾；
3. 星星爆炸、奖励闪光；
4. 宠物饥饿、哭泣、委屈求陪；
5. 逐帧重绘整只小白狐；
6. 用 image-2 为每一帧重新生图；
7. 引入新功能、新页面、新玩法。
```

## 4. 技术处理建议

### 4.1 输入目录

建议：

```text
assets/source/xiaobaohu_states/
  xiaobaohu_idle.png
  xiaobaohu_waiting_soft.png
  xiaobaohu_listening.png
  xiaobaohu_thinking.png
  xiaobaohu_preparing_speech.png
  xiaobaohu_speaking.png
  xiaobaohu_image_viewing.png
  xiaobaohu_co_create.png
  xiaobaohu_paused.png
  xiaobaohu_retry.png
```

如果原图为浅暖背景，不是真透明，应先输出：

```text
assets/processed/xiaobaohu_states_clean/
```

### 4.2 输出目录

建议：

```text
assets/generated/xiaobaohu_state_loops/
  idle/frame_000.png
  idle/frame_001.png
  ...
  listening/frame_000.png
  ...
  manifest.json
```

后续开发端可再决定是否转成 WebP、animated WebP 或 Android drawable 资源。

### 4.3 配置文件

建议每个状态一个配置：

```text
assets/config/xiaobaohu_states/idle.json
assets/config/xiaobaohu_states/listening.json
...
```

配置至少包含：

```json
{
  "state": "idle",
  "source": "xiaobaohu_idle.png",
  "frames": 24,
  "breathe": {
    "enabled": true,
    "scaleMin": 0.995,
    "scaleMax": 1.008,
    "offsetY": 2
  },
  "blink": {
    "enabled": true,
    "frames": [10, 11, 12],
    "closureByFrame": {
      "10": 0.55,
      "11": 1.0,
      "12": 0.55
    },
    "eyeBoxes": {
      "left": [0, 0, 0, 0],
      "right": [0, 0, 0, 0]
    },
    "eyelidFill": "#F6E4D4",
    "lineColor": "#5A4538",
    "lineWidth": 4
  }
}
```

`eyeBoxes` 必须由开发端根据 clean PNG 实际坐标调，不允许主观猜测后不验证。

## 5. 各状态动效建议

| 状态 | 呼吸 | 眨眼 | 额外尝试 | 禁止 |
|---|---|---|---|---|
| idle | 是 | 是 | 无 | 不做兴奋欢迎 |
| waiting_soft | 是 | 是 | 呼吸更慢 | 不做催促感 |
| listening | 是 | 是 | 耳朵轻动 | 不做监听设备感 |
| thinking | 是 | 是 | 托腮状态保持稳定 | 不做加载转圈 |
| preparing_speech | 是 | 是 | 轻微抬头或光晕 | 不做焦虑等待 |
| speaking | 是 | 可选 | 2-3 帧轻口型 | 不做演讲/老师感 |
| image_viewing | 是 | 可选 | 轻微低头/光晕 | 不做扫描线/放大镜 |
| co_create | 是 | 是 | 抬爪轻动或柔光 | 不做奖励闪光 |
| paused | 慢呼吸 | 少量 | 状态收住 | 不做委屈 |
| retry | 是 | 是 | 轻微歪头感 | 不做错误警告 |

## 6. 验收标准

### 6.1 素材验收

```text
1. 每个状态第一帧与源图一致；
2. 呼吸动画不造成边缘抖动明显；
3. 眨眼只影响眼睛区域，不改变脸型、耳朵、身体、尾巴；
4. 闭眼弧线自然，不像生气或困倦；
5. 口型只在 speaking 状态启用，且不夸张；
6. 透明背景或抠图边缘干净；
7. 小尺寸下状态仍可辨识。
```

### 6.2 产品验收

```text
1. 动态让小白狐更像“活着”，但不游戏化；
2. 小白狐不会像宠物在索取陪伴；
3. 小白狐不会像老师、客服或任务发布者；
4. 状态变化服务听、想、说、看图、等待、停一下；
5. 不新增任何任务、奖励、连续使用压力。
```

## 7. 开发端边界

开发端或 Codex 可以做：

```text
1. 查找旧脚本和配置；
2. 复用或复刻 make_state_loop.py；
3. 编写背景抠图脚本；
4. 为每张图手动或半自动标注 eyeBoxes；
5. 生成每个状态的 24 帧循环；
6. 输出 manifest.json；
7. 生成预览 GIF 或 MP4 供主控审核。
```

开发端不得做：

```text
1. 自行重设小白狐形象；
2. 自行新增状态；
3. 自行新增儿童端文案；
4. 自行新增玩法、任务、奖励、作品库、故事库；
5. 把失败、等待、共创做成高刺激动画；
6. 未经主控审核直接接入 App。
```

---

## 8. 推荐下一步

下一步应让 Codex 执行“素材脚本动效试制”，而不是直接接入 App。

试制目标：

```text
1. 先完成 idle 一个状态的 clean PNG、呼吸、眨眼、manifest 和预览；
2. 主控确认效果；
3. 再批量处理其余状态；
4. 素材包通过后，再另开开发接入任务。
```

不要一上来把 10 个状态全部接入 Android。先试制、看效果、再批量。
