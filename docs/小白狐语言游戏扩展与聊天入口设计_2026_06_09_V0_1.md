# 小白狐语言游戏扩展与聊天入口设计 v0.1

项目：`ai-child`

日期：2026-06-09

## 0. 主控重新判断

上一版语言游戏中，“接一句故事”“三个怪答案”偏轻共创，适合陪伴，但重复玩动力不够强。儿童可能玩一两次后新鲜感下降。

下一版语言游戏应更接近：

```text
聊天过程中，小白狐可以问孩子：
我们随便聊聊天，还是玩一个小游戏？
```

游戏要有清楚规则、短回合、可重复玩，但不能变成学习测验、考试、刷题或竞赛。

核心原则：

```text
1. 像玩，不像学；
2. 有规则，但不压迫；
3. 有挑战，但不惩罚；
4. 有结束，但不通关；
5. 可以语音玩，也可以点选玩；
6. 5-10 岁分层，但首版不要复杂分龄；
7. 小白狐要像陪孩子玩，不像老师考孩子。
```

## 1. 聊天中游戏入口

### 1.1 入口时机

小白狐不应一进来就强推游戏。建议在以下场景出现：

```text
1. 普通 opening 后；
2. 孩子连续短句或不知道聊什么时；
3. 孩子主动说“玩游戏”“脑筋急转弯”“接龙”“猜谜语”；
4. 奇怪小门结束并点击“先聊别的”后；
5. 家长模式不触发。
```

### 1.2 入口话术

```text
我们随便聊聊天
还是玩一个小游戏？
```

按钮：

```text
随便聊聊
玩个小游戏
```

如果点“玩个小游戏”，进入小游戏选择。

### 1.3 小游戏选择页 / 气泡

标题：

```text
想玩哪一个？
```

首批游戏按钮：

```text
脑筋急转弯
词语接龙
成语接龙
猜谜语
乱词造句
```

底部：

```text
先聊别的
```

### 1.4 入口边界

```text
不叫学习
不叫练习
不叫挑战
不叫闯关
不出现分数
不出现等级
不出现奖励
不出现连续答对
不出现排行榜
```

## 2. 游戏一：脑筋急转弯

### 2.1 一句话定义

```text
小白狐问一个轻松、不吓人、没有学科压力的脑筋急转弯，孩子可以语音回答或点提示。
```

### 2.2 适合原因

脑筋急转弯有明确规则和意外答案，重复玩动力比“接一句故事”强。

儿童吸引点：

```text
我想看看小白狐这次会问什么怪问题。
```

### 2.3 首版题库示例

题 1：

```text
什么东西越洗越脏？
答案：水
提示：它常常在杯子里、河里、盆里
```

题 2：

```text
什么门永远关不上？
答案：球门
提示：它常常在操场上
```

题 3：

```text
什么东西越走越少？
答案：路
提示：你走过以后，它就被你走掉了一点点
```

题 4：

```text
什么瓜不能吃？
答案：傻瓜
提示：它不是一种真的瓜
```

题 5：

```text
什么布剪不断？
答案：瀑布
提示：它不是用来做衣服的布
```

### 2.4 回合结构

```text
出题 -> 孩子回答 -> 判断接近/答对/没答出 -> 小白狐解释 -> 下一题/换游戏/先聊别的
```

### 2.5 答对反馈

```text
对，就是{answer}
这个答案有点绕
小白狐刚才也差点没绕出来
```

### 2.6 答错反馈

```text
这个答案也有点意思
不过这题的小机关不是它
我给你一个提示
{hint}
```

### 2.7 放弃 / 不知道

```text
我偷偷告诉你
答案是{answer}
是不是有点拐弯？
```

按钮：

```text
下一题
换个游戏
先聊别的
```

### 2.8 禁止

```text
不说答错了
不说不对
不说你不会
不做连续答对
不做积分
题目不能恐怖、羞辱、成人、危险
```

## 3. 游戏二：词语接龙

### 3.1 一句话定义

```text
小白狐说一个词，孩子接一个词，只要后一个词的开头字和前一个词的结尾字一样就可以。
```

### 3.2 为什么优先于成语接龙

词语接龙比成语接龙更适合 5-10 岁，低龄孩子不会因为不会成语而受挫。

### 3.3 入口说明

```text
我们玩词语接龙
我先说一个词
你接一个从“{lastChar}”开始的词就行
```

示例：

```text
小白狐：苹果
孩子：果汁
小白狐：汁水
孩子：水杯
```

### 3.4 判断规则

首版本地规则：

```text
1. 提取小白狐上一个词最后一个字；
2. 判断孩子回答第一个有效汉字是否相同；
3. 相同：通过；
4. 不同：温柔提示；
5. 孩子说不出来：小白狐帮接一个。
```

### 3.5 通过反馈

```text
接上啦
“{previous}”接“{childWord}”
这个小词跑得还挺快
```

### 3.6 未接上反馈

```text
这个词也可以玩
不过这次要从“{lastChar}”开始
我给你换个容易的
```

### 3.7 防卡住机制

连续 2 次没接上，小白狐自动降难度：

```text
我们换一个简单的
我说：小猫
你接一个“猫”开头的词
```

### 3.8 回合数

首版最多 5 轮。

结束语：

```text
我们已经接了好多小词
先让它们排队休息一下
```

按钮：

```text
再玩一次
换个游戏
先聊别的
```

### 3.9 禁止

```text
不判分
不倒计时
不淘汰
不说失败
不要求必须是课本词
不要求必须成语
```

## 4. 游戏三：成语接龙

### 4.1 一句话定义

```text
适合年龄稍大的孩子，小白狐说一个简单成语，孩子接一个开头字相同的成语。
```

### 4.2 主控判断

成语接龙有吸引力，但难度高。首版不能作为默认语言游戏，应作为“更难一点”的可选项。

### 4.3 入口话术

```text
这个有点难一点
我们玩轻轻版成语接龙
不会也没关系，我可以给提示
```

### 4.4 首版规则

为了避免挫败，首版不要求严格成语库全覆盖。

规则：

```text
1. 优先用白名单成语；
2. 孩子回答如果在白名单中且首字匹配，则通过；
3. 如果不是白名单，但看起来是 4 个汉字，先温柔接住；
4. 不强行纠错；
5. 小白狐可以给 2 个候选让孩子选。
```

### 4.5 示例白名单

```text
一心一意
意想不到
到处都是
是是非非
非同小可
可可爱爱
爱不释手
手舞足蹈
```

注意：其中“可可爱爱”不是传统成语，但可作为儿童轻接龙词。UI 中不要强调“标准成语”。

### 4.6 更稳的命名

不建议直接叫“成语接龙”，建议叫：

```text
四字接龙
```

因为这样可以容纳儿童化四字词，降低挫败。

按钮展示可为：

```text
四字接龙
```

而不是“成语接龙”。

### 4.7 禁止

```text
不纠正成语标准性
不说这个不是成语
不要求背诵
不解释典故
不做语文学习
```

## 5. 游戏四：猜谜语

### 5.1 一句话定义

```text
小白狐说一个儿童安全谜面，孩子猜一个生活物品或自然物。
```

### 5.2 和脑筋急转弯的区别

脑筋急转弯靠“拐弯”；猜谜语靠“描述特征”。

### 5.3 首版谜题

谜题 1：

```text
小小房子圆又圆
里面住着甜甜水
答案：橘子
```

谜题 2：

```text
白白一片天上走
有时像羊有时像狗
答案：云
```

谜题 3：

```text
有头没有脚
天天背着小朋友跑
答案：书包
```

谜题 4：

```text
一根小棍黑又尖
走到纸上留下线
答案：铅笔
```

### 5.4 提示机制

```text
提示一：它常常在哪里
提示二：它有什么颜色/形状
直接告诉答案
```

按钮：

```text
给我提示
我猜到了
换一题
```

### 5.5 禁止

```text
不猜身体隐私
不猜危险物品
不猜成人概念
不猜学校/住址/姓名
```

## 6. 游戏五：乱词造句

### 6.1 一句话定义

```text
小白狐给三个词，孩子随便造一句奇怪的话。
```

### 6.2 示例

小白狐：

```text
给你三个小词：
月亮、袜子、咕噜
你把它们放进一句话里
```

孩子：

```text
月亮穿着袜子咕噜咕噜滚走了
```

小白狐：

```text
这句话有点圆滚滚的
小白狐听见月亮在地板上咕噜了一下
```

### 6.3 吸引点

比“接一句故事”更有规则，也更好笑。

### 6.4 词组示例

```text
月亮 / 袜子 / 咕噜
杯子 / 云朵 / 慢吞吞
小车 / 枕头 / 打哈欠
铅笔 / 星星 / 蹦一下
```

### 6.5 禁止

```text
不评价语法
不要求完整句
不纠错
不出现危险词
不让孩子造隐私句
```

## 7. 游戏六：反着说

### 7.1 一句话定义

```text
小白狐说一句短句，孩子把前后东西调换，得到一个怪结果。
```

### 7.2 示例

```text
小白狐：小狐狸钻进杯子里
孩子：杯子钻进小狐狸里
小白狐：那小狐狸走路会咣当咣当地响
```

### 7.3 主控判断

这个有趣，但理解门槛高。适合 7-10 岁，不适合首版默认。

---

## 8. 游戏七：谁最像

### 8.1 一句话定义

```text
小白狐给三个选项，孩子选哪个最像某个东西，然后小白狐接一个想象反馈。
```

### 8.2 示例

```text
小白狐：
你觉得小白狐的尾巴最像什么？
一朵云
一条围巾
一把扫把
我自己说
```

孩子选：

```text
一朵云
```

小白狐：

```text
那我走路要轻一点
不然尾巴云会飘走
```

### 8.3 优点

低龄友好，点击即可玩。

### 8.4 缺点

规则感弱，重复玩动力一般。

## 9. 推荐优先级

### 第一批，最值得做

```text
1. 脑筋急转弯
2. 词语接龙
3. 猜谜语
```

原因：

```text
规则明确；
孩子熟悉；
重复玩动力强；
适合聊天过程中自然发起；
不需要大量新视觉资源。
```

### 第二批

```text
4. 乱词造句
5. 四字接龙
```

原因：

```text
更语言化；
更适合大一点的孩子；
需要更多规则和安全兜底。
```

### 暂缓

```text
反着说
谁最像
小白狐听错了
```

## 10. 聊天中游戏模式建议

### 10.1 普通聊天入口

当孩子没有明确目标时，小白狐可以说：

```text
我们随便聊聊天
还是玩一个小游戏？
```

按钮：

```text
随便聊聊
玩个小游戏
```

### 10.2 游戏选择

```text
想玩哪一个？
```

按钮第一批：

```text
脑筋急转弯
词语接龙
猜谜语
先聊别的
```

第二批再加：

```text
乱词造句
四字接龙
```

### 10.3 游戏中退出

每个游戏都必须有：

```text
换个游戏
先聊别的
```

不要把孩子困在游戏里。

### 10.4 游戏结束

```text
这个小游戏先玩到这里
我们可以换一个
也可以随便聊聊
```

按钮：

```text
换个游戏
随便聊聊
再玩一次
```

## 11. 技术实现边界建议

首版不要接后端。

建议先做 Android 本地状态：

```text
LanguageGameState
LanguageGameType
LanguageGameRound
LanguageGameResult
```

优先实现：

```text
脑筋急转弯
词语接龙
猜谜语
```

后端只保留普通 conversation，不承担游戏判断。

如果未来需要更丰富题库，再考虑后端题库服务。

## 12. 安全边界

游戏输入必须过滤：

```text
真实姓名
学校名
住址

证件
家庭隐私
医疗隐私
暴力伤害
惊吓恐怖
成人内容
危险行为
```

```text
考试
答错
错误
正确答案
分数
闯关
通关
任务
等级
奖励
金币
排名
PK

默写
纠错
```

## 13. 下一步建议

不要立刻开发所有游戏。

建议先做：

```text
LG0：聊天中游戏入口与游戏选择计划
```

LG0 只规划入口和状态，不实现游戏。

然后：

```text
LG1：脑筋急转弯 Demo
LG2：词语接龙 Demo
LG3：猜谜语 Demo
```

如果资源有限，优先：

```text
LG1 脑筋急转弯
```

原因：

```text
儿童熟悉；
规则简单；
有惊喜感；
实现成本低；
能最快验证语言游戏是否成立。
```
