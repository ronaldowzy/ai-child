# 小白狐语言类游戏方向设计 v0.1

项目：`ai-child`

日期：2026-06-09

## 0. 主控判断

当前“奇怪小门”主线已经比较完整，但继续叠加小门机制的边际收益下降。下一阶段可以探索“语言类小游戏”，但必须避免做成学习 App、识字练习、考试问答或刷题。

语言类游戏的产品目标不是教语文，而是让孩子觉得：

```text
我说一句，小白狐就能接住、变好玩、变成一个小故事或小机关。
```

适合 ai-child 的语言小游戏应满足：

```text
1. 一轮 30 秒到 2 分钟；
2. 孩子可以语音回答；
3. 答得奇怪也能被接住；
4. 不强调对错；
5. 不出现分数、等级、奖励；
6. 可以和小白狐关系感、奇怪小屋、小展台联动；
7. 文案和判断规则必须由主控设计，不能交给开发方自由生成。
```

---

## 1. 方向一：接一句故事

### 1.1 一句话定义

```text
小白狐说半句故事，孩子接一句，故事就往奇怪方向走一点。
```

### 1.2 为什么适合

这是最轻的语言共创。孩子不用答对，只要说一句就能推进。

吸引点：

```text
我说什么，故事就会变成什么。
```

### 1.3 示例体验

小白狐：

```text
我刚刚在地毯下面
发现了一只会打哈欠的小盒子
它一张嘴，里面竟然有……
```

按钮 / 语音入口：

```text
我来接一句
换一个开头
先聊别的
```

孩子说：

```text
里面有一只小青蛙
```

小白狐回应：

```text
一只小青蛙跳了出来
它没有呱呱叫
它说：我是在找一只迷路的袜子
```

然后给孩子选择：

```text
再接一句
放进小展台
先聊别的
```

### 1.4 MVP 规则

首版只做一轮或两轮，不做长篇故事。

```text
开头 -> 孩子接一句 -> 小白狐接一句 -> 结束 / 再接一句
```

### 1.5 禁止

```text
不评价孩子说得好不好
不纠正语法
不要求完整句子
不生成太长故事
不涉及恐怖、死亡、打斗、危险
不做无限续写
```

### 1.6 适合作为首个语言游戏吗

适合，优先级最高。

原因：

```text
实现简单；
孩子门槛低；
吸引力直接；
容易和小展台/小门联动；
不像学习任务。
```

## 2. 方向二：怪词变变变

### 2.1 一句话定义

```text
孩子说一个普通词，小白狐把它变成一个奇怪但温柔的新词。
```

### 2.2 示例体验

小白狐：

```text
给我一个普通的小词
我把它变奇怪一点
```

孩子说：

```text
杯子
```

小白狐：

```text
杯子变成了：咕噜咕噜月亮杯

它不是用来喝水的
它会把小小的声音装进去
```

按钮：

```text
再变一个
放进小展台
先聊别的
```

### 2.3 玩法重点

不是词汇学习，而是想象力。

孩子会想：

```text
我再说一个词，看它会变成什么。
```

### 2.4 首版词类

首版只允许普通安全名词：

```text
杯子
袜子
云朵
小车
枕头
书包
树叶
铅笔
小灯
积木
```

如果孩子说抽象、危险、隐私、作业内容，就温柔转回：

```text
这个词先不变
我们换一个小东西的名字试试
```

### 2.5 禁止

```text
不做成识字
不解释词义
不纠错
不生成攻击性词
不让孩子输入人名、学校名、地址
不生成武器、怪兽战斗、奖励词
```

### 2.6 优先级

适合第二个做。

原因：

```text
重复玩动力强；
实现可控；
可以承接小展台命名；
但需要词池和安全规则更严。
```

## 3. 方向三：小白狐听错了

### 3.1 一句话定义

```text
孩子说一句话，小白狐假装听成一个很可爱的错句，然后一起纠正或继续玩。
```

### 3.2 示例体验

小白狐：

```text
你说一句短短的话
我看看我会不会听错
```

孩子说：

```text
我今天吃了面包
```

小白狐：

```text
我听成了：我今天吃了棉袄

棉袄可不能吃呀
它只能把肚子盖暖一点
```

按钮：

```text
再说一句
这次听对点
先聊别的
```

### 3.3 吸引点

孩子会觉得小白狐有点笨、有点可爱，愿意多说几句。

### 3.4 风险

这个玩法容易依赖 ASR 错误。如果处理不好，会让孩子觉得“小白狐听不懂我”。

### 3.5 规则

首版不要真的依赖 ASR 错误，而是从孩子句子里抽一个词，确定性替换成相近但安全的可爱词。

例如：

```text
面包 -> 棉袄
铅笔 -> 铅饼
小车 -> 小蛇
书包 -> 树包
```

### 3.6 禁止

```text
不嘲笑孩子
不说“你说错了”
不说“我听不懂”
不引导孩子大喊
不把隐私内容拿来玩
```

### 3.7 优先级

适合后做。

原因：

```text
有趣，但边界更难；
对语音识别和话术把控要求更高。
```

## 4. 方向四：三个怪答案

### 4.1 一句话定义

```text
小白狐问一个没有标准答案的小问题，给孩子三个怪选项，也允许孩子自己说。
```

### 4.2 示例体验

小白狐：

```text
如果一朵云掉进小屋里
它最想躲在哪里？
```

选项：

```text
枕头下面
杯子里面
小白狐尾巴后面
我自己说
```

孩子选：

```text
小白狐尾巴后面
```

小白狐：

```text
它躲到我的尾巴后面啦
我一转身
尾巴变成了一朵软软的云扫把
```

### 4.3 吸引点

不依赖孩子长句表达，适合低龄儿童。

### 4.4 规则

问题必须是开放想象题，不是知识题。

适合的问题：

```text
如果月亮变小了，它会躲在哪里？
如果袜子会说话，它第一句会说什么？
如果杯子长出小翅膀，它会飞去哪儿？
```

### 4.5 禁止

```text
不问知识题
不问算术题
不问道德评判题
不问隐私题
不做测验
```

### 4.6 优先级

适合首批候选。

原因：

```text
低龄友好；
点击也能玩；
比纯语音更稳；
但需要准备问题库和选项库。
```

## 5. 方向五：反过来说

### 5.1 一句话定义

```text
小白狐说一句简单的话，孩子反过来说，得到一个搞笑结果。
```

### 5.2 示例体验

小白狐：

```text
我说：小狐狸钻进杯子里
你把它反过来说试试
```

孩子说：

```text
杯子钻进小狐狸里
```

小白狐：

```text
哎呀
那小狐狸肚子里会咣当咣当地响
```

### 5.3 优点

语言结构感强，有点脑筋急转弯的感觉。

### 5.4 风险

对 5-6 岁可能偏难。

### 5.5 优先级

暂不优先。

适合作为 7-10 岁分龄玩法，或者未来“更会说话的小门问题”。

---

## 6. 首批建议做哪几个

建议首批只选两个：

```text
A. 接一句故事
B. 三个怪答案
```

原因：

```text
1. 都不需要孩子必须识字；
2. 都不强调对错；
3. 都能很快产生有趣反馈；
4. 都适合语音输入；
5. 都可以做成短回合；
6. 都比较安全可控。
```

第二批再考虑：

```text
怪词变变变
小白狐听错了
```

暂缓：

```text
反过来说
```

## 7. 推荐下一步：L1 接一句故事 Demo

### 7.1 为什么先做 L1

“接一句故事”最符合 ai-child 当前方向：陪伴、想象、关系感、轻共创。

它能让孩子产生：

```text
我说的话真的进入了小白狐的小故事。
```

### 7.2 L1 首版只做 3 个固定开头

开头一：

```text
我刚刚在地毯下面
发现了一只会打哈欠的小盒子
它一张嘴，里面竟然有……
```

开头二：

```text
窗边的小星星掉下来一颗
它滚到小白狐脚边
小声说：我在找……
```

开头三：

```text
小屋门口来了一只慢吞吞的纸船
它没有水也能漂
船里装着……
```

### 7.3 儿童按钮

```text
我来接一句
换一个开头
先聊别的
```

孩子接完后：

```text
再接一句
放进小展台
先聊别的
```

### 7.4 收束原则

最多接两轮。

第二轮后小白狐说：

```text
这个小故事先停在这里
它还在小屋里轻轻晃着呢
```

按钮：

```text
再玩一个
放进小展台
先聊别的
```

### 7.5 是否接小展台

首版可以只保存故事标题和一句摘要，不保存长篇。

但如果实现复杂，L1-A 先不接小展台，只做本地短回合。

## 8. 推荐下一步：L2 三个怪答案 Demo

### 8.1 首版固定 3 个问题

问题一：

```text
如果一朵云掉进小屋里
它最想躲在哪里？
```

选项：

```text
枕头下面
杯子里面
小白狐尾巴后面
我自己说
```

问题二：

```text
如果袜子会说话
它第一句会说什么？
```

选项：

```text
我想找另一只袜子
今天不要踩水坑
我其实是一只小口袋
我自己说
```

问题三：

```text
如果铅笔晚上不睡觉
它会偷偷画什么？
```

选项：

```text
一条弯弯的小路
一只睡着的杯子
一个会飞的西瓜
我自己说
```

### 8.2 反馈原则

不评价对错。

反馈格式：

```text
那就这样吧
{孩子选择/回答} 出现在小屋里
小白狐看了看，轻轻笑了一下
```

再补一句固定想象反馈。

## 10. 主控建议

下一步不要同时做多个语言游戏。

建议开发顺序：

```text
L0：语言游戏入口计划
L1：接一句故事 Demo
L2：三个怪答案 Demo
```

其中 L0 先做计划，不编码。

L0 需要决定：

```text
1. 语言游戏从哪里进入；
2. 是否和奇怪小门并列；
3. 是否需要今日事件选择器；
4. 普通聊天页如何不被入口淹没；
5. 每个语言游戏如何退出；
6. 是否需要后端；
7. 是否接小展台；
8. 如何避免学习化。
```
