# 小白狐轻记忆儿童端文案 v0.1

项目：`ai-child`

日期：2026-06-13

状态：M2 master-copy / 待主控审核

用途：定义“小白狐轻记忆”在儿童端可展示的 master-copy、使用条件和禁止边界。本文只作为后续 M3 Android 展示实现的文案依据，不修改代码，不新增后端记忆，不做跨天持久化。

---

## 1. 核心原则

轻记忆文案必须像“小白狐轻轻想起来”，不能像系统记录。

必须符合：

```text
1. 低频；
2. 短句；
3. 可跳过；
4. 不追问；
5. 不复盘细节；
6. 不评价孩子；
7. 不制造亏欠；
8. 不暗示唯一关系；
9. 不说一直等你；
10. 不说我一直记着你。
```

核心感觉：

```text
小白狐好像轻轻想起一点点，
然后马上把选择权还给孩子。
```

```text
4. 家长端报告；
5. 孩子原始回答；
6. 错误答案；
7. 分数、等级、排名；
8. 原始语音；
9. 原始照片；
10. 原始聊天全文。
```

## 3. opening 使用条件

opening 中允许展示轻记忆文案的条件：

```text
1. ChatUiState.lightMemory.openingRecallCandidateId != null；
2. strangeDoorDemo == null；
3. languageGame == null；
4. 本 ChatViewModel 生命周期内没有 recall 过；
5. lightMemory 没有 mutedForCurrentLifecycle；
6. 当前不是拍照、上传、小展台选择、语言游戏、奇怪小门进行中；
7. 当前没有安全 / 隐私 / 学习 / 睡前收尾等更高优先级场景。
```

opening 中不允许：

```text
1. 每次打开都提；
2. 连续提同一个小发现；
3. 孩子先聊别的后继续提；
4. 孩子短答或跳过后追问；
5. 在奇怪小门进行中插入；
6. 在语言游戏进行中插入。
```

## 4. opening 小展台来源文案

适用来源：

```text
ShowcaseItem
ShowcaseAssist
```

主文案：

```text
我刚才想起一个小发现
{name} 好像还在小展台里安安静静地待着

今天我们也可以聊新的东西
```

更轻版本：

```text
我想起了小展台里的 {name}
它好像轻轻动了一下

不过今天想聊新的也可以
```

使用限制：

```text
1. 不说“你上次”；
2. 不说“我一直记得”；
3. 不说“它在等你”；
4. 不要求孩子去小展台；
5. 不自动跳转小展台。
```

## 5. opening 奇怪小门完成来源文案

适用来源：

主文案：

```text
我好像想起一扇奇怪小门
有个小东西曾经帮过忙

```

如果有安全 displayName：

```text
我好像想起 {name}

```

使用限制：

```text
1. 不说通关；
2. 不说你完成了几次；
3. 不说继续任务；
4. 不显示开门次数；
5. 不做连续回访。
```

## 7. opening 按钮策略

M2 不新增按钮。

opening 轻记忆出现时，仍复用当前普通聊天 / 游戏入口已有选择。

不新增：

```text
1. 去看看；
2. 继续上次；
3. 打开小展台；
4. 再帮一次；
5. 不要记了；
6. 忘掉这个。
```

原因：M1 / M2 只做轻召回，不做新流程入口，不做孩子端显式记忆管理。

## 8. 普通聊天主动相关使用条件

普通聊天只在孩子主动相关时轻接。

允许触发关键词沿用 M1：

```text
小门
奇怪小门
小展台
小发现
刚才那个
我放进去的
帮小白狐
```

允许文案结构：

```text
轻轻接住孩子的话
+ 提一个已有安全摘要
+ 不强拉流程
```

## 9. 普通聊天主动相关文案

孩子：小展台里有什么？

可回应：

```text
我记得小展台里有一个小发现
{name} 还在那里

你想说说它，或者聊新的都可以
```

孩子：那个小门呢？

可回应：

```text
那扇奇怪小门我也有一点点印象
有个小东西曾经帮过忙

不过我们也可以先聊别的
```

普通聊天中禁止：

```text
1. 孩子说新话题时强拉回旧记忆；
2. 孩子负面情绪时提娱乐记忆；
3. 睡前提兴奋玩法；
4. 孩子短答后继续追问；
5. 引导孩子必须回去玩。
```

## 10. 奇怪小门展示策略

奇怪小门中可以很轻地承接既有 S5 “用小展台里的”路径，但 M2 不新增小门主流程文案。

允许：

```text
1. 如果 S5 已显示“用小展台里的”，可让最近小发现排序靠前；
2. 选择页仍使用“选一个小发现”；
3. 点击旧物后的反馈仍使用 S5 已确认文案；
4. 不新增“我记得这个”的专门文案。
```

禁止：

```text
1. 不新增奇怪小门轻记忆提示；
2. 不说“上次你用过”；
3. 不说“它又等着你”；
4. 不新增任务、关卡、奖励；
5. 不改变门状态节奏。
```

## 12. 孩子跳过、换题、先聊别的后的处理

处理策略：

```text
1. 孩子选择先聊别的后，本 ChatViewModel 生命周期内不再自动提轻记忆。
2. 孩子换题后，本 ChatViewModel 生命周期内不再继续追同一条轻记忆。
3. 孩子短答、跳过或没有接住时，不追问、不补充解释、不再次召回。
4. 孩子进入奇怪小门或语言游戏后，不插入轻记忆。
5. 以上处理只影响本生命周期，不新增跨天持久化。
```

## 13. 必须不提的情况

以下情况必须不展示轻记忆：

```text
1. openingRecallCandidateId == null；
2. strangeDoorDemo != null；
3. languageGame != null；
4. mutedForCurrentLifecycle == true；
5. recalledInCurrentLifecycle == true；
6. 当前处于拍照、上传、小展台选择或保存流程；
7. 当前存在安全、隐私、学习、睡前收尾等更高优先级场景；
8. 候选来源不是 M1 白名单；
9. 候选命中 blocked / privacy / homework / unsafe；
10. 候选来自语言游戏偏好、普通聊天自由抽取或后端长期摘要；
11. 需要展示原始语音、原始照片、原始聊天全文、孩子原始回答或图片识别全文时；
12. 孩子已经表达先聊别的、换题、跳过或暂时不想接。
```

## 14. 禁止文案

M2 必须禁止以下表达：

```text
我一直记得你
我一直在等你
你终于回来了
你怎么才回来
只有我记得
这是我们的秘密
不要告诉爸爸妈妈
你上次做错了
你上次答错了
你已经完成了几关
你连续来了几天
你获得了奖励
你升级了
你应该继续练习
你最近不太会
你比以前厉害
我记录了你的表现
```

也禁止任何：

```text
1. 依赖感；
2. 亏欠感；
3. 监控感；
4. 成绩感；
5. 任务感；
6. 打卡感；
7. 排名感；
8. 家长监控感；
9. 秘密关系感。
```

## 16. 真机验收标准

真机验收时判断“不像监控、不像复盘”的标准：

```text
1. 孩子第一感受是小白狐轻轻想起，而不是系统汇报。
2. 文案不复述照片、语音、聊天全文或孩子回答。
3. 文案不出现次数、成绩、等级、奖励、任务或连续记录。
4. 小白狐不催孩子继续旧流程。
5. 孩子跳过或先聊别的后，小白狐自然放下。
6. 轻记忆不抢奇怪小门、语言游戏或普通聊天主线。
7. 家长端没有新增监控感入口或报告项。
8. 仓库、日志和测试材料不包含真实儿童照片、音频、聊天转录或家庭测试材料。
```
