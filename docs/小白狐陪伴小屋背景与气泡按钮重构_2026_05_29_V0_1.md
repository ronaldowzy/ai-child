# 小白狐陪伴小屋背景与气泡按钮重构 v0.1

项目：`ai-child`

用途：根据最新选定的“小白狐温暖小屋背景图”，对儿童主界面进行一次视觉表达重构：接入背景图、弱化白色舞台卡片、重做状态气泡、对话浮泡和底部按钮。本文只指导 UI 表达层，不改业务状态机、不改 prompt、不改家长日报、不改小白狐 10 状态资源。

更新时间：2026-05-29


主控判断：

```text
该背景方向正确：有温暖小屋、窗户、月亮、云朵、星星、地毯和柔光，中心区域干净，适合叠加小白狐动态图。
```

## 1. 本轮只做什么

本轮只做 UI 表达层：

```text
1. 接入选定背景图，并转换为合适的 WebP 资源；
2. 移除或大幅弱化当前小白狐外层白色舞台卡片；
3. 让小白狐直接站在背景图的地毯/中央空间上；
4. 放大小白狐，强化第一视觉；
5. 将状态文字重做为真正的说话气泡；
6. 将最近对话改为围绕小白狐出现的临时漂浮气泡；
7. 将底部次级入口全部改成明确的胶囊按钮；
8. 横屏去掉右侧聊天控制台感，保留轻量操作区。
```

### 3.2 转换要求

必须转换后接入，不要直接把源 PNG 原样打入 APK。

建议：

```text

2. 保留足够分辨率，优先 1600-2048 边长；

4. 输出资源体积报告；
5. 源 PNG 不提交到 Android 资源目录；
6. 只提交最终 WebP 或等价轻量资源。
```

### 3.3 适配要求

背景图使用方式：

```text

横屏：完整展示更多小屋环境，左侧或中部放小白狐；
使用 ContentScale.Crop 或等价方式；
必要时为竖屏/横屏设置不同 alignment，但不要生成多套复杂资源。
```

注意：

```text
1. 小白狐不能压在书架、窗户、月亮等复杂物体上；
2. 小白狐应站在地毯或中央干净区域；
3. 底部按钮不能压住背景中重要装饰；
4. 背景需要轻遮罩，保证气泡和按钮可读。
```

## 4. 移除白色舞台卡片感

当前问题：

```text
小白狐被放在一个大白色舞台卡片里，背景再好也会被卡片割裂。
```

本轮要求：

```text
1. 移除或极大弱化小白狐外层大白卡片；
2. 不再让小白狐像贴在白板上；
3. 小白狐应该直接处在背景图的地毯/中央空间中；
4. 可以保留非常轻的局部柔光，但不要形成明显白色矩形容器。
```

---

## 5. 小白狐尺寸与位置

要求：

```text
1. 竖屏小白狐应比当前放大约 15%-25%；
2. 横屏小白狐应比当前放大约 35%-50%；
3. 不能裁耳朵、尾巴、爪子；
4. 小白狐应落在背景地毯或中心柔光区域；
5. 不要贴顶、贴边；
6. 不要被底部按钮遮挡。
```

视觉目标：

```text
第一眼看到的是“小白狐在房间里等我”，而不是“页面上有一只小狐狸素材”。
```

## 6. 状态气泡重做

### 6.1 气泡视觉

状态气泡必须参考效果图里的说话气泡风格，而不是现在的小标签。

要求：

```text
1. 白色/奶白半透明；
2. 大圆角；
3. 轻阴影；
4. 有小尾巴或视觉指向小白狐；
5. 靠近小白狐脸旁、右上、右下或下方偏侧；
6. 不贴舞台底部；
7. 不像按钮；
8. 不像系统 Toast；
9. 不像页面脚注。
```

### 6.2 气泡文案

全部去掉句号，使用短句：

| 状态 | 文案 |
|---|---|
| idle | 我在这儿 |
| waiting_soft | 想说再说 |
| listening | 我在听 |
| thinking | 我想想 |
| preparing_speech | 我准备说 |
| speaking | 不显示状态气泡 |
| image_viewing | 我在看 |
| co_create | 一起想想 |
| paused | 先停一下 |
| retry | 再试一次 |

### 6.3 气泡动效

状态气泡应随状态动态切换：

```text
1. fade in / fade out；
2. 轻微 slide 或 scale；
3. 时长 180-260ms；
4. 不弹跳；
5. 不闪烁；
6. 不做奖励动效。
```

## 7. 对话记录改为临时漂浮气泡

当前问题：

```text
固定聊天卡片让界面重新像聊天工具。
```

本轮要求：

```text
1. 最近小白狐回复以气泡形式出现在小白狐周围；
2. 孩子刚说的话可用另一侧小气泡显示；
3. 最多同时显示 1-2 个对话气泡；
4. 气泡停留 4-8 秒后淡出；
5. 不常驻成聊天面板；
6. 空状态不显示消息卡；
7. 长文本最多显示 2 行，超出截断；
8. 不把状态气泡写进聊天记录。
```

建议位置：

```text
小白狐回复：靠近小白狐左上/右上；
孩子输入：靠近小白狐另一侧或下方；
横屏：也围绕小白狐，不放成右侧聊天列表。
```

如果现有架构不方便做自动淡出，最低限度也要：

```text
1. 改成漂浮气泡样式；
2. 不放在右侧固定白色聊天面板；
3. 最多显示最近 1 条；
4. 不压住小白狐。
```

## 8. 底部按钮重做

当前问题：

```text
除了“按一下开始说”，其他入口都是纯文字，不像按钮。
```

本轮必须修改：

```text
所有次级操作必须变成明确按钮，而不是纯文字。
```

底部结构：

```text
主按钮：按一下开始说 / 说完了；
次按钮：给小白狐看看；
次按钮：换个话题；
次按钮：停一下。
```

按钮样式：

```text
1. 奶白/白色半透明胶囊；
2. 圆角；
3. 轻边框；
4. 轻阴影；
5. 图标 + 文字，或至少有明确按钮底；
6. 高度不低于 44dp；
7. 不使用纯文字 TextButton 样式；
8. 不像任务按钮；
9. 不像链接。
```

主按钮：

```text
浅蓝柔和渐变或柔和蓝色；
不要太像系统标准纯蓝按钮；
继续保持最大、最明显。
```

## 10. 横屏重构重点

横屏不能再是右侧聊天控制台。

要求：

```text
1. 背景图铺满全屏；
2. 小白狐放大并站在背景中央或偏左；
3. 对话气泡围绕小白狐；
4. 右侧只保留轻量操作区；
5. 不再重复显示“小白狐”标题 + 消息卡；
6. 不显示右侧固定聊天面板；
7. 底部或右侧按钮都必须是明确按钮。
```

如果横屏空间有限：

```text
优先小白狐和语音主按钮；
其次拍图；
再次换话题/停一下；
最后才是最近气泡。
```

## 11. 竖屏重构重点

竖屏结构：

```text
顶部：家长入口；
中上：小白狐在背景房间中；
小白狐旁边：状态气泡；
小白狐周围：最近对话气泡，临时出现；
底部：语音主按钮 + 次级胶囊按钮。
```

禁止：

```text
1. 不显示半截消息卡；
2. 不显示脚注式“不说也没关系”；
3. 不显示“小白狐在这里。”这种带句号标签；
4. 不显示纯文字次级按钮；
5. 不让大白卡片盖住背景图。
```
