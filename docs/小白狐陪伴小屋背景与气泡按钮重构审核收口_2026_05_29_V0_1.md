## 3. 通过点

### 3.1 背景图方向正确

已新增：

```text
android/app/src/main/res/drawable-nodpi/companion_room_background.webp
```

主控判断：

```text
符合“使用一张温暖小屋背景图解决场域感”的方向。
```

真机需进一步确认：

```text
1. 竖屏裁切是否保留地毯和中心空间；
2. 横屏是否不裁掉关键窗户/星灯元素；
3. 背景是否过亮导致气泡和按钮可读性下降；
4. WebP 体积是否可接受。
```

### 3.2 对话浮泡方向正确

新增 `CompanionFloatingConversationBubbles`，并按最近消息显示浮动气泡，约 6 秒后淡出。

主控判断：

```text
方向正确，已经从固定聊天卡片转向临时浮泡。
```

真机需进一步确认：

```text
1. 气泡是否挡住小白狐脸、耳朵、爪子；
2. 长文本 2 行截断是否自然；
3. 多条气泡是否拥挤；
4. 6 秒停留是否过短或过长。
```

### 3.3 按钮方向正确

图片失败操作已经从纯文字变成按钮，InputBar 也有视觉改造。

主控判断：

```text
方向正确，但仍需真机确认“给小白狐看看 / 换个话题 / 停一下”等次级按钮是否都已经不再是纯文字。
```

### 3.4 横屏方向正确

横屏不再使用原来的 `ChatPanel`，改为 `LandscapeOperationPanel`。

主控判断：

```text
方向正确，目标是去掉右侧聊天控制台感。
```

真机需进一步确认：

```text
1. 右侧是否仍像控制台；
2. 小白狐是否足够大；
3. 背景是否在横屏下显得像完整小屋；
4. 对话浮泡是否围绕小白狐，而不是变成右侧聊天列表。
```

## 4. 需要重点看截图/真机的问题

本轮代码可以进入视觉验收，但不能仅凭代码判定最终通过。

必须看截图/真机确认：

```text
1. 小白狐是否真的像站在地毯/房间中央，而不是悬浮在背景上；

3. 状态气泡是否像说话气泡，而不是标签；

5. 最近对话是否不再常驻成聊天面板；
6. 次级入口是否都是胶囊按钮，不再是纯文字；
7. 横屏是否不再是右侧聊天控制台；
8. 竖屏是否不出现半截消息卡；
9. 背景图是否压过小白狐和按钮；
10. 图标、背景和小白狐风格是否一致。
```
