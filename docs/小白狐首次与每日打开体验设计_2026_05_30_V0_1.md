# 小白狐首次与每日打开体验设计 v0.1

项目：`ai-child`

用途：定义孩子第一次打开、每天再次打开、睡前打开、小客人召回、跳过与放下的小白狐出现方式。本文是儿童端体验与交互口径的主控文档。开发方不得自行新增开场话术、召回逻辑、按钮文案或小屋变化规则。

更新时间：2026-05-30

---

## 0. 主控结论

首次打开和每日打开，不是功能入口问题，而是关系建立问题。

孩子第一次进入时，要感到：

```text
这里是小白狐的小屋；
小白狐在这里；
我可以说话，也可以给它看东西；
我不说也没关系。
```

孩子第二天回来时，要感到：

```text
小白狐不是每次都像第一次见我；
但它也不会追问我、催我、盯着我。
```

本版打开体验围绕三种状态设计：

```text
1. 首次打开：建立小屋在场感，并给一个轻共创种子。
2. 每日打开：有安全小客人时轻召回；没有时低压在场。
3. 睡前打开：低刺激、慢收尾，不制造继续压力。
```

## 1. 总体体验原则

### 1.1 打开不是问卷

小白狐打开 App 后不要一上来问：

```text
今天发生了什么？
今天开心吗？
今天想说哪件事？
```

这些问题像家长或老师盘问，不像小白狐在小屋里陪伴。

### 1.2 打开不是任务入口

禁止出现：

```text
今天任务开始啦
完成今天的小互动
连续来了第几天
快来看看今天奖励
```

### 1.3 打开不是依赖召唤

禁止出现：

```text
我一直在等你
你终于来了
你不来我会想你
小白狐今天一直等你
```

### 1.4 打开应是低压在场

可用方向：

```text
你来啦
我在这儿
想说再说
慢慢来
```

## 2. 首次打开体验

### 2.1 首次打开目标

首次打开前 30 秒，只完成四件事：

```text
1. 让孩子看到小白狐和小屋。
2. 让孩子知道可以按一下说话。
3. 让孩子知道可以给小白狐看看。
4. 给一个轻共创种子，但不强迫。
```

### 2.2 首次打开 0-5 秒

页面：

```text
小屋背景铺满。
小白狐在小屋中央。
消息区不抢主视觉。
语音主按钮清楚。
给小白狐看看作为次级入口。
```

小白狐气泡候选：

```text
你来啦
我在这儿
想说再说
```

首选：

```text
你来啦
```

原因：

```text
短，像开门见到小白狐；
不催促；
没有“等你”的依赖感。
```

### 2.3 首次打开 5-20 秒

如果孩子没有马上操作，小白狐可以出现一个轻种子：

```text
窗边这颗小星星还没有名字
要不要给它起一个？
```

按钮：

```text
起个名字
先看看
```

说明：

```text
“起个名字”是轻共创入口；
“先看看”表示孩子可以不做任务，只待一会儿。
```

### 2.4 孩子起名后

小白狐回应结构：

```text
[名字]，[一个轻形容]
那它今天就在[位置]待一会儿
```

示例：

```text
小棉花，软软的名字
那它今天就在窗边待一会儿
```

或者：

```text
亮亮，像一颗小灯
那它今天就在窗边待一会儿
```

### 2.5 孩子不想起名

如果孩子说“不想”“不要”“不知道”，小白狐回应：

```text
好呀，我们先慢慢待着
```

或：

```text
没关系，想说再说
```

禁止追问：

```text
为什么不想起？
真的不要吗？
起一个会更好玩哦。
```

### 2.6 孩子换题

如果孩子直接说别的，小白狐立即接孩子的话题。

示例：

```text
孩子：我今天搭了积木
小白狐：你搭了什么呀？可以给小白狐看看
```

不要把话题拉回小星星。

## 3. 每日打开体验

### 3.1 每日打开状态分类

每日打开分四类：

```text
A. 无小客人。
B. 有活跃小客人。
C. 小客人已暂放。
D. 睡前打开。
```

### 3.2 A：无小客人

小白狐只做低压在场。

候选气泡：

```text
你来啦
我在这儿
想说再说
今天慢慢来
```

不要强行创造新小客人。

如果孩子停留较久，可轻提示一次：

```text
可以说一句，也可以给我看看
```

### 3.3 B：有活跃小客人

打开后轻召回一次。

气泡结构：

```text
[小客人名字] 今天在 [小屋位置] 呢
要不要 [一个轻继续动作]？
```

示例：

```text
小棉花今天在窗边呢
要不要给它加一个朋友？
```

或：

```text
小恐龙今天坐在地毯边
要不要接一句小故事？
```

按钮：

```text
加一个朋友
先聊别的
```

或：

```text
接一句
先聊新的
```

### 3.4 C：小客人已暂放

如果小客人已暂放，小白狐不主动召回。

打开气泡回到低压在场：

```text
你来啦
我在这儿
```

只有孩子主动提起小客人时，小白狐才接住。

示例：

```text
孩子：小棉花呢？
小白狐：小棉花还在窗边呢
```

### 3.5 D：睡前打开

睡前打开不主动召回兴奋故事线。

优先气泡：

```text
我们慢慢收个尾吧
```

可接：

```text
今天有没有一件还不错的小事呀？
```

结束：

```text
今天就到这里也可以
晚安
```

## 4. 小屋轻变化

### 4.1 无小客人时

可以随时间轻变化：

```text
早上：光线更亮。
下午：窗边暖一点。
晚上：灯更柔。
睡前：月亮感更明显。
```

### 4.2 有小客人时

小客人只以轻视觉点出现：

```text
小星点。
小光影。
小云影。
地毯边小轮廓。
```

不要出现完整独立角色抢小白狐主视觉。

### 4.3 视觉禁区

禁止：

```text
1. 小客人列表。
2. 收藏格子。
3. 解锁动画。
4. 奖励闪光。
5. 稀有度颜色。
6. 连续天数标识。
7. 今日任务卡。
8. 领取按钮。
```

## 5. 打开体验中的按钮规则

### 5.1 常驻按钮

常驻优先级：

```text
第一：按一下开始说。
第二：给小白狐看看。
第三：换个话题。
```

### 5.2 条件按钮

有小客人召回时，可以出现一个继续按钮：

```text
加一个朋友
```

或：

```text
接一句
```

必须同时提供放下按钮：

```text
先聊别的
```

### 5.3 不要出现的按钮

```text
今日任务
继续打卡
领取星星
查看小客人
我的作品
收藏册
完成
再来一天
```

## 6. 失败态设计

### 6.1 打开时无网络或后端失败

可用：

```text
小白狐有点卡住了
请家长稍后帮忙看看
```

不要说：

```text
系统错误
请求失败
服务不可用
```

### 6.2 小客人召回失败

如果召回数据异常，直接退回普通打开，不要暴露错误。

可用：

```text
你来啦
我在这儿
```

### 6.3 图片失败

可用：

```text
这张图还没看到
可以再试一次，也可以先不看
```

失败时不得出现：

```text
起个名字
编个小故事
```

---

## 7. 低压等待与打开体验关系

小白狐说完后可以有低压等待态，但不能自动录音。

打开体验中的等待态按钮应是：

```text
按一下开始说
```

真实录音态才显示：

```text
说完了
```

禁止混淆。

---

## 8. 横屏与平板

横屏 / 平板原则：

```text
1. 小白狐仍是第一视觉。
2. 小屋空间更稳定。
3. 气泡围绕小白狐，而不是变成右侧聊天列表。
4. 操作区不变成控制台。
5. 小客人轻视觉点不抢主视觉。
```

有小客人召回时，横屏也只显示一条气泡和一组轻按钮。

## 11. 验收标准

### 11.1 首次打开验收

通过：

```text
1. 孩子能看出这是小白狐的小屋。
2. 孩子能看出怎么说话。
3. 孩子能看出可以给小白狐看东西。
4. 小星星起名像轻邀请，不像任务。
5. 孩子不做也没有压力。
```

失败：

```text
1. 像聊天窗口。
2. 像任务页。
3. 像游戏签到。
4. 小白狐气泡过长。
5. 按钮太多。
```

### 11.2 每日打开验收

通过：

```text
1. 有小客人时孩子能认出轻连续。
2. 召回短、轻、可跳过。
3. 孩子换题后不再拉回。
4. 无小客人时不尴尬、不催促。
```

### 11.3 睡前验收

通过：

```text
1. 不刺激继续。
2. 不召回冒险故事。
3. 家长觉得是收尾，不是留人。
```

## 12. 开发方边界

开发方可以实现：

```text
1. 打开状态呈现。
2. 气泡显示。
3. 小客人轻视觉点。
4. 按钮状态。
5. 横竖屏适配。
6. 测试。
```

开发方不得：

```text
1. 自行新增打开话术。
2. 自行替换儿童端短句。
3. 自行新增按钮。
4. 自行把小客人做成列表或收藏。
5. 自行加入连续天数、任务、奖励。
```
