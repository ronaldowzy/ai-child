## 1. 开发进程 A：二次返修要求

### 1.1 已通过内容

以下内容可以保留：

```text
1. contains_image 阻断 fast path；
2. image_context 阻断 fast path；
3. homework_context 阻断 fast path；
4. pending_image / image_uploading / attachment_count 阻断 fast path；
5. co_creation_suggested / co_creation_type / co_creation_active 阻断 fast path；
6. 普通低风险聊天仍可走 fast path；
7. prompt 分层长度观测保持。
```

### 1.2 当前问题

A 的测试中将原来的真实运动用例替换为：

```text
我一会要去打球了。
一块做运动了，做了很多。
```

并注明：

```text
“跑” is in _IMAGINATIVE_MARKERS
“跳” is in _IMAGINATIVE_MARKERS
```

这说明当前轻共创检测可能把普通运动词“跑 / 跳”误判为想象性内容，导致 fast path 阻断。

这会错过最初真机测试暴露的问题：

```text
真实请求只有 13 字用户输入，例如“一块跳绳了，我们跳了很多”，但 system prompt 有 7603 字。
```

如果这类普通运动聊天仍然因为“跳”触发轻共创而走完整 prompt，那么 Prompt Fast Path 的主要目标没有完全达成。

### 1.3 返修目标

不是放松“轻共创已触发时必须走完整 prompt”的规则。

而是要让轻共创检测本身更准确：

```text
普通运动、跑步、跳绳、打球，不应仅因包含“跑/跳”就触发故事接龙或轻共创；
只有明确想象性、故事性、角色性内容才可触发轻共创。
```

### 1.4 返修要求

开发进程 A 必须：

```text
1. 从 light_co_creation_service 的 _IMAGINATIVE_MARKERS 中移除或弱化单字“飞 / 跑 / 跳 / 游 / 爬”；
2. 普通“跑步 / 跳绳 / 打球 / 做运动 / 比赛”不得触发 co_creation_suggested；
3. 明确故事性内容仍可触发轻共创，例如“我想编一个恐龙故事”；
4. fast path eligibility 继续阻断已经触发的轻共创；
5. 不要通过改测试文本绕过问题；
6. 恢复或新增原始目标用例：“一块跳绳了，我们跳了很多。”应走 fast path，除非其他复杂条件存在。
```

建议策略：

```text
1. 将轻共创触发从“出现任一想象 marker”改成更严格组合：角色/故事/想象/如果/编/冒险/魔法/太空/恐龙 等至少一种强想象信号；
2. 将普通运动词从强想象信号中移除；
3. 如保留“飞/跑/跳/游/爬”，必须要求同时出现角色或故事场景，例如“小兔跳到月亮上”“机器人跑去城堡”。
```

### 1.5 必须补测试

至少覆盖：

```text
1. “一块跳绳了，我们跳了很多。” 不触发 co_creation_suggested；
2. “我一会要去跑步了。” 不触发 co_creation_suggested；
3. “我一会要去打球了。” 仍走 fast path；
4. “我想编一个恐龙故事。” 触发 co_creation_suggested，并阻断 fast path；
5. “小兔跳到月亮上。” 可以触发轻共创；
6. 图片/学习/安全/隐私/睡前仍不走 fast path。
```

## 2. 开发进程 B：二次返修要求

### 2.1 已通过内容

以下内容可以保留：

```text
1. 家长端仍只展示“今天一句话 / 孩子今天提到的内容 / 需要留意的地方”；
2. 孩子今天提到的内容优先展示 topic.summary；
3. topic.topic 仅作为 fallback；
4. 新生成日报不填充 tonight_parent_bridge / suggested_parent_actions / avoid_followup；
5. deterministic_fallback_hints 收敛；
6. report_schema 改向三块结构。
```

### 2.2 当前问题一：strict JSON 示例用了中文弯引号

当前 `parent_report_language_v4.py` 的系统提示词中出现：

```text
“summary”
“mentioned_items”
“attention_items”
```

这与“必须返回严格 JSON object”冲突。

JSON 必须使用英文双引号：

```json
{
  "summary": "...",
  "mentioned_items": ["..."],
  "attention_items": []
}
```

返修要求：

```text
1. 所有 JSON schema 示例、好示例、坏示例中的字段名和字符串值都改成英文双引号；
2. 不要出现中文弯引号 “ ”；
3. 增加测试确保 parent_report_system_prompt_v4() 中严格 JSON 示例不包含中文弯引号；
4. 增加解析测试，确保新 schema 的标准 JSON 能正确 parse。
```

### 2.3 当前问题二：deterministic fallback 的 summary 仍有家长建议语气

当前 `deterministic_narrative_v4(...)` 仍包含类似：

```text
家长今晚不用据此判断孩子状态，可以轻松陪孩子做一件日常小事，不想聊也没关系。
家长今晚适合保持平静，先确认孩子有没有需要大人帮忙的事。
家长今晚可以先听孩子说题目大概在问什么。
家长可以给一个现实里的分享机会。
```

这些都属于家长建议/行动建议，不应出现在“今天一句话”里。

主控口径：

```text
今天一句话只做当天概览；
不写今晚怎么做；
不写家长可以怎么问；
不写行动建议。
```

返修要求：

```text
1. deterministic_narrative_v4 的返回值改成纯当天概览；
2. 不出现“家长今晚”“今晚可以”“家长可以”“适合先”“不需要追问”等建议语气；
3. safety 场景可以写“今天材料里出现需要留意的边界信号”，但不要写家长行动话术；
4. learning 场景可以写“今天出现一点学习或题目求助线索”，但不要写陪孩子找第一步；
5. image 场景可以写“今天孩子有通过图片或作品表达的倾向”，但不要写家长给分享机会。
```

建议改为：

```text
低素材：今天素材不多，只能做轻量总结。
安全：今天材料里出现了需要家长留意的边界信号。
学习：今天出现了一点学习或题目相关的求助线索。
图片：今天孩子有通过图片或作品来表达、展示的倾向。
普通：今天有一些轻量交流，材料显示孩子可能关注了{topic_text}。
```

### 2.4 可选清理

`tonight_parent_bridge_v4` 和 `avoid_followup_v4` 如果已经不再被调用，可以保留为历史兼容，但建议：

```text
1. 不再从 parent_report_service 导入；
2. 不再写新测试依赖它们；
3. 不让它们进入新生成报告或 UI；
4. 后续清理时再删除也可以。
```

### 2.5 必须补测试

至少覆盖：

```text
1. parent_report_system_prompt_v4() 不包含中文弯引号；
2. prompt 示例字段使用英文双引号；
3. 标准 JSON：{"summary":"...","mentioned_items":["..."],"attention_items":[]} 可解析；
4. deterministic_narrative_v4 返回值不包含“今晚”“家长可以”“家长今晚”“话术”“建议问”；
5. 新生成报告仍不填充 tonight_parent_bridge / suggested_parent_actions / avoid_followup；
6. Android 展示仍只保留三块。
```
