## 1. 开发进程 A 审核结论

### 1.1 已通过方向

A 已完成的方向可以保留：

```text
1. 新增 fast path 场景模板；
2. 新增轻量输出契约；
3. PromptManager.compose 支持 fast_path 参数；
4. ChildAgentRuntime 增加 fast path eligibility 判断；
5. prompt_total_chars / section_chars_by_layer 等观测指标；
6. 普通聊天 prompt 长度明显下降；
7. 学习、安全、隐私、图片、睡前等复杂场景有基础阻断；
8. 没有改家长日报和 Android UI。
```

### 1.2 必须返修：图片场景阻断不够严格

当前 `_evaluate_fast_path_eligibility(...)` 中只在 `image_context` 存在时阻断 fast path。

但主控要求是：

```text
只要本轮有图片附件或图片上传/图片处理相关状态，就不得启用普通聊天 fast path。
```

返修要求：

```text
1. 如果 conversation_metadata.contains_image == true，必须阻断 fast path；
2. 如果 image_context 存在，必须阻断 fast path；
3. 如果 homework_context 存在，必须阻断 fast path；
4. 如果存在 pending image / image upload / attachment marker / image failure 等图片相关 metadata，也必须阻断 fast path；
5. 不要用“image_context 为空”推断无图片，因为失败态、上传中、后端未确认等场景也可能没有 image_context。
```

建议 blocked_reason：

```text
has_image_or_attachment
has_image_context
has_homework_context
```

### 1.3 必须返修：轻共创触发时不得启用 fast path

当前代码注释写道：

```text
Co-creation is a hint to the model, not a prompt structure change.
The fast path prompt can handle imaginative content fine.
```

这与主控启动文档冲突。

普通聊天 fast path 可以处理普通想象性闲聊，但不能处理已经触发的轻共创流程。轻共创需要完整规则约束：不任务化、不沉淀、不连续推、不做作品库、不同时给两个入口。

返修要求：

```text
1. 如果 turn_guidance_context.co_creation_suggested 为 true，必须阻断 fast path；
2. 如果 co_creation_type 非空，必须阻断 fast path；
3. 如果 conversation metadata 表示正在轻共创或图片共创，也必须阻断 fast path；
4. 普通“孩子说了想象性内容”可以仍走 fast path，但一旦服务层已经建议/发起轻共创，就必须走完整 prompt。
```

建议 blocked_reason：

```text
co_creation_suggested
co_creation_type=<type>
```

### 1.4 必须补测试

A 返修后至少补充或修改测试：

```text
1. contains_image=true 且 image_context 为空时，fast_path_used=false；
2. image_context 存在时，fast_path_used=false；
3. homework_context 存在时，fast_path_used=false；
4. co_creation_suggested=true 时，fast_path_used=false；
5. co_creation_type 非空时，fast_path_used=false；
6. 普通低风险聊天仍 fast_path_used=true；
7. 学习、安全、隐私、睡前仍走完整 prompt。
```

### 1.5 A 不得做

```text
1. 不继续删人格；
2. 不修改家长日报；
3. 不改 Android UI；
4. 不改低压等待态；
5. 不把复杂场景也纳入 fast path；
6. 不新增儿童端文案。
```

## 2. 开发进程 B 审核结论

### 2.1 已通过方向

B 已完成的方向可以保留：

```text
1. Android 家长端四状态：正在整理 / 已生成 / 内容不足 / 失败；
2. 30 秒超时提示；
3. 失败时显示“这次没有整理成功，可以再试一次”；
4. 家长端展示收敛为 3 个区块；
5. 删除“今晚可以这样接一句 / 今晚可以这样聊 / 亲子沟通建议”等可见话术区块；
6. 删除 topic 卡片；
7. 删除“暂无结构化观察”等工程化空状态；
8. 不改儿童端聊天 prompt。
```

### 2.2 必须返修：后端仍在生成/保留家长接话字段

当前后端 `parent_report_service.py` 仍然在生成或传递以下字段：

```text
tonight_parent_bridge
suggested_parent_actions
avoid_followup
parent_bridge
relationship_parent_actions
```

并且 payload schema 仍提示：

```text
Return strict JSON. Keys: narrative_report, tonight_parent_bridge, avoid_followup.
```

这与最新产品口径冲突。

主控最新口径是：

```text
家长日报不再做“今晚接话”“建议话术”“家长指导建议”系统。
```

返修要求：

```text
1. 家长端 UI 不展示这些字段已经做对，但后端也应停止继续把它们作为模型生成目标；
2. 新生成的日报中，tonight_parent_bridge 应为空或不再作为模型要求字段；
3. suggested_parent_actions 应为空；
4. avoid_followup 如仅用于内部安全边界可保留内部字段，但不得展示，不得作为家长话术建议；
5. topic_overview 中的 parent_bridge 不再进入前端展示来源，也不要作为模型生成重点；
6. deterministic_fallback_hints 中不要再塞 relationship_parent_actions 或今晚接话建议；
7. report_schema 应调整为当前 3 区块结构，不再要求 tonight_parent_bridge。
```

建议新 schema 方向：

```text
Return strict JSON. Keys: summary, mentioned_items, attention_items.
```

或者保持兼容现有 `ParentReport` 结构，但生成逻辑应映射为：

```text
summary -> 今天一句话；
topic_overview[].summary 或 mentioned_items -> 孩子今天提到的内容；
safety_alerts -> 需要留意的地方；
tonight_parent_bridge -> null；
suggested_parent_actions -> []；
avoid_followup -> [] 或仅内部。
```

注意：若为兼容旧接口暂时保留字段，必须保证新生成内容不再填充话术建议。

### 2.3 必须返修：“孩子今天提到的内容”不能只显示 topic 标签

当前 Android 展示：

```kotlin
text = "· ${topic.topic}"
```

这可能导致页面显示成：

```text
· 运动
· 图片分享
· 情绪表达
```

这仍然像 topic 标签或兴趣画像，不是主控要求的“短点列表”。

返修要求：

```text
1. “孩子今天提到的内容”应显示具体短点，不是标签；
2. 优先使用 topic.summary；
3. 如果 topic.summary 为空，再使用 topic.topic；
4. 每条建议 12-28 字左右，过长要裁剪或由后端控制；
5. 不显示 parent_bridge；
6. 不显示 emotion_tone；
7. 不显示 child_intent；
8. 不做 topic 卡片；
9. 不做兴趣标签。
```

示例：

```text
• 和同学一起跳绳，玩了很多轮。
• 提到比赛输赢，但没有继续深聊。
• 分享了一个自己画的小东西。
```

### 2.4 必须补测试

B 返修后至少补充或修改测试：

```text
1. 新生成日报不填充 tonight_parent_bridge；
2. 新生成日报 suggested_parent_actions 为空；
3. Android 不显示“今晚可以这样接一句 / 今晚可以这样聊 / 建议话术 / 亲子沟通建议”；
4. 孩子今天提到的内容优先展示 topic.summary；
5. topic.summary 为空时才 fallback 到 topic.topic；
6. 不显示 parent_bridge / child_intent / emotion_tone；
7. 需要留意的地方为空时整个区块不显示；
8. 不展示 raw transcript。
```

### 2.5 B 不得做

```text
1. 不改 Prompt Fast Path；
2. 不改 ChildAgentRuntime；
3. 不改儿童端聊天 prompt；
4. 不做成长档案；
5. 不做家长话术建议系统；
6. 不新增周报/月报；
7. 不做多孩子/多家长治理；
8. 不展示儿童原文全文。
```

## 3. 合并边界

A/B 返修仍然可以并行，但必须遵守边界：

```text
A 只改普通聊天 fast path eligibility、prompt 观测和对应测试；
B 只改家长日报生成/展示和对应测试；
A 不碰家长日报；
B 不碰儿童聊天 prompt；
任何一方如果需要跨界，先停止并回报主控。
```

## 4. 主控收口标准

A 返修收口标准：

```text
1. 低风险普通聊天使用 fast path；
2. 图片、学习、安全、隐私、睡前、记忆召回、轻共创均完整阻断；
3. prompt 分层长度仍可观测；
4. 普通聊天 prompt 仍明显下降；
5. 复杂场景不被削弱。
```

B 返修收口标准：

```text
1. 家长端只展示今天一句话、孩子今天提到的内容、需要留意的地方；
2. 不展示接话、建议话术、亲子沟通建议；
3. 新生成日报后端不再要求或填充 tonight_parent_bridge 这类家长话术字段；
4. 孩子今天提到的内容是具体短点列表，不是 topic 标签；
5. 不展示 raw transcript 或儿童原文全文；
6. 四状态机保持：正在整理 / 已生成 / 内容不足 / 失败。
```
