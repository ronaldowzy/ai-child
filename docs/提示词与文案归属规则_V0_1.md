# 提示词与文案归属规则 v0.1

Project: `ai-child` / `ronaldowzy/ai-child`

用途：明确提示词、家长日报、儿童端文案、家长端文案和小白狐人格表达的归属边界，避免代码执行会话自行“优化”产品语言。

---

## 1. 核心规则

主控会话拥有所有产品敏感语言的设计权。

包括：

```text
1. system prompt；
2. scene prompt；
3. 小白狐儿童端行为指令；
4. 家长日报 prompt；
5. 家长日报好/坏示例；
6. deterministic fallback 文案；
7. 儿童端 UI 文案；
8. 家长端 UI 文案；
9. 安全、隐私、学习、健康使用相关文案；
10. 小白狐人格、语气和视觉状态说明；
11. 任何定义上述内容的任务文档。
```

代码执行会话只拥有实现权。

---

## 2. 代码执行会话可以做什么

```text
1. 按主控会话给出的精确文案复制到代码；
2. 把文案接入 prompt、schema、UI 或测试；
3. 为适配语言语法调整转义、缩进、换行、字符串拼接；
4. 报告编译、测试或运行问题，请主控会话修正文案；
5. 写测试固定关键短语、禁用短语和输出边界。
```

---

## 3. 代码执行会话不能做什么

```text
1. 自行发明 prompt 或产品文案；
2. 自行把文案“改自然”“加强”“润色”；
3. 重写家长日报示例；
4. 自行判断中国家长自然会怎么说；
5. 自行决定心理、教育、亲子边界；
6. 未经确认用意思相近的句子替换主控给出的原句；
7. 加入老师评语、客服语、工程日志、监控感文案；
8. 加入积分、连续打卡、排行、任务、依赖、挽留、FOMO 或留存钩子。
```

---

## 4. 未来涉及语言的任务必须怎么写

任何触及 prompt 或产品文案的任务，必须满足以下三种之一：

```text
A. 主控会话直接给出精确替换文本；
B. 任务链接到主控维护的 master-copy 文档；
C. 明确标记：等待主控会话提供文案前，该任务阻塞。
```

以下表达不能作为执行指令：

```text
“优化 prompt”
“让它更自然”
“加强家长日报 prompt”
“按你理解润色一下”
```

除非同时给出精确文本或 master-copy。

---

## 5. 家长日报特别规则

家长日报是高敏感产品语言。

主控会话必须提供：

```text
1. 模型 system prompt 或 master-copy prompt 文件；
2. 好输出示例；
3. 坏输出示例；
4. deterministic fallback 文案；
5. 禁止出现的家长端表达；
6. visible-quality 测试预期。
```

家长日报不得变成：

```text
1. 聊天监控；
2. 老师评语；
3. 心理诊断；
4. 行为评分；
5. 使用统计；
6. 引导家长盘问孩子和小白狐聊了什么的清单。
```

---

## 6. 审查规则

凡是代码执行会话提交了 prompt 或文案变更，主控会话审查时必须拒绝以下情况：

```text
1. 文案不是任务或 master-copy 中出现过的精确文本；
2. 文案语义相近但改变产品意图；
3. 文案听起来像教师、客服、工程日志、监控报告或游戏留存机制；
4. 家长日报暴露孩子和小白狐的私密互动细节；
5. 儿童端文案出现 backend、provider、ASR、error、policy、prompt 等内部术语。
```

---

## 7. 固定指令

```text
不再要求代码执行会话设计产品语言。
除非主控会话提供或明确批准最终精确文本，否则任何产品文案变更都不算完成。
```
