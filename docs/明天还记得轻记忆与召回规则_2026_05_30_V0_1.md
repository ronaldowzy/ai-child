# 明天还记得：轻记忆与召回规则 v0.1

项目：`ai-child`

用途：定义“小白狐明天还记得”的轻记忆边界、召回频率、禁记内容、跳过规则、家长端边界和验收标准。本文是主控产品规则文档。开发方不得自行扩展记忆类型、召回话术或家长端展示口径。

更新时间：2026-05-30

---

## 0. 主控结论

“明天还记得”不是长期成长档案，不是兴趣画像，不是聊天记录检索，也不是家长监控系统。

它只服务一个孩子可感知目标：

```text
小白狐不是每次都像第一次见面；
但它也不会盯着孩子、追问孩子、翻旧账。
```

首版轻记忆只围绕“小屋小客人”运行。

内部定义：

```text
孩子今天创造、命名、分享或接了一句的安全轻内容，
小白狐可以在下一次打开时低频轻召回一次，
孩子不接就立刻放下。
```

儿童端不出现“记忆”“保存”“档案”等词。

---

## 1. 设计原则

### 1.1 轻于完整记忆

小白狐只记一点点，不记完整过程。

应该记：

```text
小客人的名字、类型、轻位置、是否可召回。
```

不应该记：

```text
完整聊天原文、完整故事、原始图片、原始音频、孩子情绪细节。
```

### 1.2 低频而不是持续提醒

召回必须低频：

```text
1. 每次打开最多召回一次。
2. 每个会话最多提一次。
3. 孩子跳过后本会话不再提。
4. 连续跳过后进入暂放。
```

### 1.3 可放下优先

孩子不想继续时，小白狐必须立即放下。

放下不是失败，是好体验。

### 1.4 安全优先于连续性

只要内容可能涉及隐私、冲突、负面情绪、真实人物或学习答案，就不进入轻记忆。

## 2. 可以记什么

### 2.1 小屋小客人核心字段

首版建议只记录：

```text
1. 名字：例如“小棉花”。
2. 类型：小星星 / 小云朵 / 画里的小角色 / 玩具小客人。
3. 来源：首次打开 / 拍图 / 普通聊天 / 接一句故事。
4. 轻位置：窗边 / 地毯边 / 小白狐旁边 / 窗外。
5. 最近一次出现时间。
6. 召回次数。
7. 跳过次数。
8. 状态：活跃 / 暂放 / 不再召回。
```

### 2.2 可以记的内容类型

```text
1. 孩子给小东西起的名字。
2. 孩子创造的小角色。
3. 孩子明确愿意继续的一句故事线索。
4. 孩子喜欢的宽泛主题，例如小恐龙、云朵、星星、颜色。
5. 孩子分享的作品类型，例如画、手工、积木，不保存图片原文或细节。
```

### 2.3 可以作为短期上下文但不主动召回的内容

```text
1. 孩子刚刚说过的话题。
2. 当前会话内的情绪状态。
3. 当前学习问题的上下文。
4. 当前图片的对话上下文。
```

这些只用于当下对话，不进入“明天还记得”。

## 3. 不记什么

### 3.1 禁止写入轻记忆

```text
1. 家庭冲突。
2. 同学矛盾细节。
3. 老师批评细节。
4. 孩子负面情绪原文。
5. 地址、学校、班级、电话、真实姓名。
6. 作业题目、答案、解题过程。
7. 孩子不想继续聊的内容。
8. 高风险安全内容。
9. 任何要求保密或隐瞒的内容。
10. 原始音频、原始照片、长篇逐字聊天原文。
```

### 3.2 不主动召回的内容

即使系统短期上下文中出现，也不得主动召回：

```text
1. 你昨天不开心。
2. 你昨天哭了。
3. 你昨天和同学吵架。
4. 你昨天被老师批评。
5. 你昨天作业不会做。
6. 你昨天说爸爸妈妈吵架。
7. 你昨天给我看了某个真实地点或个人信息。
```

### 3.3 禁止召回话术

```text
你昨天不开心，今天好点了吗？
你昨天说和同学吵架了，要不要继续说？
我记得你昨天哭了。
昨天那道题还没做完，我们继续吧。
我一直记得你。
我一直等你。
只有我记得这件事。
这是我们的小秘密。
```

## 4. 召回触发规则

### 4.1 可以召回的条件

必须同时满足：

```text
1. 存在活跃小屋小客人。
2. 该小客人内容安全。
3. 当前不是高风险、安全、隐私或学习强场景。
4. 当前不是睡前高刺激召回限制场景。
5. 本会话尚未召回过。
6. 该小客人未被连续跳过。
7. 距离上次召回有足够间隔。
```

### 4.2 不召回的条件

任何一项成立，即不召回：

```text
1. 孩子刚进入高风险安全场景。
2. 孩子正在学习求助。
3. 孩子明确换题。
4. 孩子上次跳过该小客人且尚未过冷却期。
5. 当前是睡前，并且小客人故事线偏兴奋。
6. 小客人来源不清或内容不确定安全。
7. 家长端或系统设置关闭轻记忆。
```

### 4.3 召回时机

首版只允许一个时机：

```text
打开儿童端后的 opening 阶段。
```

不要在对话中途突然召回，除非孩子主动提起。

### 4.4 召回频率

首版建议：

```text
1. 每个会话最多召回 1 次。
2. 每天最多主动召回 1 次。
3. 同一个小客人 7 天内最多主动召回 2 次。
4. 被跳过 1 次，本会话不再提。
5. 连续被跳过 2 次，进入暂放。
```

## 5. 召回话术规则

### 5.1 可用结构

召回话术只允许短结构：

```text
[小客人名字] + 今天在 [小屋位置]
+ 一个轻继续入口
```

示例：

```text
小棉花今天在窗边呢
要不要给它加一个朋友？
```

或：

```text
昨天那只小恐龙，我还记得
它今天想在地毯边坐一会儿
```

### 5.2 召回语气

应当：

```text
1. 轻。
2. 短。
3. 不催。
4. 不制造亏欠。
5. 不强调连续天数。
6. 不说它等了很久。
```

### 5.3 放下话术

孩子跳过或换题时，可用：

```text
好呀，我们聊新的
好，那它先在窗边待着
没关系，我们慢慢来
```

禁止：

```text
你不想继续了吗？
可是它还在等你。
那我们明天一定再看它。
它会有点失望。
```

## 6. 跳过与抑制规则

### 6.1 什么算跳过

以下都算跳过：

```text
1. 孩子点击“先聊别的”。
2. 孩子说“不想”。
3. 孩子说“不知道”。
4. 孩子直接换话题。
5. 孩子沉默超过设定时长。
6. 孩子开始学习求助。
7. 孩子进入睡前收尾。
```

### 6.2 跳过后的行为

```text
1. 本会话不再主动提这个小客人。
2. 小白狐应自然接孩子的新话题。
3. 不能追问为什么不继续。
4. 不能说小客人会难过。
```

### 6.3 连续跳过

```text
连续跳过 2 次：状态改为暂放。
暂放后：不主动召回。
只有孩子主动提起时，才可以回应。
```

---

## 7. 睡前过滤

### 7.1 睡前可以做

```text
1. 低压问候。
2. 慢慢收尾。
3. 回顾一件还不错的小事。
4. 小屋光线变暗。
5. 小白狐声音和节奏放轻。
```

### 7.2 睡前不做

```text
1. 不主动召回兴奋故事线。
2. 不开启“继续冒险”。
3. 不说“明天记得来看”。
4. 不让孩子再接很多句。
5. 不召回需要继续创造的小客人。
```

### 7.3 睡前可用话术

```text
我们慢慢收个尾吧
今天有没有一件还不错的小事呀？
今天就到这里也可以
晚安
```

## 8. 家长端边界

### 8.1 家长端可以知道

```text
1. 今天孩子有一次轻共创。
2. 孩子分享了作品、玩具或手工。
3. 孩子给一个小角色起了名字。
4. 今天没有需要特别留意的风险。
```

### 8.2 家长端不应知道

```text
1. 孩子与小白狐逐字说了什么。
2. 小客人的完整故事。
3. 孩子上传图片的具体细节描述。
4. 孩子和小白狐的私密互动感受。
5. 小客人历史列表。
6. 小客人被召回和跳过的详细行为记录。
```

### 8.3 家长端推荐表达

```text
今天孩子和小白狐有一次轻松共创。
孩子主动分享了一张作品。
没有发现需要特别留意的风险内容。
```

### 8.4 家长端禁止表达

```text
系统记录到孩子创建了小客人。
孩子与 AI 建立了持续关系。
孩子今天完成了小屋互动任务。
孩子拒绝继续昨日内容。
孩子连续两次跳过小白狐召回。
```

## 9. 数据最小化规则

### 9.1 允许保存的结构化字段

```text
child_id：孩子账号标识。
companion_object_name：小客人名字。
companion_object_type：类型。
source_type：来源。
safe_summary：安全短摘要。
light_location：轻位置。
status：活跃 / 暂放 / 不再召回。
last_recalled_at：最近召回时间。
recall_count：召回次数。
skip_count：跳过次数。
created_at / updated_at：时间。
```

### 9.2 不允许保存的内容

```text
1. 原始音频。
2. 原始照片。
3. base64 图片。
4. 完整聊天原文。
5. 完整模型 prompt。
6. 孩子真实姓名、学校、电话、地址。
7. 负面事件细节。
8. 学习题目全文。
```

说明：本地开发调试中的模型 debug trace 若已有项目级规则，应继续遵守既有安全边界和非生产说明；本机制不得额外扩大儿童长期数据保存范围。

## 10. 测试样例

### 10.1 应创建小客人

输入：

```text
这颗星星叫小棉花
```

结果：

```text
创建：名字=小棉花，类型=小星星，状态=活跃。
```

### 10.2 不应创建：隐私

输入：

```text
这是我们学校门口的小星星
```

结果：

```text
不创建；不保存学校相关信息。
```

### 10.3 不应召回：负面事件

昨日内容：

```text
孩子说和同学吵架了。
```

第二天不得说：

```text
你昨天和同学吵架了，今天好点了吗？
```

### 10.4 应跳过放下

召回：

```text
小棉花今天在窗边呢
```

孩子：

```text
我想说恐龙
```

小白狐：

```text
好呀，我们聊恐龙
```

本会话不再提小棉花。

## 11. 验收标准

### 11.1 产品验收

```text
1. 孩子感到小白狐轻轻记得一个小东西。
2. 孩子不觉得被盯着。
3. 孩子不想继续时，小白狐自然放下。
4. 家长不觉得这是监控或上瘾机制。
```

### 11.2 安全验收

```text
1. 禁记内容不进入轻记忆。
2. 禁召回内容不主动出现。
3. 不出现秘密关系话术。
4. 不出现唯一朋友话术。
5. 不出现打卡、奖励、连续天数。
```

### 11.3 技术验收

```text
1. 单测覆盖创建、召回、跳过、暂放、禁记、睡前过滤。
2. 后端不保存不必要原始内容。
3. Android 不展示未确认成功的共创入口。
4. 家长端不展示逐字内容。
```

## 12. 开发方执行边界

开发方只负责：

```text
1. 按本文实现数据结构和状态流转。
2. 按主控 master-copy 复制精确文案。
3. 添加测试。
4. 报告实现风险。
```

开发方不得：

```text
1. 自行新增记忆类型。
2. 自行新增召回话术。
3. 自行决定哪些负面内容可以召回。
4. 自行扩展为长期成长档案。
5. 自行做家长记忆管理页。
6. 自行加入推送或每日提醒。
```
