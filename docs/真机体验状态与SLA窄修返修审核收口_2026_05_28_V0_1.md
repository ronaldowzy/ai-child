## 1. 已确认完成点

### 1.1 WaitingForChild 不再是 recording

`VoiceUiState.isRecording` 已收敛为：

```kotlin
val isRecording: Boolean
    get() = inputMode == VoiceInputMode.Listening
```

这意味着：

```text
1. WaitingForChild 不再被当成真实录音；
2. Listening 才表示真实录音；
3. InputBar 中 voice.isRecording 为 false 时，点击主按钮会进入 startVoiceRecording；
4. 低压等待态不再触发 stopVoiceRecordingAndUpload。
```

### 1.2 WaitingForChild 文案已改为低压等待

当前 WaitingForChild 状态显示：

```text
想说的时候再说。
```

对应主按钮：

```text
按一下开始说
```

这符合当前产品定义：小白狐说完后可以轻轻等，但不要求孩子直接说，也不暗示正在录音。

### 1.3 NATURAL_WAITING_ENABLED 取代 NATURAL_SPEAKING_ENABLED

配置已改为：

```kotlin
const val NATURAL_WAITING_ENABLED = true
const val NATURAL_WAITING_TIMEOUT_MS = 5_000L
```

这比 `NATURAL_SPEAKING_ENABLED` 更准确，避免开发方继续把低压等待误解成自动开麦或免按键对话。

### 1.4 旧自动开麦逻辑已改为视觉等待

旧 `autoStartRecordingAfterTts` 已改为 `showWaitingForChildAfterTts`。

当前逻辑只做：

```text
1. 设置 childTurnPhaseHint = WaitingChild；
2. 设置 voice.inputMode = WaitingForChild；
3. 设置小白狐低压等待状态；
4. 启动视觉等待超时收回。
```

当前逻辑不再做：

```text
1. 不调用 SpeechInputController.startRecording；
2. 不设置 onSilenceDetected；
3. 不启动 AndroidWavAudioRecorder；
4. 不靠环境声音自动结束并上传。
```

### 1.5 视觉等待可自然收回

等待超时后会进入：

```text
想说的时候再说。
```

随后调用 `cancelVoiceInput()` 回到普通状态。方向符合“孩子不说也没关系”。

### 1.6 旧通过能力未被回滚

本次返修没有回滚上一提交中已经通过的方向：

```text
1. latency trace；
2. opening 本地降级；
3. opening 晚到丢弃；
4. 文字先于 TTS；
5. TTS 首段 audio_ready 先下发；
6. 图片失败态：“这张图还没给小白狐看到”；
7. 图片失败低压按钮：“我们可以再试一次”“先不看也可以”。
```

## 3. 当前产品口径定稿

从本收口文档起，当前阶段统一使用以下口径：

```text
说完后的低压等待态
```

定义：

```text
小白狐说完后，短暂保持“我在听 / 想说的时候再说 / 不说也没关系”的陪伴状态；
手机不自动录音；
麦克风不自动打开；
孩子点击或按住主按钮后才进入真实录音；
真实录音时按钮才显示“说完了”。
```

当前阶段不要再称为：

```text
自然接话；
免按键对话；
自动续听；
自然说话能力。
```

这些表述会误导开发端或测试方，以为小白狐说完后要自动打开麦克风。

## 4. 真机复测重点

下一步不是继续开发，而是用真机复测确认体感。

必须检查：

```text
1. 小白狐说完后，按钮是否显示“按一下开始说”，而不是“说完了”；
2. 小白狐说完后，状态是否像“低压等待”，不是录音；
3. 等待期间系统是否没有录音权限提示、没有录音波形、没有自动上传；
4. 孩子点击按钮后，才进入“正在听 / 说完了”；
5. 等待 5 秒左右后，是否自然收回；
6. 睡前、学习、安全、隐私等场景是否不进入等待；
7. 停一下是否仍能停止迟到语音；
8. 图片失败态和 opening 降级是否保持正常；
9. trace 是否能显示从录音结束到文字、首段声音的耗时。
```

## 5. 后续边界

本轮收口后，不要继续围绕“低压等待态”追加功能。

只允许：

```text
1. 真机复测；
2. 明显 bug 修复；
3. 轻微文案调整；
4. 等待时长微调；
5. trace 观察和性能定位。
```

不允许：

```text
1. 恢复自动录音式自然接话；
2. 做免按键对话；
3. 做常开麦克风；
4. 做环境声触发录音；
5. 围绕这件事继续堆 prompt；
6. 新增任务、奖励、作品库、故事库；
7. 新一轮界面大改。
```

---

## 6. 主控结论

P0 窄修的“等待 / 录音状态拆分”返修可以收口。

下一步应先真机复测：

```text
低压等待是否真正不再让孩子困惑；
按钮是否不再提前显示“说完了”；
等待是否自然、短、无压力；
文字/TTS/图片失败态是否保持上一轮改善。
```

真机复测通过后，再考虑进入 P1：家长日报状态机与展示去重复。
