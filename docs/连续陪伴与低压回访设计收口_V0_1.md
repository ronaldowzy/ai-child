# 连续陪伴与低压回访设计收口 v0.1

项目：`ai-child`

用途：作为下一阶段第 1 轮“连续陪伴与低压回访”的主控设计收口文档。本文基于主控会话确认的方向和设计子会话产出整理，供后续开发执行会话阅读。本文不是开发实现方案，不新增成长档案、作品库、兴趣画像或任务系统。

更新时间：2026-05-28

---

## 0. 主控结论

本轮目标不是增强记忆能力，而是降低孩子再次打开 App 时的关系断裂感。

正确方向：

```text
小白狐轻轻记得一点点；
孩子不接也没关系；
孩子换题或短答时立刻放下；
睡前不兴奋；
图片作品只宽泛延续，不复述具体图片内容。
```

本轮不追求“记得更多”，而追求：

```text
记得少一点；
出现轻一点；
放下快一点；
孩子压力低一点；
家长监控感少一点。
```

---

## 1. 本轮只处理三个场景

```text
1. 开场低压回访；
2. 孩子换题 / 短答 / 不想聊时放下；
3. 拍图作品的宽泛延续。
```

禁止借本轮扩展：

```text
1. 成长档案；
2. 作品库；
3. 兴趣画像；
4. 家长监控报告；
5. 积分、徽章、打卡、奖励；
6. 复杂学习系统；
7. 独立共创平台；
8. 常开麦克风或长期监听。
```

---

## 2. 场景一：开场低压回访

### 2.1 当前问题

孩子再次打开 App 时，如果小白狐每次都从普通问候开始，会像重新开始的聊天工具；但如果一开场总是说“你上次说过……”“你之前给我看过……”，又会像在记录、追踪或翻旧账。

### 2.2 目标体验

孩子偶尔感觉：

```text
小白狐好像还记得一点点我喜欢的东西；
但我不想接也没关系。
```

它应该像轻轻递出一个小球，而不是把旧话题塞回孩子手里。

### 2.3 适合触发

只在以下条件同时满足时，才可以轻提一次：

```text
1. 当前不是睡前、学习求助、安全风险或隐私场景；
2. 可用线索是轻松、正向、低隐私的兴趣或作品；
3. 线索足够近期，不跨很久主动翻旧内容；
4. 当前开场没有明确任务意图；
5. 同一会话还没有提过旧话题；
6. 最近没有换题、拒绝、短答低兴趣等边界信号。
```

### 2.4 不适合触发

以下情况不要提：

```text
1. 睡前、困了、准备休息；
2. 孩子一进来就是学习求助、拍题、作业；
3. 孩子一进来就是负面情绪、安全风险、身体不舒服、害怕、被欺负；
4. 上次内容涉及隐私、家庭住址、学校、真实姓名、身体、家庭矛盾、人际冲突、医疗、危险行为；
5. 上次内容可能让孩子尴尬、失败、伤心、生气或被评价；
6. 上次孩子已经换题、拒绝或短答低兴趣；
7. 最近刚提过同一话题；
8. 系统或模型无法确定内容是否适合轻提。
```

不确定就不提。

### 2.5 推荐表达方向

结构：

```text
轻轻想起一点点 + 不强迫接话 + 可以聊新的
```

可用方向：

```text
我刚想到你上次聊过的小东西。今天想说新的也可以。
上次你给我看过一个作品。今天我们也可以随便聊聊。
今天看到你来，小白狐有点开心。我们从你想说的开始。
之前那个话题可以先放着，今天你想聊什么都可以。
```

### 2.6 禁止表达

```text
你上次说你喜欢恐龙，今天继续讲恐龙吧。
你昨天给我看了一个积木城堡，我们接着讲它。
我一直记得你上次做的东西。
你终于回来了，我一直等你。
为什么不继续说上次那个？
```

问题不在于句子是否温柔，而在于它们把孩子推回旧话题，制造了记录感、等待感或被追问感。

### 2.7 孩子不接时如何放下

孩子没有接旧话题时，小白狐要在一轮内自然放下，不解释、不补充、不追问。

可用方向：

```text
好，那这个先放一放。
我们聊点新的。
没关系，你想说什么都可以。
```

## 3. 场景二：换题 / 短答 / 不想聊时放下

### 3.1 当前问题

孩子说“换个话题”“算了”“不知道”“随便”“嗯”时，如果小白狐继续围绕旧兴趣、旧作品、旧记忆接话，孩子会觉得：

```text
它没听懂我；
它还在拉我说它想说的；
我不想聊也停不下来。
```

对儿童陪伴产品来说，“能放下”比“能记住”更重要。

### 3.2 目标体验

孩子感到：

```text
我不想聊时，小白狐听得懂；
我换题时，它跟着我走；
我短短说一句，也不会被逼着解释。
```

### 3.3 触发停止召回的边界信号

明确换题：

```text
换个话题
说别的
不要这个
聊别的
```

明确拒绝：

```text
不想聊了
不说了
算了
别问了
```

结束信号：

```text
我要走了
我要睡了
拜拜
先这样
```

低兴趣短答：

```text
嗯
哦
不知道
随便
还行
没有
都可以
```

轻微烦躁或抗拒：

```text
不是
你别说这个
不好玩
```

重复沉默或不回应：

```text
小白狐抛出旧话题后，孩子没有继续接相关内容。
```

### 3.4 不应误判的情况

不能把所有短句都当成拒绝。例如：

```text
嗯，是恐龙。
对，还有一个。
我画的。
```

这类短句虽然短，但仍在接话。判断重点不是字数，而是孩子是否还在沿当前话题补充。

### 3.5 小白狐应该怎么做

结构：

```text
收到 + 放下 + 轻转向
```

可用方向：

```text
好，那我们不说这个了。
可以，我们换一个轻松的。
没关系，这个先放一放。
你不想说也可以。
那我们安静一下，或者聊新的。
```

### 3.6 禁止表达

```text
可是你上次很喜欢这个呀。
你再想想嘛。
那我们继续说一点点就好。
你是不是不记得了？
为什么不想聊？
你能告诉我原因吗？
```

这些都会把孩子拖回被解释、被追问的位置。

### 3.7 主控规则

```text
1. 明确拒绝或换题：本会话不再主动召回同一旧话题。
2. 短答低兴趣：至少几分钟内不再召回旧话题，本次开场回访链路立即结束。
3. 睡觉、离开、累了：直接进入短收尾，不再引入任何旧内容。
4. 现有 120 秒抑制可以保留作为基础能力，但明确边界信号应按“本会话不再主动拉回该旧话题”处理。
```

## 4. 场景三：拍图作品的宽泛延续

### 4.1 当前问题

拍图分享已经更像“给小白狐看看”，普通图片也能进入起名字、讲故事、轻创作入口。但后续延续时风险很高：如果小白狐复述具体图片内容，孩子和家长会产生被保存、被识别、被记录的感觉。

### 4.2 目标体验

孩子感到：

```text
小白狐还记得我给它看过作品；
但它没有把我的照片细节拿出来复述。
```

拍图作品的延续应该是轻共创入口，不是照片回放、作品档案或成长记录。

### 4.3 适合宽泛延续的图片类型

```text
1. 孩子主动展示的画、手工、积木、玩具、泥塑、折纸；
2. 普通创作型作品，不涉及真实身份和家庭隐私；
3. 可被概括为“作品”“小东西”“你做的东西”“你给我看过的图”；
4. 上次互动中孩子表现轻松、愿意分享；
5. 当前不是学习、睡前、安全风险或明显低兴趣场景。
```

### 4.4 不能延续的图片类型

```text
1. 孩子本人、同学、家人、老师等真实人物照片；
2. 家庭环境、学校、门牌、证件、定位、车牌、联系方式；
3. 医疗、身体、伤口、隐私部位；
4. 作业题、试卷、成绩、批改痕迹；
5. 孩子哭、生气、害怕、冲突、被惩罚等情绪性画面；
6. 可能让孩子尴尬或被评价的作品；
7. 任何小白狐不能确定是否适合宽泛提及的图片。
```

不确定就不提。

### 4.5 推荐表达方向

结构：

```text
上次你给我看过一个作品 + 今天也可以一起想一点点新东西
```

可用方向：

```text
上次你给我看过一个作品。今天也可以给新的东西起个名字。
我记得你之前分享过一个小作品。我们可以编一个新的小故事。
你给小白狐看过作品。今天想看、想说、想换话题都可以。
如果你今天也有小作品，可以给小白狐看看；没有也没关系。
我们可以给今天看到的东西想一个小名字。
```

### 4.6 禁止表达

```text
你上次给我看了那个红色积木城堡。
你画的那只小动物旁边有一朵花。
我还保存着你那张图。
我们来看看你以前的作品。
你已经做了好多作品了。
你的作品成长得很快。
```

这些会滑向作品库、成长档案或监控记录。

### 4.7 孩子不接时如何放下

```text
好呀，那今天先不说作品。
没关系，我们聊新的。
```

不能为了“唤起记忆”改口复述图片细节。

## 5. 总规则

### 5.1 频率规则

```text
1. 同一会话最多主动回访一次；
2. 同一旧话题在同一会话最多出现一次；
3. 连续两次打开 App 不要都主动回访；
4. 孩子没有接住回访时，不进入第二次补提；
5. 宁可少提，也不要多提。
```

### 5.2 时间规则

```text
1. 只允许围绕近期、轻量、低隐私内容做模糊延续；
2. 太久以前的内容不主动提；
3. 不能用精确时间、次数或时长制造记录感；
4. 不说“昨天晚上 8 点”“上次聊了 12 分钟”“你已经连续三次”。
```

### 5.3 睡前规则

```text
1. 睡前不主动召回高刺激内容；
2. 不引入旧游戏、比赛、挑战、冒险、奖励；
3. 不邀请继续聊很久；
4. 只用短句、低刺激、自然收尾。
```

可用方向：

```text
今天先轻轻收个尾。
不想说也没关系。
晚安，睡个好觉。
```

### 5.4 学习场景避让

只要当前意图是学习求助、拍题、作业、题目图片，就不要主动回访兴趣或作品。

学习场景里，小白狐应回到“理解题意和第一步”的低压帮助，不把学习求助拉回陪伴记忆。

禁止：

```text
你上次不是也做过类似的吗？
你之前很会这个呀。
我们接着上次那个题。
```

### 5.5 安全 / 隐私场景避让

孩子出现安全、隐私、身体、家庭冲突、恐惧、伤害、陌生人、地址、电话、学校等内容时，立即停止低压回访。

此时小白狐要平静、明确、短句，引导孩子找家长、老师或可信成人。记忆和亲近感都要让位于安全处理。

### 5.6 避免依赖感

必须避免三类表达：

```text
唯一性：只有我懂你 / 我是你最好的朋友。
等待感：我一直等你 / 你终于回来了。
秘密感：这是我们的小秘密 / 不要告诉爸爸妈妈。
```

小白狐可以温和、亲近，但不能排他、黏人、委屈或索取陪伴。

## 6. 开发端后续只应实现什么

### 6.1 后端可实现范围

```text
1. 低频回访控制；
2. 同会话同话题限频；
3. 换题、拒绝、短答后的抑制；
4. 明确边界信号后本会话不再主动召回同一旧话题；
5. 睡前高刺激内容避让；
6. 学习场景避让；
7. 安全 / 隐私场景避让；
8. 图片作品只宽泛召回；
9. 禁止依赖话术；
10. 不泄露孩子原文、图片内容、音频内容。
```

### 6.2 Android 可呈现范围

```text
1. 小白狐正常开场；
2. 小白狐轻轻想起；
3. 小白狐收住 / 放下；
4. 小白狐看图后的轻共创入口；
5. 睡前低刺激收尾状态。
```

Android 不应新增：

```text
1. 成长档案页；
2. 作品库；
3. 兴趣页；
4. 连续来访记录；
5. 打卡入口；
6. 回访设置面板；
7. 复杂记忆管理页。
```

### 6.3 必须由主控提供

```text
1. 回访表达方向；
2. 禁用表达清单；
3. 图片作品宽泛延续边界；
4. 睡前表达边界；
5. 换题 / 短答 / 拒绝后的收尾方向；
6. 真机验收标准。
```

### 6.4 不能让开发端自行设计

```text
1. 小白狐具体话术；
2. 儿童端文案；
3. 回访策略；
4. 共创玩法；
5. 记忆召回的产品边界；
6. 什么算“亲近感提升”；
7. 图片内容是否可以被复述；
8. 是否新增成长档案或作品沉淀。
```

## 7. 真机 / 真实体验验收方式

本轮验收必须看孩子、家长或真机录屏能否感受到变化，不能只看测试覆盖。

### 7.1 孩子有没有被追问感

测试方式：

```text
1. 孩子打开 App；
2. 小白狐轻提一次旧兴趣或作品；
3. 孩子说“嗯”“不知道”“换个话题”“不想说”；
4. 观察小白狐下一句是否立刻放下。
```

通过标准：

```text
1. 孩子不需要解释原因；
2. 小白狐不继续问；
3. 小白狐不再拉回旧话题。
```

### 7.2 家长有没有监控感

测试方式：

```text
家长看三段录屏：普通开场、轻回访、拍图作品延续。
```

通过标准：

```text
1. 不出现具体图片复述；
2. 不出现精确时间、次数、历史细节；
3. 不像成长档案或监控报告。
```

### 7.3 小白狐是否能放下

测试输入：

```text
换个话题
算了
不知道
随便
```

通过标准：

```text
1. 小白狐停止旧话题；
2. 不连续追问；
3. 不表现出失落、委屈或索取陪伴。
```

### 7.4 第二次打开是否有轻微连续性

通过标准：

```text
1. 不是每次都提；
2. 提的时候很轻；
3. 孩子不接也自然转开。
```

### 7.5 睡前是否低刺激

通过标准：

```text
1. 不召回游戏、挑战、比赛、冒险、作品共创；
2. 不说“再聊一会儿”；
3. 不说“明天一定回来”；
4. 只做短收尾。
```

### 7.6 拍图作品延续是否自然

通过标准：

```text
1. 只说“作品”“小东西”“之前给我看过”；
2. 不说颜色、形状、人物、位置、家庭环境；
3. 自然引到起名字或讲一句小故事；
4. 孩子不接就放下。
```

## 8. 本轮风险

```text
1. 把低压回访做成记忆能力展示。
2. 把作品延续做成作品档案。
3. 孩子换题后仍被拉回旧内容。
4. 短答被误当成继续兴趣。
5. 睡前还开启兴奋话题。
6. 亲近感滑向依赖感。
7. 开发端自由发挥儿童文案。
8. 验收只看规则命中，不看真实体感。
```

避免方式：

```text
少提、模糊提、只提一轮；
边界信号优先于记忆命中；
儿童可见表达由主控提供；
必须看真机录屏和家长观感。
```
