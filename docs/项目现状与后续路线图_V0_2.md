# 项目现状与后续路线图 v0.3

项目：`ai-child`

用途：作为当前阶段主控会话、执行会话和后续规划的统一事实源。当前路线图已经从“继续扩展内部能力”调整为“用五轮以内修改收尾孩子可感知体验”。

更新时间：2026-05-27

---

## 0. 当前一句话结论

`ai-child` 已经完成基础功能和一轮全局提示语优化。下一阶段不再优先堆内部治理和复杂规划，而是优先提升孩子直接感受到的体验：看见小白狐、愿意说话、等待不长、愿意拍图分享、能感到小白狐轻轻记得自己。

当前产品形态：

```text
儿童默认用语音和小白狐自由交流；
小白狐是孩子首先感知到的角色，而不只是聊天框旁边的图；
后端统一完成语音识别、模型、安全、图片、记忆、朗读和家长日报；
安全、隐私、学习和睡前只是护栏，不应压过陪伴体验；
家长端看自然日报和现实接话建议，不看逐字聊天记录。
```

## 1. 当前优先级调整

根据近期测试和产品判断，以下方向不作为当前五轮主线：

```text
1. 反复完整功能测试：已经测过多轮，除非围绕新改动，不再为了测试而测试。
2. 家长日报持续打磨：以后真实使用中觉得不好再局部调整。
3. 学习深度优化：先后置，只保持不直接给答案和引导审题第一步。
4. 合规和家长治理重设计：家庭内测阶段不做重型治理，正式上线前再补。
5. 多孩子、多家长、长期成长档案：后置。
6. 继续大范围提示语优化：已完成一轮，后续只做真实问题驱动的局部修改。
```

当前真正要做的是孩子体验收尾，详见：

```text

docs/孩子体验五轮执行提示词_V0_1.md
```

## 2. 五轮内收尾目标

### 第一轮：界面和小白狐形象丰富

当前状态：小白狐资源和状态机已有基础，但界面仍偏聊天产品，小白狐不是第一视觉中心。

优化后：孩子打开页面首先感到“小白狐在这里”，听、想、说、拍图、出错、困了、开心等状态清楚但不刺激。

验收：打开儿童页 3 秒内小白狐存在感明确；录音、思考、说话、图片处理中状态明显；横屏不挤压主要内容；不加入奖励、打卡、积分。

### 第二轮：语音朗读延迟收敛

当前状态：小白狐语音链路已接入，但等待感仍强。说话慢会让孩子出戏。

优化后：文字先出现，小白狐及时进入思考/说话状态，第一段语音尽快播放，后续分段不乱序、不混播。

验收：能记录文字出现、语音请求、第一段语音可播、安卓开始播放时间；停止朗读后旧音频不继续；不使用系统音色冒充小白狐。

### 第三轮：自然说话机制

当前状态：儿童默认语音优先，但仍主要是点击说话。自动识别说话可以做，但必须等语音延迟改善后。

优化后：小白狐说完后短时间等待孩子自然接话；孩子开口时旧语音停止；不说话时自动收回等待；可关闭退回点击说话。

验收：安静环境、普通噪声、孩子停顿、孩子打断小白狐说话都可验证；不长期开麦；文案无技术词。

### 第四轮：拍图分享体验

当前状态：拍图上传和图片理解已接入，但还不够像孩子表达和创作入口。

优化后：孩子拍画、玩具、积木、书页、自然物，小白狐能回应一个安全具体细节，并给起名字、讲故事、说发生了什么等轻入口。只有明确作业求助时才进入学习分支。

验收：普通图片不默认作业；有图片上下文时不说看不到；看不清时承认；多数测试图片能给一个安全具体细节或自然表达入口。

### 第五轮：轻量记忆和连续陪伴

当前状态：已有记忆和兴趣种子，但还没有形成孩子可感知的“轻轻记得”。

优化后：小白狐偶尔记得孩子近期兴趣或展示过的小东西，但不追踪、不施压、不贴标签、不反复拉旧话题。

验收：同一兴趣不会高频重复；孩子拒绝或换题后不继续拉回；睡前不召回兴奋话题；不出现“我一直等你”“只有我懂你”等依赖表达。

## 3. 可并行方式

可以多会话并行，但必须按依赖关系。

可并行：

```text
1. 第一轮：安卓界面会话 + 小白狐资源状态会话。
2. 第二轮：后端语音延迟会话 + 安卓播放队列会话。
3. 第四轮：安卓拍图体验会话 + 后端图片回应会话。
```

不可提前：

```text
1. 第三轮自然说话机制必须等第二轮语音等待感改善后再做。
2. 第五轮轻量记忆最好等第一轮和第二轮稳定后再做。
3. 学习深度优化和合规重设计不进入当前五轮。
```

## 4. 已经与早期规划不一致的地方

这些不是错误，而是项目在真实测试和产品判断后形成的正式修订。后续实现和文档都应以后者为准。

| 早期口径 | 当前口径 | 后续处理 |
|---|---|---|
| 面向单个 8 岁儿童 | 面向 5-10 岁儿童，先做中国大陆家庭内测 | 文案和界面按儿童理解能力设计 |
| “父亲”作为产品术语 | 统一改为“家长” | 文档和界面优先改“家长” |
| 安卓端可用系统语音作为小白狐声音 | 正式小白狐声音走后端音频链接；系统语音不是正式兜底 | 第二轮重点解决等待感 |
| 语音输入需要先确认再发送 | 儿童默认识别成功自动发送；确认只保留调试模式 | 第三轮再做自然说话机制 |
| 固定场景是主流程 | 默认自由交流；场景和时间只是上下文或护栏 | 不再新增固定任务场景 |
| 图片能力偏拍题 | “拍给小白狐看”是表达和创作入口 | 第四轮优化拍图分享 |
| 家长日报持续作为主优化点 | 真实使用中觉得不好再局部调 | 当前不再作为主线 |
| 合规和家长治理现在就做重 | 家庭内测先轻，正式上线前再补 | 当前后置 |
| 健康使用主要靠复杂治理 | 先靠体验边界和禁用机制，不做重型指标体系 | 当前不作为主线 |

## 5. 当前已经实现或基本实现

```text
1. 后端基础架构、对话接口、流式接口、健康检查、请求标识、结构化日志。
2. 安卓儿童主界面、家长设置、家长日报、登录和会话薄片。
3. 后端模型注册、提示语管理、安全引擎、意图识别、场景编排等核心边界。
4. 默认自由对话底座。
5. 安卓语音优先：点击录音、上传后端识别、识别成功自动发送。
6. 后端本地语音识别与外部识别兜底框架。
7. 后端小白狐语音、音频链接、安卓远程音频播放、停止和静音控制。
8. 流式文字、分段语音、安卓渐进显示和音频队列。
9. 小白狐状态图、状态解析、状态保持和防闪烁基础。
10. 系统相机/相册真实图片上传、图片理解和普通图片分享路径。
11. 家长日报模型优先，失败不展示规则日报冒充成功。
12. 基础账号：一个孩子一个账号，家长代创建和管理。
13. 一轮全局提示语和儿童端文案优化已经完成。
```

## 7. 当前推荐启动方式

现在可以启动第一轮并行：

```text
会话 A：安卓界面和小白狐主视觉。
会话 B：小白狐资源和状态丰富接入。
```

这两个会话完成后，再启动第二轮：

```text
会话 C：后端语音延迟收敛。
会话 D：安卓语音播放队列和打断。
```

如果第二轮语音等待感没有明显改善，不启动第三轮自然说话机制，继续处理语音等待问题。
