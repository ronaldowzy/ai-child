# 项目现状与后续路线图 v0.2

Project: `ai-child` / `ronaldowzy/ai-child`

用途：作为当前阶段主控会话、代码执行会话和后续规划的统一事实源，替代已经过期的英文交接文档和单轮过程任务文档。

更新时间：2026-05-27

---

## 0. 当前一句话结论

`ai-child` 已经从最初的“后端 + Android MVP 基础闭环”推进到家庭内测前的体验加固阶段。

当前产品形态已经不是早期的固定场景儿童聊天入口，而是：

```text
儿童默认 voice-first 与小白狐自由交流；
后端统一完成 ASR、模型、安全、图片、记忆、TTS、日报；
时间、家长寄语、图片、记忆作为上下文；
安全、隐私、学习、睡前作为护栏；
家长端看自然日报和现实接话建议，不看逐字聊天记录。
```

## 1. 已经与早期规划不一致的地方

这些不是错误，而是项目在真实测试和产品判断后形成的正式修订。后续实现和文档都应以后者为准。

| 早期规划 / 旧文档口径 | 当前修订后的口径 | 后续处理 |
|---|---|---|
| 面向单个 8 岁儿童 | 面向 5-10 岁儿童，先做中国大陆家庭内测 | 文案和分龄回复继续按 age band 设计 |
| “父亲”作为产品术语 | 统一改为“家长”，代码里历史 class 名可后续单独 refactor | 文档和 UI 文案优先改“家长” |
| Android 端可用系统 TTS 作为小白狐声音 | 正式音色走后端 MiMo VoiceClone audio_url；系统 TTS 不是正式小白狐 fallback | 继续排查 VoiceClone 延迟和失败态 |
| 语音输入需要 confirm-before-send | 儿童默认 ASR 成功后自动发送；确认面板只保留 DevSettings / 家长调试模式 | AGENTS 和旧设计文档要同步修正 |
| ASR 优先探索 MiMo | 当前 ASR 第一选择是后端本地 sherpa-onnx + SenseVoice-Small int8；MiMo 只做 fallback 且受授权/policy gate 控制 | 设备 QA 重点验证儿童语音、噪音、延迟 |
| 固定 after_school / homework / bedtime / photo 场景是主流程 | 默认 `conversation.open`；场景和时间只作为上下文或护栏 | 不再新增固定任务场景 |
| 图片能力偏“拍题 / OCR / homework” | 产品动作是“拍给小白狐看”；作业只是明确学习意图下的分支 | 继续做图片分享 v2 和创作脚手架 |
| 家长日报可由规则拼接生成 | 家长日报必须 model-first；程序只构造受控 evidence packet；v3 改成自然叙事段落 | 先验收 v3 真实输出质量 |
| 家长端可以看结构化观察 | 家长端看现实接话和低压力提醒，不展示 raw transcript，不做监控面板 | 父母报告继续防盘问、防诊断、防教师评语 |
| 家庭 MVP 可以接受 mock/fake 路径冒充能力 | 家庭内测运行时不得把 mock provider、mock 图片、mock 语音当产品能力 | 启动门禁和 QA 记录必须区分 PASS/FAIL/BLOCKED |
| 小白狐形象只是简单占位 | 小白狐是正式陪伴角色，使用预渲染 3D/WebP 状态图和轻量动画 | 后续做横屏双栏和状态表现 QA |
| 健康使用主要靠原则说明 | 已形成 Healthy Engagement 禁止项，但可观测指标还不完整 | 后续补 metrics 和 QA 场景 |

## 2. 当前已经实现 / 基本实现

```text
1. FastAPI 后端基础架构、conversation API、stream API、health/detail、request_id、结构化日志。
2. Android Compose 儿童主界面、家长设置、家长日报、登录/会话薄片。
3. 后端 ModelRegistry、PromptManager、SafetyEngine、IntentClassifier、SceneOrchestrator 等核心边界。
4. 默认 conversation.open 的 freedom-first 对话底座。
5. Android voice-first：点击录音、上传后端 ASR、ASR 成功自动发送，调试确认模式保留。
6. 后端本地 SenseVoice ASR provider 与 MiMo ASR fallback 框架。
7. 后端 MiMo VoiceClone TTS、audio_url、Android 远程音频播放、停止/静音控制。
8. Streaming v1：后端 NDJSON pseudo streaming、分句/分段 TTS、Android 渐进显示和 audio segment queue。
9. 小白狐状态图、状态 resolver、phase reducer、runtime min-hold anti-flicker。
10. 系统相机/相册真实图片 multipart 上传、后端 vision/image context、普通图片分享路径。
11. ParentReport model-first v2/v3：日报由模型生成；失败不展示规则日报冒充成功；v3 使用 narrative_report / tonight_parent_bridge / avoid_followup。
12. 家庭内测正式运行门禁：不允许 mock provider / mock image / mock voice 冒充成功路径。
13. 基础账号：一个孩子一个账号，家长代创建和管理，孩子不管理密码。
14. 健康陪伴底线：禁止积分、连击、排行榜、抽卡、宠物饥饿、FOMO、秘密朋友、raw transcript 导出。
```

7. Parent Report v3 在真实素材上的自然度、实用性和无监控感。

## 4. 当前最高优先级

### P0-1：验收 Parent Report Narrative v3

目标：确认家长日报真的从“拼接字段 / 监控感 / 老师评语”变成自然、短、诚实、可执行的家庭连接 note。

验收点：

```text
1. 普通聊天日：能说整体状态和轻量话题，不追问逐字聊天。
2. 图片分享日：用“展示或分享倾向”表述，不问具体哪张图。
3. 学习求助日：建议陪孩子看题意 / 第一步，不替孩子做答案。
4. 安全/隐私日：优先平静确认和成人支持，不逼问细节。
5. 素材少：诚实说明素材不多，不硬凑。
6. 模型失败：显示可重试失败态，不用规则日报冒充成功。
```

### P0-2：完成设备 QA 闭环

目标：不再只看 backend-only 或自动化测试结论，用 Redmi K60 / Honor Pad 5 走完家庭内测主链路。

必须记录：

```text
backend request_id + Android logcat timing + 视频/手工时间点。
```

没有同一慢 turn 的完整证据，不做猜测式 latency 修复。

### P0-3：文档事实源收口

目标：让新主控会话、代码 agent 和未来子会话只看少量当前事实源，不再从过期 task 文档或旧 handoff 里恢复旧设计。

处理规则：

```text
1. 已完成且被路线图吸收的单轮 task 文档删除。
2. 长期设计文档保留，但必须在本文件说明其历史部分如何解释。
3. README / AGENTS / PRODUCT_DECISIONS / 本文件互相一致。
4. 新产品决策先进入 PRODUCT_DECISIONS，再进入实现任务。
```

## 5. 后续还要做哪些事

### A. Voice-first conversation polish

状态：计划中，Parent Report v3 验收后优先。

目标：孩子始终知道小白狐是在听、识别、思考、说话、需要重说，且停止、静音、重说、取消都自然。

待做：

```text
1. 开始录音时安全停止当前 TTS 和音频队列。
2. retry / cancel / permission / network copy 全部非技术化。
3. confirm-before-send 只留在 DevSettings / 家长调试模式。
4. 确认静音只影响后续自动朗读，不隐藏文字内容。
5. 迟到 opening 不能打断儿童正在进行的 voice flow。
6. 补 Android ViewModel / UI 状态转换测试。
```

### B. First response / audio latency productization

状态：计划中。

目标：文字先快出现，小白狐 visibly thinking/speaking，音频随后自然跟上；音频慢或失败时，孩子仍感觉对话继续了。

待做：

```text
1. 持续采集 first_text_ms、tts_started_ms、first_audio_ms、Android playback start。
2. 优化首句长度和分句点。
3. 基于证据评估短句 TTS cache。
4. 不恢复 Android 系统 TTS 作为正式小白狐 fallback。
5. 不用上瘾动画掩盖延迟。
```

### C. Relationship Memory v2 / gentle continuity

状态：计划中。

目标：小白狐轻轻记得孩子关心什么，但不追踪、不施压、不贴标签、不反复拉回旧话题。

待做：

```text
1. 记忆 callback 加频率限制和衰减。
2. 区分 child-facing memory 和 parent-report memory。
3. 使用可拒绝的 soft recall 文案。
4. 尊重 bedtime、low-expression、boundary modes。
5. 禁止“我一直等你”等依赖钩子。
6. 加低压力回忆和不应回忆场景测试。
```

### D. Conversation Arc v2

状态：计划中。

目标：每次对话有轻柔弧线：欢迎、孩子主导、轻微深入、停/转话题识别、可选现实桥接、平静结束。

待做：

```text
1. 定义 ConversationArcState。
2. 标记 opening / exploring / deepening / shifting / closing / boundary-respecting。
3. 按年龄限制连续追问和长回复。
4. 把 child stop/shift intent 传给 Android、memory、report。
5. 增加短答、边界请求、高参与度、睡前 synthetic trace tests。
```

### E. Healthy Engagement metrics and QA

状态：计划中。

目标：不仅禁止不健康机制，还能观测系统是否尊重停止、换话题、睡前收尾、不过度追问、不制造依赖。

待做：

```text
1. 记录连续 agent questions。
2. 记录 child stop/shift signal 及是否被尊重。
3. 记录 bedtime extension risk。
4. 用 session duration band 作为安全信号，不作为留存目标。
5. QA 覆盖“我不想聊了”“换个话题”“我困了”“嗯/不知道”等短答。
6. 不为这些指标保存 raw transcript。
```

### F. Image sharing v2 / creative scaffolds

状态：部分实现，后续计划。

目标：“拍给小白狐看”成为孩子自然表达和创作循环，而不是作业上传工具。

待做：

```text
1. 不挤压 voice-first 控件的前提下优化 follow-up chips。
2. 只有在孩子意图明确时给 story / name / what-happened-next 创作提示。
3. homework 只作为明确学习分支。
4. 家长日报继续用宽泛表达，不追问具体图片。
5. CameraX 和长期 gallery 继续后置。
```

### G. Learning scaffold v2

状态：计划中。

目标：帮助孩子理解题意、找已知、走一步，而不是外包最终答案。

待做：

```text
1. 改进 homework intent detection。
2. 区分“分享 worksheet 图片”和“求助做题”。
3. 增加 scaffold state：理解题意 -> 已知信息 -> 提示 -> 孩子尝试 -> 下一提示。
4. 补最终答案 guardrail 和 tests。
5. 家长日报学习部分保持家庭连接，不写老师评价。
```

### H. Parent governance and compliance foundation

状态：后置，但家庭 beta 扩大前必须做。

目标：形成可扩展的家长治理层，包括同意、保留、删除、账号安全和儿童数据控制。

待做：

```text
1. 加固 auth/session。
2. 增加数据保留和删除控制。
3. UI 说明 audio/image/text retention policy。
4. 默认只导出 summary，不导出 raw transcript。
5. 单孩子路径稳定后，再做 multi-child / multi-guardian。
```

### I. Growing Nest / expression footprint UI

状态：延期。

目标：给家长和孩子一个低压力、非游戏化的成长回看面，而不是积分、打卡或成就系统。

待做：

```text
1. 只用摘要和孩子同意保留的作品，不展示 raw transcript。
2. 避免 streak、badge、ranking。
3. 使用“最近的小事情”“我们做过/聊过的小东西”这类低压力框架。
4. 必须在日报、语音循环、记忆连续性稳定后再做。
```

## 6. 文档治理规则

```text
1. README：只做项目入口和当前状态，不承载详细路线图。
2. AGENTS：只放代码 agent 必须遵守的硬规则，不写长篇历史。
3. PRODUCT_DECISIONS：记录家长/产品负责人确认的产品决策，代码不得冲突。
4. 本文件：记录当前实现现实、历史偏差修正、后续路线图。
5. 长期设计文档：保留，如 voice、image、healthy engagement、freedom-first，但历史段落以本文件解释为准。
6. 单轮 CODE_AGENT_TASK 文档：完成并被吸收后删除；如需追历史看 git commit。
7. Prompt / copy / 家长日报文案：必须由主控会话设计或明确批准，代码 agent 只负责实现和测试。
```
